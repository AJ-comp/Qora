# Qora

**量子プログラミングを学びやすくする学習用言語。**
Q#·C# に近い構文で量子回路を書き、検証済みの **OpenQASM 3** を出力します。

**Qora は量子プログラミングを学ぶための学習用言語です。** 量子コードは Qiskit や Cirq のような
Python ライブラリで書かれることも多く、また理論をすでに知っている前提の研究用言語で書かれることもあります。
初めて学ぶ立場では、すばやい実験と正確な概念の間を行き来するのが簡単ではありません。

Qora はその間を埋めることに焦点を置いています。なじみのあるプログラミング構文で回路を書くと、
コンパイラがまず「このプログラムは本当に量子コンピュータにできることか」を確認します。
そのあと標準 **OpenQASM 3** を作り、一行ずつ読んだり実際のツールチェーンへ渡したりできるようにします。

この方向性は設計基準にもつながっています:

- **エラーは出力より先に出ます。** コンパイルを通過したなら、その OpenQASM は有効です。
- **コンパイラの内部を確認できます。** コマンド一つで、ソースが AST、型付き IR、シンボルテーブル、合成された逆演算、QASM へ変わる流れを見られます。
- **量子のルールはコンパイラが直接検査します。** すでに使ったキュービットの再利用、測定の逆転、単一キュービットの位置へのレジスタ渡しはコンパイル段階で止められます。

OpenQASM でそのまま安全な名前は変更されません。
ただし、ユーザー名がキーワード、stdgates のゲート名、またはすでに出力された別の名前と衝突するときだけ、
Qora は出力名を変更し、その理由を `// Qora:` ノートとして残します。

## クイックスタート

最速の方法は VS Code 拡張機能から始めることです。
コンパイラ全体が拡張機能の中に入っているため、**追加でインストールするものはなく**、.NET も Python も不要です。

1. VS Code マーケットプレイスから [Qora Language](https://marketplace.visualstudio.com/items?itemName=qora-lang.qora-language) をインストールします。
2. `bell.qor` というファイルを作り、上の例を貼り付けます。
3. 保存すると、すぐに次の機能を使えます。

- **入力中に**エラーへ下線が付き、下線にホバーすると理由を確認できます。`CNOT(q[0], q[0])` も試してみてください。
- ゲートやキーワードにホバーすると、`H`、`use`、`M` などの短い説明が出ます。
- **`Ctrl+Shift+P` → `Qora: Transpile to OpenQASM`** を実行すると、コンパイル結果が横のエディターに開きます。
- **`Qora: Show Compilation Stages`** は、保存するたびにソース → AST → IR → シンボルテーブル → (逆 IR) → QASM の流れを表示します。

リポジトリから直接実行する場合は、.NET 10 SDK だけで足ります。

```bash
git clone https://github.com/AJ-comp/Qora.git
cd Qora
dotnet run --project src/Qora          # デモプログラムをパースして OpenQASM を出力
```

VS Code 拡張機能も同じバイナリを `qora --json` で実行し、コンパイルごとに `{success, qasm, errors[]}` 形式の JSON 一行を受け取ります。

## 5分でわかる言語ツアー

**キュービットとゲート。** 新しいキュービットレジスタは `use q = Qubit[N];` の形で割り当て、配列のようにインデックスでアクセスします。
ここでの `use` は C# の `using` のようにブロック末尾の自動解放を表すものではなく、Qora プログラムで使うキュービットレジスタを作る構文です。
ゲート呼び出しは関数呼び出しと同じ形です。

```
use q = Qubit[3];              // キュービットレジスタ q を割り当て、サイズは 3
H(q[0]);                       // 重ね合わせ
CNOT(q[0], q[1]);              // もつれ
Rx(pi/2, q[2]);                // 回転は角度が先
Reset(q[2]);                   // |0⟩ に戻す
```

組み込みゲート: `H X Y Z S T` · `CNOT/CX CY CZ SWAP CCX` · `Rx Ry Rz` · `Reset ResetAll`。

**測定**はキュービットの測定結果を古典 `bit` に入れる演算であり、Qora で `bit` を得る方法も測定だけです。

```
bit r = M(q[0]);               // 測定、元に戻せません
r = M(q[0]);                   // 同じ bit に再測定
```

**古典制御フロー**は期待どおりで、測定結果に反応できます:

```
const int n = 2;
var count = 0;
for i in 0..n {  H(q[i]);  }
if (r == 1) {  X(q[1]);  } else {  Z(q[1]);  }
while (count < 2) {  count = count + 1;  }
repeat {  H(q[0]);  r = M(q[0]);  } until (r == 1);
```

ユーザー定義演算は、名前とパラメータの後ろにブロックを付けて書き、`Main` がエントリポイントになります。
**ファンクタ**はユーザー定義演算やゲートを変形します:

```
Prep(Qubit[2] q)
{
    H(q[0]);
    Controlled X(q[0], q[1]);   // 任意のゲートに制御を追加
    T(q[1]);
}

Adjoint Prep(q);                // ★ コンパイラが Prep の逆演算を合成します:
                                //   ゲートは逆順+逆に、for ループは逆走、
                                //   if 分岐は内部反転、ネスト呼び出しは推移的に
```

この `Adjoint` の一行は Qora の代表的な機能の一つで、「この計算を元に戻して」という要求をコード一行で表すと、コンパイラが実際の逆向き回路を作ります。
また、この機能は [自動 uncomputation](docs/TODO.md) への出発点でもあります。

**名前空間と import** を使うと、大きなプログラムを複数ファイルに分けられます。
`import` は Python のようにファイルパスを文字列で書き、拡張子も含めます。

`gates_lib.qor`

```qora
namespace GatesLib
{
    Bell(Qubit[2] q)
    {
        H(q[0]);
        CNOT(q[0], q[1]);
    }
}
```

`rotations.qor`

```qora
namespace Rotations
{
    Tilt(Qubit q)
    {
        Ry(pi/4, q);
        Rz(pi/8, q);
    }
}
```

`main.qor`

```qora
import "gates_lib.qor";         // 同じディレクトリの gates_lib.qor を読み込む
import "rotations.qor";         // 同じディレクトリの rotations.qor を読み込む

namespace App
{
    open GatesLib;
    open Rotations;

    Prepare(Qubit[2] q)
    {
        Bell(q);                // open 経由で GatesLib.Bell を名前だけで呼ぶ
        Tilt(q[0]);             // open 経由で Rotations.Tilt を名前だけで呼ぶ
    }
}

Main()
{
    use q = Qubit[2];
    App.Prepare(q);             // App 名前空間の演算を修飾付きで呼ぶ
    Rotations.Tilt(q[1]);       // import した別ファイルの演算を直接修飾して呼ぶ
}
```

名前解決の規則は、名前空間を使うなじみのあるプログラミング言語の方式に近いものです。
名前は自分の名前空間、グローバル名前空間、`open` した名前空間の順に探します。
ただし、`open` した二つの場所に同じ名前がある場合、Qora は黙って片方を選ばず、候補を表示するコンパイルエラーを出します。

`import "lib/gates.qor";` は、現在のファイルがあるディレクトリを基準に `lib/gates.qor` を読みます。
パスは引用符の中にそのまま書き、拡張子も含めます。
同じファイルは複数の経路から到達しても一度だけロードされ、循環 import があるとそのチェーンをエラーメッセージで表示します。

組み込みゲート名も同じ規則に従い、組み込みゲートは暗黙の `Qora.Intrinsic` 名前空間にあります。
そのため、名前空間の中では `Rx` のような名前の演算を直接定義できます。
ただし、ある呼び出しがユーザー定義演算と組み込みゲートの両方を意味しうる場合、明示的な曖昧性エラーになります。
このとき、ユーザー定義演算は `L.Rx(q)`、組み込みゲートは `Qora.Intrinsic.Rx(pi/2, q)` のように書きます。

スコープ内に同じ名前のユーザー候補がなければ、裸のゲート名はそのまま組み込みゲートを意味します。
つまり、ドキュメントで学んだゲート名をプログラムでもそのまま使えます。

**間違えたときは、コンパイラが理由を教えてくれます** — 一度のコンパイルですべての違反を、出力の前に:

```
[QSEM001] in `Main`: `Adjoint Meas` cannot be compiled: it measures a qubit,
          and measurement is irreversible
[QSEM006] in `Main`: `Bell` expects 1 argument(s) but got 2
[QSEM016] in `Main`: index `q[5]` is out of range; `q` has 2 qubit(s) (valid: 0..1)
[QSEM018] in `Go`: `Flip` is ambiguous here: it could be `LibA.Flip` or `LibB.Flip`
          — qualify the call (e.g. `LibA.Flip(...)`)
[QSEM021] imports form a cycle: main.qor -> lib.qor -> main.qor — remove one of these imports
```

**プログラムの実行。** ユーザー定義演算、ループ、古典変数、`while` を含む Qora プログラムは、Amazon Braket のローカルシミュレータで実行できます:

```bash
pip install amazon-braket-sdk            # Python 3.10–3.13
python tools/run_braket.py yourfile.qor --shots 1000
```

両方の実行経路は、実測統計によるリリース検証にも使われます。
`tools/braket_validation.py` は、全ユーザー定義演算の `Adjoint` が状態を `|0…0⟩` に戻すことまで含めて、言語全体が実行できることを確認します。
`tools/qiskit_validation.py` は、Qiskit の `qasm3.loads(...)` が受け付ける平坦な部分集合を検証します。
ただし Qiskit のインポータはまだ `def`、`int`、変数ループインデックスを読めないため、Qiskit と実機へつなぐ平坦化出力モードを追加する予定です。

## コンパイラの仕組み

正式な段階別の説明は  **[Qora コンパイルパイプライン設計](https://aj-comp.github.io/Qora/ja/architecture.html)** にあります。

要約すると、Qora はパーサーが作った AST を自分の IR に移し、その IR の上で名前解決、検証、
特殊化、逆合成、名前整理、出力を順番に進めます。

| グループ | 段階 | 出力 | 役割 |
|---|---|---|---|
| 1. 読み取る | ソース → AST | エンジン AST | Janglim がソーステキストを構文木に変えます |
| 2. Qora が所有する | AST → Qora IR | 型付き IR | `QoraLowering` がパーサーの木を Qora 専用の構造へ移します |
| 3. 名前を解決する | import ロード → 統合 → 名前解決 | FQN IR | 複数ファイルを統合し、すべての呼び出し先を完全修飾名に固定します |
| 4. 意味を検証する | 検証 | エラー一覧またはクリーンな IR | 問題があればすべてのエラーを集めて、ここで停止します |
| 5. 変換する | ジェネリック特殊化 → adjoint 実体化 | 出力準備済み IR | `Qubit[n]` を実際のサイズにし、`Adjoint` 呼び出しを合成された逆演算へ変えます |
| 6. 出力を準備する | 名前マングリング → 参照検査 | OpenQASM 安全 IR | OpenQASM の名前衝突を避け、すべての参照が解決されているか確認します |
| 7. 出力する | OpenQASM 出力 | OpenQASM 3 | 最後のエミッタが、決定済みの IR をテキストとして出力します |

- **型付き IR**(`src/Qora.Core/Ir/`)はコンパイラが最後まで所有します; パースエンジンは lowering の
  境界で止まります。
- **モジュールローダー**は `import` グラフ全体を一つのプログラムに統合します。<br>その後 **リゾルバ** が演算の呼び出し先を完全修飾名(FQN)に固定し、以降の各呼び出しは一つの演算だけを指します。<br>名前空間を使う他のプログラミング言語と同じように、曖昧な場合はエラーになります。詳しくは [設計ドキュメント](docs/namespaces-design.md) にあります。
- **バリデータ**(QSEM001–025)は収集型です。<br>一度のコンパイルで、引数の種類と個数、レジスタサイズ、インデックス範囲、予約名、再帰、誤った位置の `use`、未宣言または宣言前に使われた識別子、逆演算できない `Adjoint` 対象までまとめて報告します。<br>また、バリデータが使う統合シンボルテーブルもコンパイル段階ビューに表示されます。
- **ネームマングラー**は名前空間のドットを平坦化し(`MyLib.Bell` → `MyLib_Bell`)、出力識別子が
  キーワード、stdgates のゲート名、またはすでに出力された別の名前と衝突するときだけ `_` を付けます。
  エラーとコンパイル段階ビューは元の名前/FQNを保ち、QASM だけが衝突しない名前を表示します。
- **adjoint 実体化器**は純粋な IR→IR インバータを使い、全演算 `Adjoint` 呼び出しを合成された
  逆演算へ変換してから名前マングリングへ渡します。将来、自動 uncomputation を注入するパスも
  この機構をそのまま再利用します。
