들어가며. 왜 Qora로 양자 게이트를 배울까?

양자컴퓨팅을 처음 배우면 낯선 말이 한꺼번에 나와요.
예를 들어 큐비트, 중첩, 진폭, 위상, 간섭, 얽힘, 측정 같은 말인데, 각 단어를 따로 들으면 멋있어 보이지만,
실제로 회로를 읽으려고 하면 금방 막히게 돼요.

예를 들어 H 게이트를 처음 보면, 보통 아래와 같은 식을 먼저 만날 거예요.

H 게이트(|0⟩) = (|0⟩ + |1⟩) / √2

이 문장 자체는 맞지만, 처음 배우는 사람에게는 |0⟩가 무엇인지, 왜 |1⟩와 더하는지, 왜 2가 아니라 √2로 나누는지, 이 식을 측정하면 무엇이 나오는지 같은 질문이 한꺼번에 생겨서 설명이 금방 어려워져요.

여기서는 이렇게 살펴볼게요

이런 질문을 그냥 넘기지 않고, 작은 회로 하나를 놓고 상태가 어떻게 바뀌는지 한 단계씩 따라가보면서,
그 과정마다 Qora 코드도 함께 확인해봐요.

Qora는 여기서 두 가지 역할을 맡아요.

첫째 양자 회로를 짧고 읽기 쉬운 형태로 적을 수 있게 해 줘요.
둘째 추상적인 설명을 다시 실행 가능한 코드로 돌려놓아요.
첫 번째 Qora 코드

큐비트 하나에 H 게이트를 적용하고 측정해보자

operation Main() {
    // 큐비트 하나 준비
    use q = Qubit[1];

    // 첫 번째 큐비트 q[0]에 H 게이트 적용
    H(q[0]);

    // q[0]을 측정하고 결과를 result에 저장
    bit result = M(q[0]);
}

자, 이제 각 줄이 큐비트 상태와 진폭을 어떻게 바꾸는지 차례대로 읽어볼까요?

1부

양자를 읽기 위한 준비를 해보자

1부에서는 게이트 설명에 들어가기 전에 필요한 공통 언어를 먼저 만들어볼게요.
모든 양자역학을 다루려는 자리는 아니에요.
H 게이트, X 게이트, Z 게이트, CNOT 게이트 설명을 읽기 위해 꼭 필요한 개념만 먼저 잡고 갈 거예요.

1장. 큐비트 상태에는 무엇이 들어 있을까?

자, 이제 큐비트 상태가 무엇을 담고 있는지 살펴볼까요?
큐비트 하나를 측정하면 측정 결과값 0 또는 측정 결과값 1, 둘 중 하나가 나와요.
다만 큐비트 상태는 단순히 측정 결과값 하나만으로 설명되지 않아요.

측정하기 전의 큐비트 상태에는 가능한 측정 결과값마다 숫자가 하나씩 붙어 있고, 이 숫자를 진폭이라고 불러요.
큐비트 하나에는 가능한 측정 결과값이 두 개 있으므로, 측정 결과값 0에 붙은 진폭과 측정 결과값 1에 붙은 진폭을 함께 봐야 해요.

|0⟩와 |1⟩는 한 큐비트 상태의 이름이에요 |0⟩는 큐비트를 측정하면 측정 결과값 0이 확실히 나오는 상태를 가리켜요.
|1⟩는 큐비트를 측정하면 측정 결과값 1이 확실히 나오는 상태를 가리켜요.
아직은 이 기호를 어려운 수식으로 보지 말고, 측정 결과값 0 쪽에 진폭이 전부 있는 상태와 측정 결과값 1 쪽에 진폭이 전부 있는 상태를 부르는 이름으로 보면 돼요.

먼저 |0⟩ 상태를 보자

|0⟩ 상태는 측정 결과값 0의 진폭이 1이고, 측정 결과값 1의 진폭이 0인 상태예요.

가능한 측정 결과값진폭 값측정하면 나올 확률
측정 결과값 01100%
측정 결과값 100%

이 상태를 측정하면 측정 결과값 0이 반드시 나와요.
반대로 측정 결과값 1의 진폭은 0이므로 측정 결과값 1은 나오지 않아요.
Qora에서 큐비트를 새로 준비하면 기본적으로 |0⟩ 상태에서 시작한다고 생각하면 돼요.

Qora로 실습해보기

준비한 직후의 큐비트를 측정해보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    // 측정 결과값 0의 진폭 = 1
    // 측정 결과값 1의 진폭 = 0
    use q = Qubit[1];

    // q[0] 측정
    // 이상적인 실행 환경: 측정 결과값 0
    bit result = M(q[0]);
}

이번에는 |1⟩ 상태를 보자

|1⟩ 상태는 반대예요.
측정 결과값 0의 진폭이 0이고, 측정 결과값 1의 진폭이 1인 상태예요.

가능한 측정 결과값진폭 값측정하면 나올 확률
측정 결과값 000%
측정 결과값 11100%

Qora에서는 |0⟩ 상태로 준비한 큐비트에 X 게이트를 적용하면 |1⟩ 상태를 만들 수 있어요.
X 게이트는 뒤에서 자세히 배우겠지만, 일단은 측정 결과값 0과 측정 결과값 1을 뒤집는 게이트로 보면 돼요.

Qora로 실습해보기

X 게이트로 |1⟩ 상태를 만들어보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // X 게이트 적용
    // |0⟩ 상태가 |1⟩ 상태로 바뀜
    X(q[0]);

    // 이상적인 실행 환경: 측정 결과값 1
    bit result = M(q[0]);
}
|0⟩ + |1⟩와 |01⟩: 헷갈리기 쉬운 표기 (|0⟩ + |1⟩) / √2는 한 개의 큐비트가 측정 결과값 0과 측정 결과값 1에 대한 진폭을 동시에 가진 상태인 반면에 |01⟩은 두 개의 큐비트가 있을 때 쓰는 상태 표기예요.

2장. 진폭과 확률은 어떻게 연결될까?

이제 진폭과 확률이 어떻게 연결되는지 살펴볼까요?
큐비트 상태에는 가능한 측정 결과값마다 진폭이 붙어 있어요.
그렇다면 진폭이 곧 확률일까요?
정확히 말하면, 진폭과 확률은 다르게 봐야 해요.
측정 확률은 진폭 자체가 아니라, 진폭의 길이를 제곱해서 얻는 값이에요.

측정 확률 = 진폭 길이²

|0⟩ 상태에서는 측정 결과값 0의 진폭이 1이에요.
진폭의 길이를 제곱해도 1이므로, 측정 결과값 0이 나올 확률은 100%가 돼요.

측정 결과값 0의 확률 = 1² = 1 = 100%
측정 결과값 1의 확률 = 0² = 0 = 0%

왜 50% 확률인데 진폭은 1/2이 아닐까?

처음에는 측정 결과값 0과 측정 결과값 1이 반반이면, 진폭도 각각 1/2일 것처럼 보여요.
하지만 진폭은 1/2이 아니라 1/√2예요.

틀린 생각

진폭을 1/2로 두면 어떻게 될까?

각 측정 결과값의 확률은 25%가 돼요.

(1/2)² + (1/2)² = 1/4 + 1/4 = 1/2

전체 확률이 50%밖에 되지 않으므로 큐비트 상태로 맞지 않아요.

올바른 진폭

진폭을 1/√2로 두면 왜 맞을까?

각 측정 결과값의 확률이 50%로 맞춰져요.

(1/√2)² + (1/√2)² = 1/2 + 1/2 = 1

전체 확률도 100%로 맞아요.

Qora로 실습해보기

H 게이트로 50% 상태를 만들어보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    // 측정 결과값 0의 진폭 = 1
    // 측정 결과값 1의 진폭 = 0
    use q = Qubit[1];

    // H 게이트 적용
    // 측정 결과값 0의 진폭 = 1/√2
    // 측정 결과값 1의 진폭 = 1/√2
    // 두 측정 결과값의 확률은 각각 50%
    H(q[0]);

    // q[0] 측정
    // 이상적인 실행 환경: 측정 결과값 0과 1이 각각 약 50%
    bit result = M(q[0]);
}

여기서 중요한 점은 H 게이트가 확률을 직접 반으로 나누는 게 아니라는 점이에요.
H 게이트가 바꾸는 것은 확률 자체가 아니라 진폭이고, 그 진폭을 제곱했을 때 50%라는 확률이 나와요.

3장. 중첩은 그냥 "모르는 상태"일까?

H 게이트를 적용한 뒤의 상태를 다시 한번 볼까요?

(|0⟩ + |1⟩) / √2

이 상태를 중첩 상태라고 불러요.
다만 중첩 상태를 "큐비트가 사실은 0 또는 1인데 우리가 모르는 상태"라고 이해하면 곤란해요.

동전을 던진 뒤 손으로 덮어 두면, 동전은 이미 앞면이거나 뒷면으로 정해져 있어요.
단지 우리가 모를 뿐이에요.
하지만 양자 중첩은 그와 다른 이야기예요.
측정하기 전의 큐비트 상태에는 측정 결과값 0의 진폭과 측정 결과값 1의 진폭이 함께 들어 있으니까요.

왜 확률만 보면 부족할까요? 진폭은 나중에 다른 게이트를 만나 서로 더해지거나 상쇄될 수 있어요.
그래서 양자 상태를 이해할 때는 측정 확률만 보면 안 되고, 확률 뒤에 있는 진폭의 부호와 방향까지 봐야 해요.

같은 50%처럼 보여도 정말 같은 상태일까?

상태 A

측정 결과값 1의 진폭이 양수인 상태

(|0⟩ + |1⟩) / √2

바로 측정하면 측정 결과값 0과 1이 각각 50% 확률로 나와요.

상태 B

측정 결과값 1의 진폭이 음수인 상태

(|0⟩ - |1⟩) / √2

바로 측정하면 이 상태 역시 측정 결과값 0과 1이 각각 50% 확률로 나와요.

두 상태는 바로 측정하면 같은 확률을 보이지만, 같은 상태는 아니에요.
측정 결과값 1의 진폭 부호가 다르고, 이 부호 차이가 다른 게이트를 한 번 더 적용할 때 결과 차이를 만들어요.

4장. 간섭은 언제 생길까?

간섭은 양자컴퓨팅에서 가장 중요한 개념 중 하나예요.
그렇다고 처음부터 거창하게 생각할 필요는 없고, 여기서는 다음 문장 하나로 출발해봐요.

같은 측정 결과값에 도착한 진폭 기여들은 서로 더해져요.

어떤 게이트는 입력 진폭을 여러 측정 결과값 쪽으로 나눠 보내요.
그렇게 나뉜 진폭 기여가 같은 측정 결과값에 도착할 수 있고, 바로 그 순간, 같은 측정 결과값에 도착한 진폭 기여들이 서로 더해져요.

보강 간섭

같은 부호의 진폭 기여: 보강

+1/2 + +1/2 = +1

측정 결과값 0의 진폭이 커져요.

상쇄 간섭

반대 부호의 진폭 기여: 상쇄

+1/2 + -1/2 = 0

측정 결과값 1의 진폭이 사라져요.

H 게이트를 두 번 적용하면 간섭이 어떻게 보일까?

Qora로 실습해보기

H 게이트를 두 번 적용해보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // 첫 번째 H 게이트 적용
    // 측정 결과값 0과 측정 결과값 1의 진폭이 모두 1/√2
    H(q[0]);

    // 두 번째 H 게이트 적용
    // 측정 결과값 0의 진폭 기여는 보강
    // 측정 결과값 1의 진폭 기여는 상쇄
    H(q[0]);

    // 이상적인 실행 환경: 측정 결과값 0
    bit result = M(q[0]);
}

첫 번째 H 게이트 뒤에는 두 진폭이 남아 있어요.

측정 결과값 0의 진폭 = 1/√2
측정 결과값 1의 진폭 = 1/√2

두 번째 H 게이트를 적용하면, 측정 결과값 0 쪽으로 도착한 진폭 기여는 이렇게 더해져요.

+1/2 + +1/2 = +1

측정 결과값 1 쪽으로 도착한 진폭 기여도 같은 방식으로 계산할 수 있어요.

+1/2 + -1/2 = 0

그래서 최종 상태는 다시 |0⟩ 상태로 돌아오게 돼요.
이 계산은 H 게이트 장에서 조금 더 자세히 다시 살펴볼게요.

5장. 양자 게이트는 왜 되돌릴 수 있어야 할까?

이제 양자 게이트가 왜 되돌릴 수 있어야 하는지 살펴볼까요?
양자 게이트는 아무 변환이나 될 수 없고, 반드시 되돌릴 수 있어야 해요.
되돌릴 수 있다는 말은, 출력 상태를 알면 입력 상태가 무엇이었는지 거꾸로 복원할 수 있어야 한다는 뜻이에요.

입력 상태출력 상태문제
입력 A출력 C출력 C만 보고 입력을 알 수 없음
입력 B출력 C

이런 변환은 거꾸로 되돌릴 수 없어요.
출력 C만 보고는 입력이 A였는지 B였는지 알 수 없기 때문이에요.
그래서 양자 게이트는 이런 식으로 정보를 잃어버리는 변환이 될 수 없어요.

H 게이트의 마이너스 부호는 왜 필요할까?

H 게이트는 |0⟩ 상태와 |1⟩ 상태를 다음처럼 바꿔요.

H 게이트(|0⟩) = (|0⟩ + |1⟩) / √2
H 게이트(|1⟩) = (|0⟩ - |1⟩) / √2

여기서 중요한 부분은 두 번째 줄의 마이너스 부호예요.
만약 마이너스 부호가 없다면 두 입력 상태가 같은 출력 상태로 합쳐져요.

H 게이트(|0⟩) = (|0⟩ + |1⟩) / √2
H 게이트(|1⟩) = (|0⟩ + |1⟩) / √2

그러면 출력 상태만 보고는 입력이 |0⟩였는지 |1⟩였는지 알 수 없고, 변환을 거꾸로 되돌릴 수도 없어요.
따라서 그런 변환은 양자 게이트로 인정할 수 없게 돼요.

여기서 핵심 H 게이트의 마이너스 부호는 장식이 아니에요.
H 게이트가 되돌릴 수 있는 양자 게이트가 되기 위해 필요한 부호이고, 이 마이너스 부호 덕분에 H 게이트를 두 번 적용하면 처음 상태로 돌아올 수 있어요.
H 게이트(H 게이트(|0⟩)) = |0⟩

6장. Qora 코드는 어떻게 읽을까?

마지막으로, 앞으로 자주 보게 될 Qora 코드의 기본 모양을 함께 살펴볼게요.
Qora 프로그램은 operation으로 시작하고, 가장 먼저 실행되는 작업에는 보통 Main이라는 이름을 붙여요.

기본 형태

가장 작은 Qora 프로그램부터 보자

operation Main() {
    // 여기에 실행할 양자 회로 작성
}

큐비트 준비하기

큐비트 준비는 use로 시작해요.

use q = Qubit[1];

이 코드는 큐비트 하나를 준비한 뒤, 그 큐비트 묶음에 q라는 이름을 붙여요.
Qubit[1]은 큐비트가 한 개라는 뜻이고, 그 큐비트는 q[0]으로 가리켜요.

게이트 적용하기

게이트는 함수 호출처럼 적어주면 돼요.

H(q[0]);

이 코드는 q[0]에 H 게이트를 적용하는 줄이에요.
X 게이트는 X(q[0]);처럼, 두 큐비트 게이트인 CNOT은 CNOT(q[0], q[1]);처럼 쓰면 돼요.

측정하기

측정에는 M을 써요.

bit result = M(q[0]);

이 줄은 q[0]을 측정하고, 그 결과를 고전 비트 result에 저장해요.
측정하기 전의 큐비트는 진폭을 가진 양자 상태이고, 측정한 뒤에는 측정 결과값 0 또는 측정 결과값 1 중 하나가 고전 비트에 저장돼요.

주석 쓰기

Qora에서는 //로 한 줄 주석을 쓸 수 있어요.
주석은 회로 실행에 영향을 주지 않으므로, 이 책의 Qora 실습 코드에서도 상태 변화를 적을 때 자주 사용할게요.

주석이 있는 Qora 코드

코드 줄마다 상태 변화를 적어보자

operation Main() {
    // 큐비트 하나 준비
    use q = Qubit[1];

    // q[0]에 H 게이트 적용
    H(q[0]);

    // q[0] 측정
    bit result = M(q[0]);
}

고전적인 NOT 연산과 가장 비슷하게 느껴지는 게이트이므로, 양자 게이트를 처음 익히기에 좋은 출발점으로 삼기 좋아요.

7장. X 게이트는 무엇을 뒤집을까?

X 게이트는 한 큐비트에서 가장 먼저 만나기 좋은 게이트예요.
고전적인 NOT 연산처럼 느껴지기 때문이에요.
다만 양자 회로에서는 측정 결과값 자체를 뒤집는다고 말하기보다, 측정 결과값 0에 있던 진폭과 측정 결과값 1에 있던 진폭의 위치를 서로 바꾼다고 읽는 편이 더 정확해요.

X 게이트는 측정 결과값 0의 진폭과 측정 결과값 1의 진폭을 서로 바꿔요.
입력 상태X 게이트 적용 후 상태측정하면 어떻게 보일까?
|0⟩|1⟩측정 결과값 1이 나와요
|1⟩|0⟩측정 결과값 0이 나와요

일반적인 한 큐비트 상태를 측정 결과값 0의 진폭 a₀와 측정 결과값 1의 진폭 a₁로 쓰면, X 게이트는 두 값을 서로 바꿔요.
그래서 X 게이트 적용 전에는 측정 결과값 0 쪽에 있던 진폭이 적용 후에는 측정 결과값 1 쪽으로 가고, 적용 전에는 측정 결과값 1 쪽에 있던 진폭이 적용 후에는 측정 결과값 0 쪽으로 와요.

X 게이트 적용 전: 측정 결과값 0의 진폭 = a₀, 측정 결과값 1의 진폭 = a₁
X 게이트 적용 후: 측정 결과값 0의 진폭 = a₁, 측정 결과값 1의 진폭 = a₀
X 게이트는 측정한 뒤의 숫자를 바꾸는 게 아니에요 X 게이트는 측정 전에 큐비트 상태의 진폭 배치를 바꿔요.
그 다음에 측정하면 바뀐 진폭 배치에 따라 측정 결과가 나와요.
그래서 X 게이트를 “결과를 뒤집는 명령”이 아니라 “측정 결과값 0과 1에 붙은 진폭의 자리를 바꾸는 게이트”로 읽어야 해요.

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // X 게이트 적용
    // |0⟩ 상태가 |1⟩ 상태로 바뀜
    X(q[0]);

    // 이상적인 실행 환경: 측정 결과값 1
    bit result = M(q[0]);
}

8장. H 게이트는 진폭을 어떻게 나눌까?

H 게이트는 0을 그대로 0에 묶어 두지 않고, 한 값에 모여 있던 진폭을 측정 결과값 0과 측정 결과값 1 쪽으로 나눠 보내는 게이트예요.
그래서 |0⟩ 상태에 H 게이트를 적용하면 측정 결과값 0의 진폭과 측정 결과값 1의 진폭이 둘 다 생겨요.

H 게이트(|0⟩) = (|0⟩ + |1⟩) / √2

여기서 중요한 점은 측정 확률이 아니라 진폭이 먼저 바뀐다는 점이에요.
H 게이트가 만든 두 진폭의 크기는 각각 1/√2이고, 이 값을 제곱하면 각각 50%가 돼요.

가능한 측정 결과값H 게이트 적용 후 진폭측정 확률
측정 결과값 01/√250%
측정 결과값 11/√250%

H 게이트의 정확한 진폭 변환 규칙

H 게이트는 입력 상태의 한쪽 진폭만 사용하지 않아요.
적용 전 측정 결과값 0의 진폭과 적용 전 측정 결과값 1의 진폭이 모두 적용 후의 두 측정 결과값에 기여해요.
같은 측정 결과값으로 도착한 진폭 기여들은 서로 더해져요.

적용 후 측정 결과값 0의 진폭 = a₀ × 1/√2 + a₁ × 1/√2
적용 후 측정 결과값 1의 진폭 = a₀ × 1/√2 + a₁ × (-1/√2)
마이너스 부호는 입력 1에서 출력 1로 가는 기여에만 붙어요 네 가지 입력-출력 조합 중 입력도 1이고 출력도 1인 조합에만 마이너스 부호가 붙어요.
이 부호 덕분에 H 게이트를 두 번 적용했을 때 측정 결과값 0 쪽 진폭은 보강되고, 측정 결과값 1 쪽 진폭은 상쇄돼요.

H 게이트를 두 번 적용하면 왜 다시 돌아올까?

|0⟩ 상태에서 시작하면 첫 번째 H 게이트 뒤의 두 진폭은 모두 1/√2예요.
이 값을 H 게이트 변환 규칙에 다시 넣으면 측정 결과값 0의 진폭은 1이 되고, 측정 결과값 1의 진폭은 0이 돼요.

두 번째 H 뒤 측정 결과값 0의 진폭 = 1/2 + 1/2 = 1
두 번째 H 뒤 측정 결과값 1의 진폭 = 1/2 - 1/2 = 0
H 게이트(H 게이트(|0⟩)) = |0⟩

    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // 첫 번째 H 게이트 적용
    // 진폭이 측정 결과값 0과 1로 나뉨

    // 두 번째 H 게이트 적용
    // 보강과 상쇄가 일어나 다시 |0⟩ 상태가 됨
    H(q[0]);

    // 이상적인 실행 환경: 측정 결과값 0
    bit result = M(q[0]);
}

9장. Z 게이트는 무엇의 부호를 바꿀까?

Z 게이트는 측정 결과값 0의 진폭은 그대로 두고, 측정 결과값 1의 진폭에만 마이너스 부호를 붙이는 게이트예요.
그래서 Z 게이트를 바로 측정만 해 보면 아무 일도 일어나지 않은 것처럼 보일 수 있어요.
하지만 나중에 H 게이트처럼 진폭을 다시 섞는 게이트를 만나면, 그 부호 차이가 측정 결과를 바꿔요.

입력 상태Z 게이트 적용 후 상태바로 측정하면?
|0⟩|0⟩측정 결과값 0
|1⟩-|1⟩측정 결과값 1

측정 확률은 진폭의 길이를 제곱해서 얻어요.
1과 -1은 부호가 다르지만 길이는 같으므로, 바로 측정하면 둘 다 같은 확률을 만들어요.
그래서 Z 게이트의 효과는 “확률을 바꾸는 효과”라기보다 “나중에 간섭할 때 드러나는 부호 변화”로 읽는 편이 좋아요.

Z 게이트의 효과는 H 게이트를 다시 만날 때 드러나요

먼저 H 게이트로 |0⟩ 상태를 (|0⟩ + |1⟩) / √2 상태로 만들어요.
그 다음 Z 게이트를 적용하면 측정 결과값 1의 진폭 부호가 바뀌어서 (|0⟩ - |1⟩) / √2 상태가 돼요.
마지막으로 H 게이트를 한 번 더 적용하면 상쇄와 보강의 방향이 바뀌어서 최종 상태가 |1⟩이 돼요.

|0⟩ → H 게이트 → (|0⟩ + |1⟩) / √2 → Z 게이트 → (|0⟩ - |1⟩) / √2 → H 게이트 → |1⟩
Qora로 실습해보기

H, Z, H 순서로 적용해보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // H 게이트 적용
    // 측정 결과값 0과 1의 진폭이 모두 1/√2
    H(q[0]);

    // Z 게이트 적용
    // 측정 결과값 1의 진폭 부호가 음수로 바뀜
    Z(q[0]);

    // H 게이트 적용
    // 부호 차이가 간섭으로 드러나 |1⟩ 상태가 됨
    H(q[0]);

    // 이상적인 실행 환경: 측정 결과값 1
    bit result = M(q[0]);
}

10장. CNOT 게이트는 두 큐비트를 어떻게 연결할까?

CNOT 게이트는 두 큐비트에 작용하는 게이트예요.
첫 번째 큐비트를 제어 큐비트라고 부르고, 두 번째 큐비트를 대상 큐비트라고 불러요.
제어 큐비트가 측정 결과값 1인 쪽에 있을 때만 대상 큐비트에 X 게이트가 적용돼요.

CNOT 게이트는 제어 큐비트가 1일 때만 대상 큐비트를 뒤집어요.
입력 상태제어 큐비트대상 큐비트 변화출력 상태
|00⟩0그대로|00⟩
|01⟩0그대로|01⟩
|10⟩10이 1로 바뀜|11⟩
|11⟩11이 0으로 바뀜|10⟩

여기서 |10⟩은 첫 번째 큐비트의 측정 결과값이 1이고 두 번째 큐비트의 측정 결과값이 0인 두 큐비트 상태를 뜻해요.
따라서 |10⟩에서 CNOT 게이트를 적용하면 제어 큐비트가 1이므로 대상 큐비트가 뒤집히고, 최종 상태는 |11⟩이 돼요.

제어 큐비트가 중첩이면 어떤 일이 생길까?

첫 번째 큐비트에 H 게이트를 적용하면 첫 번째 큐비트의 진폭이 측정 결과값 0과 1 쪽으로 나뉘어요.
그 상태에서 CNOT 게이트를 적용하면, 제어 큐비트가 0인 가지에서는 대상 큐비트가 그대로 있고 제어 큐비트가 1인 가지에서는 대상 큐비트가 뒤집혀요.
그래서 두 큐비트가 |00⟩과 |11⟩ 두 가능성으로 함께 묶인 상태가 돼요.

|00⟩ → H(q[0]) → (|00⟩ + |10⟩) / √2 → CNOT(q[0], q[1]) → (|00⟩ + |11⟩) / √2
이 상태가 얽힘을 이해하는 출발점이에요 마지막 상태를 측정하면 두 큐비트가 각각 따로 무작위로 나오는 것처럼 보이지 않아요.
측정 결과는 00 또는 11로만 나와요.
첫 번째 큐비트와 두 번째 큐비트의 측정 결과가 서로 맞물려 있기 때문이에요.

operation Main() {
    // 큐비트 두 개 준비

    use q = Qubit[2];

    // 첫 번째 큐비트에 H 게이트 적용
    // q[0]의 진폭이 측정 결과값 0과 1로 나뉨
    H(q[0]);

    // CNOT 게이트 적용
    // q[0]이 제어 큐비트, q[1]이 대상 큐비트
    CNOT(q[0], q[1]);

    // 이상적인 실행 환경: 00 또는 11
    bit first = M(q[0]);
    bit second = M(q[1]);
}

단계상태읽는 방법
준비 직후|00⟩두 큐비트가 모두 측정 결과값 0 쪽에 있어요
H(q[0]) 뒤(|00⟩ + |10⟩) / √2첫 번째 큐비트의 진폭이 0과 1로 나뉘어요
CNOT 뒤(|00⟩ + |11⟩) / √2제어 큐비트가 1인 가지에서 대상 큐비트가 뒤집혀요

이 회로를 읽을 때 핵심은 “첫 번째 큐비트를 먼저 측정해서 0 또는 1을 얻은 뒤 두 번째 큐비트를 바꿔요”가 아니에요.
H 게이트가 만든 두 진폭 가지가 CNOT 게이트를 지나면서 |00⟩ 가지와 |11⟩ 가지로 정리된다고 읽어야 해요.
그리고 마지막에 측정하면 두 큐비트의 결과가 00 또는 11로 함께 나타나요.
