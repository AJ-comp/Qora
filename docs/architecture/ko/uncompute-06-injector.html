6. 주입기 — 문장 붙여넣기 기계

자동 청소가 실제로 하는 일

← 5. Y·CY가 차단되는 이유 — 위상 도장 코스 목차 7. 루프와 호출 — 왜 판정이 갈리는가 →

판정이 safe라고 하면 컴파일러(rung ④)는 다음 절차를 실행합니다:

앤실라를 쓴 문장들의 목록을 가져와, 각 문장의 역을, 역순으로, 사망점에 붙여넣습니다.

붙여넣기의 재료는 문장 단위 역 카탈로그입니다: X→X, CNOT→CNOT, Y→Y†, 호출 Foo(...) → Adjoint Foo(...).

예제 — 3중 컨트롤을 앤실라로 조립하고 자동 청소

// a AND b AND c 일 때만 d를 뒤집고 싶은데 3중 컨트롤 게이트가 없음 → 중간값 t 사용
use t = Qubit[1];
CCX(a, b, t);    // ① compute:  t = a AND b
CCX(t, c, d);    // ② action:   d ^= t AND c   ← t의 마지막 사용 = 사망점
CCX(a, b, t);    // ③ uncompute: 주입된 역 (CCX는 자기 역) — t → |0⟩

t를 쓴 문장은 ① 하나이므로, 그 역(자기 자신) 하나가 사망점 ② 직후에 붙습니다. ①·②·③ 어느 것도 계수 칸을 건드리지 않으므로(전부 값 편집) 생존자 a·b·c·d는 완전히 보존됩니다. 사용자는 ③을 쓰지 않습니다 — 언어 규칙에 따라 컴파일러가 넣습니다.

주입 지점은 스코프 끝이 아니라 사망점입니다

재료 조건(데이터시트 7장 rung ③의 4번 조항)은 "앤실라를 만든 재료가 사망점까지 그 값 그대로"를 요구합니다. 주입이 사망점에서 일어나기 때문입니다. 앤실라가 컨테이너(if/for) 안에서 마지막으로 쓰였다면 주입 지점은 최외곽 컨테이너 직후로 늦춰지며, 재료 검사 구간도 같이 연장됩니다 — 검사와 주입이 항상 같은 지점을 바라봅니다.
