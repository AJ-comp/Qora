8. 담요 문제 — 질의 폭과 문장 폭

레지스터 통짜 쓰기와 원소 질문의 충돌

← 7. 루프와 호출 — 왜 판정이 갈리는가 코스 목차 9. 차단의 두 종류 — 원리적 한계와 보수적 거부 →

한 문장이 레지스터 전체를 한 덩어리로 쓰는 경우 — 브로드캐스트 X(a), 레지스터를 통째로 넘긴 호출, 인덱스가 컴파일 타임에 정해지지 않는 루프 — 를 담요 쓰기라고 부릅니다. 문제의 본질은 폭의 불일치 하나입니다:

역연산의 폭은 문장 폭인데 질문의 폭이 원소라면 — 문장을 원소로 썰 수 없습니다.

담요 + 전원 스크래치 — ✅ 문제없음

operation Bcast(Qubit[2] p){
  for i in 0..1 { X(p[i]); }
}
use a = Qubit[2];
Bcast(a);       // 아무도 측정 안 함
// a: cleanup candidate — safe

질의 폭(레지스터 전체) = 역연산 폭(문장 전체)이므로 썰 필요가 없습니다. Adjoint Bcast(a) 한 번이면 끝입니다.

담요 + 운명 혼합 — ❌ 원소가 막힘

use a = Qubit[2];
Bcast(a);
bit c = M(a[0]);  // a[0]만 측정
// a[1] blocked: statement-wide
//   write of `a` — ...

치울 것은 a[1]뿐인데 Adjoint Bcast(a)는 측정된 a[0]까지 되써 버립니다. 원소 질의에 담요 쓰기는 차단됩니다.

리포트는 원소 단위 진실을 함께 보여줍니다 — 막힌 원소에는 이유가, 깨끗한 원소에는 후보 표시가 붙습니다:

a: ancilla, measured — promoted to output, not a cleanup candidate
   — per element: a[1] blocked: statement-wide write of `a` — its adjoint
     cannot be sliced to `a[1]` and would rewrite the sibling elements ...

혼합 운명 에러를 받았을 때의 수정 경로

수정코드의미
① 구조 변경 (권장)Flip(a[0]); Flip(a[1]); 원소별 호출, 또는 운명별로 레지스터 분리 선언역사가 정밀해져 컴파일러가 자동으로 치웁니다
② 측정bit d = M(a[1]);출력으로 승격 — 반납 의무가 사라집니다
③ 명시적 ResetReset(a[1]);비가역 지우기를 코드로 선언합니다 (얽힌 큐빗에는 생존자 오염 주의)
④ 직접 역연산X(a[1]);물리적으로 완전한 수정 — 단 컴파일러가 이를 "반납 완료"로 인정하는 방식은 향후 명시 선언문으로 제공될 예정입니다
