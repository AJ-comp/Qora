5. 곱에서 얽힘으로 — 프로그램 단계별 확인

H로 만든 곱 상태에 CNOT 한 줄을 더하면 얽힘이 됩니다

← 4. 상태벡터를 큐빗별로 분해할 수 있는가 — 곱 상태와 얽힌 상태 코스 목차 6. 언제 위상이 측정에 드러나는가 — 전역 위상과 상대 위상 →

앞 장에서 곱 상태와 얽힌 상태가 무엇인지, 그리고 상태벡터가 큐빗별로 분해되느냐로 둘을 가른다는 것을 보았습니다.
그런데 그 정의가 실제 프로그램에서 어떻게 나타나는지는 아직 직접 보지 않았습니다.
이 장에서는 짧은 프로그램 하나를 단계마다 따라가며, H 게이트만 쓰면 곱 상태가 되고 CNOT 한 줄을 더하면 얽힌 상태가 되는 과정을, 앞 장에서 세운 분해 검사로 직접 확인합니다.

1단계 — 기준선: H만 쓰면 상태벡터가 곱으로 분해됩니다

먼저 얽힘이 없는 경우를 기준선으로 세워 두겠습니다.
아래 프로그램은 a에만 H 게이트를 적용할 뿐, 두 큐빗을 잇는 CNOT 게이트는 아직 쓰지 않습니다.
그래서 a는 |0⟩과 |1⟩의 중첩이 되고, b는 처음의 |0⟩ 그대로 남습니다. 

use a = Qubit[1];
use b = Qubit[1];
H(a[0]);       // a를 |0⟩·|1⟩ 중첩으로 만들고, b는 |0⟩ 그대로

이 프로그램을 한 줄씩 실행하며 상태벡터가 어떻게 변하는지 따라가 보겠습니다.

실행 코드b값a값진폭
use a = Qubit[1];
use b = Qubit[1];		001
H(a[0]);001/√2
011/√2
각 줄은 그 코드까지 실행한 직후의 상태벡터입니다.
H가 a를 |0⟩·|1⟩의 중첩으로 만들어 상태벡터가 두 항이 되지만, b는 두 항 모두에서 0 그대로입니다.

이제 H를 실행한 뒤의 상태벡터 (|00⟩+|01⟩)/√2가 곱인지 얽힘인지, 앞 장의 대각선 곱 비교로 판별해 보겠습니다.
네 기저 상태의 진폭을 2×2 자리에 놓고 대각선 곱을 비교합니다.

a=0a=1
b=01/√21/√2
b=100
↘ αδ = 1/√2 × 0 = 0   ↗ βγ = 1/√2 × 0 = 0

두 대각선 곱이 같으므로 (0 = 0) 이 상태는 곱 상태입니다.
실제로 b=|0⟩과 a=(|0⟩+|1⟩)/√2로 각각 따로 적을 수 있습니다.

2단계 — CNOT 한 줄을 더하면 상태벡터가 분해되지 않습니다

이제 1단계의 프로그램에 딱 한 줄, b가 a의 값을 기록하는 CNOT 게이트를 끼워 넣습니다. 

use a = Qubit[1];
use b = Qubit[1];
H(a[0]);
CNOT(a[0], b[0]);   // ← 추가된 한 줄: b가 a의 값을 복사
실행 코드b값a값진폭
use a = Qubit[1];
use b = Qubit[1];		001
H(a[0]);001/√2
011/√2
CNOT(a[0], b[0]);
← 추가된 한 줄		001/√2
111/√2
CNOT이 a의 값을 b에 복사해, a=0인 항은 b=0, a=1인 항은 b=1이 됩니다 — 그래서 두 항에서 b와 a의 값이 항상 같아집니다.

이제 CNOT을 더한 상태벡터 (|00⟩+|11⟩)/√2에 같은 대각선 곱 판별을 적용합니다.

a=0a=1
b=01/√20
b=101/√2
↘ αδ = 1/√2 × 1/√2 = 1/2   ↗ βγ = 0 × 0 = 0

두 대각선 곱이 다르므로 (1/2 ≠ 0) 이 상태는 얽힌 상태입니다.
b와 a의 상태를 각각 따로 적을 방법이 없습니다 — 한쪽 값을 알면 다른 쪽이 정해집니다.

1장에서 방치된 앤실라가 측정 결과를 바꾼 것이 정확히 이 상황입니다.
CNOT 직후 q와 t의 상태벡터는 방금의 두 항과 같은 모양이 되는데, 두 항이 t의 값에서 이미 달라져 있으므로 이후 H 게이트가 q를 아무리 섞어도 그 두 항은 같은 항으로 합쳐져 상쇄될 수 없습니다.

두 경우를 나란히 놓으면 다음과 같이 요약됩니다.

곱 상태 — 분해됩니다 (1단계)

H(a[0])만 실행한 상태입니다.
대각선 곱이 서로 같아(αδ = βγ) 곱 상태입니다.

얽힌 상태 — 분해되지 않습니다 (2단계)

CNOT까지 실행한 상태입니다.
대각선 곱이 서로 달라(αδ ≠ βγ) 얽힌 상태입니다.
