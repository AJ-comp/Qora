3. MeasureConditionLowering — 조건 속 측정을 측정 문장과 비트 조건으로 분리

병합된 QProgram의 측정 조건을 후속 패스가 처리할 수 있는 IR로 정규화

← 2. ModuleLoader 패스 목차

2장에서 ModuleLoader는 import 대상의 연산과 open을 하나의 QProgram으로 병합했습니다.
import 오류가 없다면 MeasureConditionLowering은 이 병합 결과를 입력받아, 조건 안의 M(q[i])를 독립된 측정 문장과 그 결과를 읽는 비트 조건으로 분리합니다.

PASS SPECIFICATIONMeasureConditionLowering
입력

입력: ModuleLoader가 병합한 QProgram

이 패스는 QProgram.Operations의 모든 연산을 기존 순서대로 방문합니다.
아래 도식에서는 2장의 병합 결과 가운데 Operations[0]인 Main을 열어, 측정 호출이 남은 QIf.Cond를 확인합니다.

1

QCond.HasCall을 확인한 뒤 M(register[index])를 찾습니다

먼저 HasCall이 false이면 측정 패턴을 검색하지 않고 조건 내용을 유지합니다.
true이면 QCond.Text에서 대문자 M 호출을 찾아 레지스터 이름과 인덱스 문자열을 분리합니다.

2

if 앞에 QDecl을 넣고 조건의 측정 호출을 비트 이름으로 바꿉니다

도식 읽기

왼쪽 소스 코드는 패스의 입력이 아니라, 강조한 코드 범위에 대응하는 입력 IR이 가운데에서 어떤 출력 IR로 바뀌는지를 찾기 위한 기준입니다.

새 선언의 Value는 QMeasure(QQubitArg("q", "0"))이며, 조건에서 M(q[0])가 있던 자리에는 임시 비트 이름 __m0이 들어갑니다.
측정 호출이 모두 제거된 조건은 HasCall=false가 되며, 기존 Then과 Else의 문장 순서는 유지됩니다.

3

연산 안에서 이미 차지한 이름을 건너뛰어 임시 비트 이름을 정합니다

임시 번호는 연산마다 0부터 시작합니다.
파라미터와 QUse·QDecl·QFor.Var에서 수집한 이름을 후보 집합과 대조하며, 이미 있는 이름은 건너뜁니다.

4

while은 첫 검사 전에 측정하고 본문 끝에서 다시 측정합니다

while의 조건은 반복을 시작하기 전과 매 반복이 끝난 뒤 다시 평가됩니다.
따라서 첫 측정은 루프 앞의 QDecl로 만들고, 같은 측정은 루프 본문 마지막의 QAssign으로 다시 배치합니다.

5

repeat-until은 본문을 실행한 뒤 측정하여 Until에서 읽습니다

repeat는 본문을 먼저 실행한 다음 Until 조건을 검사합니다.
따라서 측정 선언을 반복문 밖으로 꺼내지 않고, 기존 본문의 마지막 문장으로 추가합니다.

6

조건이 들어 있는 본문만 재귀적으로 방문하고 다른 호출 위치는 바꾸지 않습니다

본문 안의 제어문은 같은 규칙으로 다시 방문합니다.
다만 QFor에서는 Body만 방문하며, 범위식이나 게이트 인자처럼 조건이 아닌 위치의 호출은 이 패스의 대상이 아닙니다.

원본 코드의 호출 위치이 패스의 처리남는 IR
if (M(q[0]) == 1)측정을 QDecl로 분리QIf.Cond = "__m0 == 1"
while (M(q[0]) == 1)첫 측정과 반복 끝 재측정으로 분리QDecl + QWhile + QAssign
repeat ... until (M(q[0]) == 1)본문 끝에 측정 선언 추가QRepeat.Body + QDecl
if (Foo(q[0]) == 1)측정 호출이 아니므로 변경하지 않음QCond.HasCall = true
Rx(M(q[0]), q[1])조건이 아니므로 변경하지 않음호출 표식이 있는 기존 인자
var x = M(q[0]) + 1조건이 아니므로 변경하지 않음QText.HasCall = true
for i in 0..M(q[0])범위식은 방문하지 않음호출 문자열이 남은 QFor.To

측정을 분리한 뒤에도 다른 호출이 조건에 남아 있으면 새 QCond의 HasCall은 계속 true입니다.
이 패스는 그 호출을 오류로 확정하지 않고 그대로 다음 패스에 전달합니다.

출력

변환한 Operations를 같은 QProgram 구조에 넣어 반환합니다

각 연산의 Body 처리가 끝나면 새 연산 목록을 QProgram.Operations에 넣습니다.
Imports와 Opens는 변경하지 않으며, 이 패스는 별도의 오류 목록 없이 QProgram 하나만 반환합니다.

바뀌는 값Operations[*].Body의 측정 조건
유지되는 값Imports, Opens, 연산 순서
반환값변환된 QProgram 하나

이후 QasmEmitter는 이 QDecl을 OpenQASM의 비트 선언과 측정 대입으로 출력합니다.
비트 선언을 연산 위쪽으로 올리거나 조건의 1을 true로 출력하는 작업은 방출 단계의 책임이며, MeasureConditionLowering의 IR 출력에는 포함되지 않습니다.

패스 계약

항목계약
입력import 오류 없이 병합된 QProgram
탐색 대상모든 QOperation.Body와 그 안의 제어문 본문
변환 대상QIf.Cond, QWhile.Cond, QRepeat.Until 안의 M(register[index])
출력측정이 QMeasure 문장으로 분리된 QProgram
오류이 패스에서는 생성하지 않으며, 남은 호출과 측정 대상의 유효성은 후속 검증에서 판정
다음 패스Resolver; 이후 QoraValidator가 구조화된 측정과 조건을 검증

MeasureConditionLowering은 조건 안에 섞여 있던 측정 호출을 실행 위치가 분명한 QMeasure와 일반 비트 조건으로 분리합니다.
