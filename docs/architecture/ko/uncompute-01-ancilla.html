1. 앤실라와 청소 후보 — 용어와 판별

use로 만든 지역 작업공간, 측정과 승격, 판별 사다리

코스 목차 2. 상태를 읽는 법 — 장부 모델 →

앤실라(ancilla)는 계산 도중에만 잠깐 쓰이는 보조 큐빗입니다. 클래식 프로그램에서 최종 결과를 만들기 위해 잠시 거쳐 가는 임시 지역변수와 똑같은 역할을 합니다. Qora에서는 use 문으로 만든 큐빗 레지스터가 앤실라입니다. use로 만든 레지스터는 그 연산만이 소유하고, 항상 |0⟩ 상태로 시작합니다. 즉 "0으로 초기화된 지역변수"와 정확히 같은 출발점을 가집니다. Q#의 use, Quipper의 with_ancilla, Qiskit의 AncillaRegister가 모두 이 개념을 가리키는 이름들입니다. 

// 클래식:  int t = 0;          (0으로 초기화된 지역 임시 변수)
// 양자:
use t = Qubit[1];        // 이 연산이 소유하고 |0⟩로 시작하는 보조 큐빗 = 앤실라

클래식 지역변수와 다른 점은 단 하나입니다. 클래식 변수는 스코프가 끝나면 그냥 버리면 그만이지만, 양자 앤실라는 계산 도중 다른 큐빗과 얽히기 때문에 그냥 버릴 수 없습니다. 얽힌 채로 방치된 앤실라는 살아남은 큐빗들의 간섭을 망가뜨려서, 프로그램의 최종 결과 자체를 바꿔 버립니다.

예제 — 방치된 앤실라가 결과를 실제로 바꾸는 최소 사례

H를 두 번 연달아 적용하면 원래 상태로 돌아옵니다. 그래서 아래 왼쪽 프로그램의 측정 결과는 언제나 0입니다. 오른쪽은 두 H 사이에서 앤실라 t가 q의 값을 기록하게 한 것뿐이고, t는 그 뒤로 아무 데도 쓰이지 않습니다.

앤실라 없음 — 결과는 항상 0

use q = Qubit[1];
H(q[0]);
H(q[0]);
bit m = M(q[0]);
// m = 0 (확률 100%)

얽힌 앤실라를 방치 — 결과가 동전 던지기로

use q = Qubit[1];  use t = Qubit[1];
H(q[0]);
CNOT(q[0], t[0]);  // t가 q를 기록 (얽힘)
H(q[0]);           // t는 방치
bit m = M(q[0]);
// m = 0 또는 1 (각 50%)

t는 읽히지도 측정되지도 않았는데, 존재만으로 q의 결과를 결정론에서 무작위로 바꿔 놓았습니다. 이유는 간섭의 조건에 있습니다. 두 번째 H가 결과를 0으로 되돌리려면 q의 두 갈래(0이었던 세계와 1이었던 세계)가 서로 겹쳐 상쇄되어야 하는데, t가 그 두 갈래를 각각 기록해 버렸기 때문에 두 세계는 더 이상 같은 세계가 아니고, 따라서 겹치지 못합니다. 갈래와 간섭을 정확히 읽는 방법은 2장에서 다룹니다.

해결책은 버리기 전에 기록을 지우는 것입니다. 두 번째 H 이전에 CNOT(q[0], t[0])를 한 번 더 적용하면 t는 |0⟩로 돌아가 얽힘이 풀리고, 측정 결과는 다시 100% 확률로 0이 됩니다. 이렇게 앤실라를 |0⟩로 되돌리는 일을 uncomputation이라 부르며, Qora는 이 정리 코드를 컴파일러가 자동으로 삽입합니다. 그래서 언어 규칙도 하나입니다: 측정하지 않은 use 레지스터는 스코프가 끝날 때 |0⟩로 반납되어야 합니다.

판별 사다리 — 컴파일러가 던지는 질문 세 개

컴파일러는 큐빗마다 세 가지 질문을 차례로 던져서 자동 정리 대상을 가려냅니다.

단계질문판별 기준API
0. 앤실라인가?누구의 큐빗인가use로 만들었으면 앤실라입니다. 파라미터는 호출자의 소유이므로 제외됩니다.IsAncilla
1. 청소 대상인가?값이 배달되었는가측정되지 않은 앤실라만 청소 후보입니다. 측정된 앤실라는 출력으로 승격되어 제외됩니다.IsCleanupCandidate
2. 청소 가능한가?역사가 깨끗한가계산 이력이 4개 조항을 통과해야 합니다 (4장–8장).UncomputeSafety
측정은 앤실라 여부를 바꾸지 않습니다 — 역할을 바꿉니다

측정된 use 레지스터도 앤실라로 태어났다는 사실은 변하지 않습니다. 다만 측정으로 값이 bit 변수를 통해 프로그램에 배달되었으므로, 스크래치가 아니라 출력으로 승격되어 청소 대상에서 빠질 뿐입니다. 그래서 용어를 세 가지로 구분해 부릅니다. 태생을 가리키는 말이 앤실라, 측정되지 않아 정리해야 하는 앤실라가 청소 후보, 판정까지 통과해 자동 정리가 확정된 후보가 safe입니다.

예제 — 세 분류가 리포트에 그대로 나타납니다

operation Flip(Qubit p){ X(p); }
operation Main(){
    use a   = Qubit[1];          // 앤실라
    use out = Qubit[1];          // 앤실라
    Flip(a[0]);                  // a에 중간값 생성
    CNOT(a[0], out[0]);          // 중간값을 out에 반영 — a의 마지막 사용
    bit c = M(out[0]);           // out을 측정 → 출력으로 승격
}
// --stages 의 uncompute 뷰:
p:   input (parameter) — caller-owned, not an ancilla        ← 0단 탈락 (호출자 소유)
a:   ancilla, cleanup candidate — safe to uncompute           ← 0·1·2단 전부 통과
out: ancilla, measured — promoted to output, not a cleanup candidate  ← 1단 탈락 (승격)

a의 값은 out으로 배달이 끝났고, 그 뒤로는 아무도 a 자체를 읽지 않습니다. 클래식 코드의 t = ...; result = f(t);에서 t가 임시 변수인 것과 같은 상황입니다. 이런 큐빗이 청소 후보가 되고, 컴파일러는 마지막 사용 직후(사망점)에 정리 코드를 자동으로 넣습니다. 그 정리 코드가 정확히 무엇이고 어떻게 만들어지는지는 6장에서 다룹니다.
