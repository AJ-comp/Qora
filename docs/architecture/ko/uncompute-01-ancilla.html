1. 앤실라와 청소 후보 — 용어와 판별

use로 만든 지역 작업공간, 측정과 승격, 판별 사다리

코스 목차 2. 상태를 읽는 법 — 장부 모델 →

앤실라(ancilla)는 계산 중간에 잠깐 쓰는 보조 큐빗, 즉 연습장입니다. 정의는 탄생 기준 하나입니다: use로 만든 큐빗 레지스터 — 이 연산이 소유하고, |0⟩로 시작하는 지역 작업공간. 클래식의 "0으로 초기화된 지역변수"에 정확히 대응하며, Q#의 use, Quipper의 with_ancilla, Qiskit의 AncillaRegister와 같은 개념입니다. 

// 클래식의 지역변수:  int t = 0;   →  양자의 앤실라:
use t = Qubit[1];        // 이 연산 소유, |0⟩로 시작 — 앤실라

클래식과 다른 점은 하나뿐입니다. 클래식 지역변수는 스코프가 끝나면 그냥 버리면 되지만, 양자 앤실라는 계산 중 다른 큐빗과 얽히기 때문에 그냥 버릴 수 없습니다. 얽힌 채 방치된 앤실라는 살아남은 큐빗들의 간섭을 오염시켜 최종 결과를 틀리게 만듭니다. 그래서 언어 규칙이 필요합니다: 측정하지 않은 use 레지스터는 스코프 끝에 |0⟩로 반납되어야 합니다.

측정은 앤실라 여부를 바꾸지 않습니다 — 역할을 바꿉니다

측정된 use 레지스터도 앤실라로 태어난 것은 변하지 않습니다. 다만 측정으로 값이 프로그램에 배달되었으므로(bit 변수로) 출력으로 승격되어 청소 풀에서 빠질 뿐입니다. 한 개념 = 한 용어: 앤실라(탄생) / 청소 후보(측정되지 않은 앤실라) / safe(판정을 통과한 후보).

예제 — 세 분류가 리포트에 그대로 나타납니다

operation Flip(Qubit p){ X(p); }
operation Main(){
    use a   = Qubit[1];          // 앤실라
    use out = Qubit[1];          // 앤실라
    Flip(a[0]);                  // a = 중간값 생성
    CNOT(a[0], out[0]);          // 중간값을 out에 반영 — a의 마지막 사용
    bit c = M(out[0]);           // out 측정 → 출력으로 승격
}
// --stages 의 uncompute 뷰:
p:   input (parameter) — caller-owned, not an ancilla        ← 0단 탈락 (호출자 소유)
a:   ancilla, cleanup candidate — safe to uncompute           ← 0·1·2단 전부 통과
out: ancilla, measured — promoted to output, not a cleanup candidate  ← 1단 탈락 (승격)

a의 값은 out으로 배달이 끝났고 그 뒤로 아무도 a 자체를 읽지 않습니다 — 클래식의 t = ...; result = f(t);에서 t가 임시 변수인 것과 같은 상황입니다. 이런 큐빗이 청소 후보이며, 컴파일러가 사망점(마지막 사용 직후)에 정리 코드를 자동으로 넣는 것이 자동 uncomputation입니다.
