2. 상태를 읽는 법 — 장부 모델

가지와 계수 — 이후 모든 판정을 설명하는 단 하나의 틀

← 1. 앤실라와 청소 후보 — 용어와 판별 코스 목차 3. 게이트는 장부를 어떻게 편집하는가 →

이후 장들의 모든 판정, 예컨대 "이 게이트는 왜 위험한가"라는 질문은 상태를 읽는 틀 하나로 전부 설명됩니다. 그 틀은 기계 전체의 상태를 장부 한 권으로 읽는 것입니다. 장부의 한 줄을 가지라고 부르며, 한 줄에는 모든 큐빗의 값 조합 하나와 그 조합의 계수 하나가 적힙니다. 예를 들어 아래 프로그램은 상태 (|0,0⟩ + |1,1⟩)/√2(표기 |b,a⟩)를 만드는데, 이 상태를 장부로 쓰면 그 아래의 두 줄짜리 표가 됩니다. 

use a = Qubit[1];  use b = Qubit[1];
H(a[0]);             // a가 0과 1 두 갈래로 갈라짐
CNOT(a[0], b[0]);    // b가 a의 값을 기록
줄(가지)b 값a 값계수
1행001/√2
2행111/√2

이 표를 읽는 규칙은 두 가지입니다. 첫째, 한 줄에는 모든 큐빗의 값이 한 세트로 적혀 있습니다. 그러므로 가지는 특정 큐빗이나 레지스터가 아니라 기계 전체의 값 조합 하나를 가리킵니다. 둘째, 계수 칸은 줄마다 하나뿐입니다. "b의 계수 칸"과 "a의 계수 칸"이 따로 있는 것이 아니라, 그 값 조합 전체에 계수 하나가 붙습니다.

얽힘 — 장부를 곱으로 쪼갤 수 없다는 사실

이 장부 모델은 1장에서 이름만 쓰고 지나간 얽힘이 정확히 무엇인지 답해 줍니다. 기준은 다음과 같습니다. 큐빗마다 작은 장부를 하나씩 골라서, 큰 장부의 각 줄 (b값, a값)의 계수가 항상 "b장부에서 b값의 계수 × a장부에서 a값의 계수"와 일치하게 만들 수 있으면 그 상태는 곱 상태입니다. 그런 작은 장부 쌍이 아예 존재하지 않으면 얽힌 상태입니다. 즉 얽힘은 신비한 힘이 아니라, 장부를 곱으로 쪼갤 수 없다는 사실 그 자체입니다.

곱 상태 — 쪼개집니다

use a = Qubit[1];  use b = Qubit[1];
H(a[0]);   // b는 손대지 않음
// 장부 (b,a): (0,0)→1/√2, (0,1)→1/√2

b만의 장부를 "0의 계수 1" 한 줄로, a만의 장부를 "0의 계수 1/√2, 1의 계수 1/√2" 두 줄로 잡으면 큰 장부가 그대로 복원됩니다. 예컨대 (0,1) 줄의 계수는 1 × 1/√2 = 1/√2로 일치합니다.

얽힌 상태 — 쪼개지지 않습니다

use a = Qubit[1];  use b = Qubit[1];
H(a[0]);
CNOT(a[0], b[0]);
// 장부 (b,a): (0,0)→1/√2, (1,1)→1/√2

페이지 맨 위 표의 상태입니다. 어떤 작은 장부 쌍을 골라도 이 두 줄을 곱으로 복원할 수 없습니다. 그 이유는 아래에서 확인합니다.

오른쪽 장부가 쪼개지지 않는다는 사실은 곱 규칙만으로 확인됩니다. (0,0) 줄의 계수가 1/√2이므로 b장부의 0 계수와 a장부의 0 계수는 둘 다 0이 아니어야 하고, (1,1) 줄 때문에 b장부의 1 계수와 a장부의 1 계수도 0이 아니어야 합니다. 그런데 네 값이 모두 0이 아니라면 곱 규칙에 따라 (0,1) 줄의 계수도 0이 아니어야 하는데, 실제 장부에 (0,1) 줄은 없습니다. 어떤 작은 장부 쌍을 골라도 같은 모순이 나오므로 이 상태는 곱으로 쪼개지지 않고, 따라서 얽힌 상태입니다. 1장에서 방치된 앤실라가 측정 결과를 바꾼 것이 정확히 이 상황입니다. CNOT 직후 q와 t의 장부는 오른쪽과 같은 두 줄이 되는데, 두 줄이 t의 값에서 이미 달라져 있으므로 이후 H가 q를 아무리 섞어도 그 두 줄은 같은 줄로 합쳐져 상쇄될 수 없습니다.

위상(계수)은 줄의 속성이지, 한 큐빗의 소유물이 아닙니다

수식으로 보면 i·(|0⟩b ⊗ |1⟩a) = (i|0⟩b) ⊗ |1⟩a = |0⟩b ⊗ (i|1⟩a)입니다. 곱셈이므로 i를 어느 쪽에 붙여 읽어도 같은 상태입니다. 그러므로 "이 i가 b의 것이냐 a의 것이냐"는 답이 없는 질문이고, 물리적 사실은 "이 조합(줄)의 계수가 i다" 하나뿐입니다. 이 사실이 5장의 Y 문제를 만듭니다.

전역 위상 vs 상대 위상 — 위험의 기준

계수에 붙는 부호가 모든 줄에 똑같이 붙으면 전역 위상이라 부릅니다. 전역 위상은 물리적으로 아무 의미가 없어서 측정으로 절대 구분할 수 없습니다. 반면 부호가 줄마다 다르게 붙으면 상대 위상이라 부르며, 나중에 H 같은 간섭 게이트가 그 부호 차이를 측정 확률의 차이로 바꿔 읽습니다. 위험한 것은 위상이 있다는 사실 자체가 아니라 줄마다 다르게 붙었다는 사실입니다.

예제 — 같은 Z 한 번이 전역 위상이면 무해하고, 상대 위상이면 결과를 뒤집습니다

전역 위상 — Z가 있으나 없으나 같음

use q = Qubit[1];
X(q[0]);       // 장부: (1)→1, 줄 하나
Z(q[0]);       // 유일한 줄에 부호 − → 전역 위상
bit m = M(q[0]);
// m = 1 (확률 100%) — Z를 지워도 동일

상대 위상 — H가 확률 차이로 읽어냄

use q = Qubit[1];
H(q[0]);       // 장부: (0)→1/√2, (1)→1/√2
Z(q[0]);       // 1인 줄에만 부호 − → 상대 위상
H(q[0]);
bit m = M(q[0]);
// m = 1 (확률 100%) — Z가 없으면 m = 0

왼쪽에서는 부호가 장부의 모든 줄에 붙었습니다. 줄이 하나뿐이므로 그 한 줄이 곧 모든 줄이고, 측정 확률은 계수의 크기만 보기 때문에 결과는 Z가 없을 때와 완전히 같습니다. 오른쪽에서는 같은 Z가 두 줄 중 1인 줄에만 부호를 붙였고, 뒤따르는 H가 그 부호 차이를 간섭으로 바꿔 읽어 측정 결과가 0에서 1로 확실하게 뒤집힙니다.
