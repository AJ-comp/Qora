4. 청소의 계약과 qfree

되돌릴 수 있는 쓰기의 조건

자동 청소가 지켜야 하는 계약은 한 문장입니다.

앤실라만 |0⟩로 빼내고, 생존자들의 장부는 부호 하나까지 그대로 둡니다.

이 계약을 지킬 수 있는 쓰기가 어떤 것인지는 3장의 편집 분류로 결정됩니다. 핵심 기준은 "그 쓰기가 계수 칸을 건드렸는가"입니다.

쓰기의 종류게이트판정이유
값 칸만 고침X, CNOT, CCX, SWAP✅ 청소 가능 (qfree)역연산도 값 칸만 도로 고칩니다 — 계수 무접촉이므로 생존자 보존
값을 안 씀 (계수만)Z, S, T, CZ, Rz✅ 애초에 쓰기가 아님읽기로 기록 — 청소할 것 자체가 없습니다. 위상 킥백 패턴(Grover 오라클)이 성립하는 이유
값 + 값의존 위상Y, CY❌ 차단 (NonQfreeWrite)얽힌 타깃이면 도장이 상대 위상이 됩니다 — 5장
줄 쪼개기 (중첩 생성)H, Rx, Ry❌ 차단 (NonQfreeWrite)생존자가 값을 기록한 뒤에는 쪼개진 줄을 도로 접을 수 없습니다

이 기준을 학계 용어로 qfree라고 부릅니다: 기저 값을 기저 값으로 보내고 계수를 건드리지 않는 연산.

예제 — qfree 체인은 왜 안전한가

use a = Qubit[1];  use out = Qubit[1];
X(a[0]);              // a: |0⟩ → |1⟩ (값 칸만)
CNOT(a[0], out[0]);   // out: 값 배달 (값 칸만) — a의 사망점
X(a[0]);              // ← 자동 주입될 청소: a → |0⟩
bit c = M(out[0]);    // 결과 1, 결정론 — 주입 전과 완전 동일

처음부터 끝까지 어떤 편집도 계수 칸을 건드리지 않았습니다. 그래서 청소를 넣어도 생존자 out의 장부는 부호 하나까지 원본 그대로입니다. 이것이 qfree 체인의 안전성이며, 교과서의 앤실라 사용 패턴(가역 논리 체인, Grover 오라클, 산술 회로)이 전부 이 부류에 속합니다.
