4. 청소의 계약과 qfree

되돌릴 수 있는 쓰기의 조건

자동 청소가 지켜야 하는 계약은 한 문장입니다.

앤실라만 |0⟩로 빼내고, 생존자들의 장부는 부호 하나까지 그대로 둡니다.

이 계약을 지킬 수 있는 쓰기가 어떤 것인지는 3장의 편집 분류로 결정됩니다. 핵심 기준은 "그 쓰기가 계수 칸을 건드렸는가"입니다.

쓰기의 종류게이트판정이유
값 칸만 고침X, CNOT, CCX, SWAP✅ 청소 가능 (qfree)역연산도 값 칸만 도로 고칩니다 — 계수 무접촉이므로 생존자 보존
값을 안 씀 (계수만)Z, S, T, CZ, Rz✅ 애초에 쓰기가 아님읽기로 기록 — 청소할 것 자체가 없습니다. 위상 킥백 패턴(Grover 오라클)이 성립하는 이유
값 + 값의존 위상Y, CY❌ 차단 (NonQfreeWrite)얽힌 타깃이면 도장이 상대 위상이 됩니다 — 5장
줄 쪼개기 (중첩 생성)H, Rx, Ry❌ 차단 (NonQfreeWrite)생존자가 값을 기록한 뒤에는 쪼개진 줄을 도로 접을 수 없습니다

이 기준에는 학계에서 쓰는 이름이 있습니다. 기저 값을 기저 값으로 보내고 계수를 건드리지 않는 연산을 qfree라고 부릅니다.

예제 — qfree 체인은 왜 안전한가

use a = Qubit[1];  use out = Qubit[1];
X(a[0]);              // a: |0⟩ → |1⟩ (값 칸만)
CNOT(a[0], out[0]);   // out: 값 배달 (값 칸만) — a의 사망점
X(a[0]);              // ← 자동 주입될 청소: a → |0⟩
bit c = M(out[0]);    // 결과 1, 결정론 — 주입 전과 완전 동일

처음부터 끝까지 어떤 편집도 계수 칸을 건드리지 않았습니다. 그래서 청소를 넣어도 생존자 out의 장부는 부호 하나까지 원본 그대로입니다. 이것이 qfree 체인의 안전성이며, 교과서의 앤실라 사용 패턴(가역 논리 체인, Grover 오라클, 산술 회로)이 전부 이 부류에 속합니다.

예제 — 값을 안 쓰는 계수 편집(Z)은 왜 무해한가

Z는 값 칸을 전혀 고치지 않고 그 줄의 부호만 바꿉니다. 그래서 앤실라에 Z를 찍어도 쓰기 이력에는 아무것도 추가되지 않고, 이미 세워 둔 청소 계획이 그대로 유효합니다. 아래 프로그램에서 앤실라 a를 쓰는 편집은 CNOT 하나뿐이므로, 중간에 Z가 끼어 있어도 청소는 여전히 CNOT 한 번입니다. 

use q = Qubit[1];  use a = Qubit[1];
H(q[0]);              // q: 줄 쪼개기
CNOT(q[0], a[0]);     // a가 q의 값을 기록 (값 칸만 — 쓰기)
Z(a[0]);              // a=1인 줄의 부호만 −로 (계수만 — 읽기)
CNOT(q[0], a[0]);     // ← 자동 주입될 청소: a → |0⟩ (Z는 쓰기 이력에 없음)
H(q[0]);
bit m = M(q[0]);      // m = 1 (확률 100%)
시점장부 (|q,a⟩)일어난 일
H · CNOT 후|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2a가 q의 값을 기록 (얽힘)
Z(a) 후|0,0⟩ ×1/√2 · |1,1⟩ ×−1/√2값 칸 무접촉 — 부호만 편집
청소 CNOT 후|0,0⟩ ×1/√2 · |1,0⟩ ×−1/√2a는 모든 줄에서 0 — |0⟩ 반납 완료
H(q) 후|1,0⟩ ×1m = 1 확정

Z가 값 칸을 건드리지 않았기 때문에 Z 이전과 이후의 값 칸 배치는 완전히 같고, 따라서 쓰기 이력의 역순인 CNOT 한 번으로 a는 정확히 |0⟩로 돌아갑니다. 도장이 사라지는 것도 아닙니다. −1 부호는 a가 |0⟩로 반납된 뒤에도 q=1인 줄에 그대로 남아서, 마지막 H의 간섭 방향을 바꾸고 측정 결과를 0에서 1로 뒤집습니다. 실제로 위 프로그램에서 Z 한 줄을 지우면 두 CNOT과 두 H가 전부 상쇄되어 m은 100% 확률로 0이 됩니다. 값을 안 쓰는 계수 편집은 이렇게 청소를 방해하지 않으면서 정보(부호)만 생존자에게 옮겨 주며, Grover 오라클의 위상 킥백이 정확히 이 패턴입니다.
