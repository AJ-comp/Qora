11. 주입기 — 문장 붙여넣기 기계

자동 청소가 실제로 하는 일

← 10. Y·CY가 차단되는 이유 — 위상 코스 목차 12. 루프와 호출 — 왜 판정이 달라지는가 →

2장의 판별 사다리를 통과해 청소 후보가 된 앤실라가 되돌림 안전 판정(rung ③)까지 통과해 safe가 되면, 이제 컴파일러(rung ④)가 그 앤실라를 |0⟩로 되돌리는 정리 코드를 실제로 만들어 넣습니다.
이 단계는 회로를 새로 설계하지 않습니다.
앞에 이미 적힌 문장들을 재료로 삼는 기계적인 붙여넣기이며, 절차 전체가 다음 한 문장으로 요약됩니다.

앤실라를 편집한 문장들의 목록을 가져와, 각 문장의 역을, 역순으로, 사망점에 붙여넣습니다.

이 한 문장에 나오는 네 용어의 정확한 정의는 다음과 같습니다.

용어정의
목록효과분석이 쓰기로 기록한 문장들, 곧 그 앤실라의 기저 상태 값을 실제로 바꾼 문장들을 프로그램 순서대로 늘어놓은 것입니다.
앤실라를 컨트롤로 읽기만 한 문장은 값을 바꾸지 않았으므로 되돌릴 것이 없고, 목록에서 빠집니다.
역어떤 문장이 양자 상태에 유니터리 U를 적용했다면, 그 문장의 역은 U†U = I를 만족하는 수반 유니터리 U†를 적용하는 문장입니다.
safe 판정을 통과한 앤실라의 목록에 든 문장은 비가역 접촉이 이미 배제되어 전부 유니터리이므로, 역이 항상 존재합니다.
역순목록의 마지막 문장의 역에서 시작해 첫 문장의 역으로 끝나는, 원래 순서를 뒤집은 순서입니다.
두 문장이 차례로 유니터리 U₁, U₂를 적용했다면 전체 효과는 곱 U₂U₁이고, 그 역은 (U₂U₁)† = U₁†U₂†이므로 마지막 편집의 역이 가장 앞에 와야 합니다.
사망점그 앤실라가 프로그램에서 마지막으로 등장하는 문장 바로 다음 지점입니다.
2장에서 "마지막 사용 직후"라고 부른 바로 그 지점입니다.

역순 등식 (U₂U₁)† = U₁†U₂†를 익숙한 순서로 옮기면 양말과 신발입니다.
양말을 신고 신발을 신었다면 벗을 때는 신발이 먼저입니다.
마지막에 한 편집을 가장 먼저 되돌려야 그 아래에 있던 이전 편집이 드러나기 때문입니다.
이제 남은 물음은 각 문장에 대해 붙여넣을 "역"이 구체적으로 어떤 문장이냐는 것입니다.

그 역은 구체적으로 어떤 문장일까요 — 자기 역 게이트 X·CNOT·CCX

각 문장의 역은 즉석에서 계산하는 것이 아니라, 문장 종류별로 미리 정해진 카탈로그에서 꺼내 옵니다.
카탈로그의 항목을 하나씩 상태벡터 위에서 확인합니다.
가장 흔한 항목은 기저 값만 뒤집는 게이트들이고, 이들은 자기 자신이 역입니다.
X 게이트는 모든 항에서 타깃의 기저 값을 뒤집는 편집이므로(8장), 한 번 더 적용하면 모든 항이 제자리로 돌아옵니다.
얽힌 상태벡터에서도 마찬가지라는 것을 단계별로 확인해 보겠습니다.

시점상태벡터 (|s,t⟩)일어난 일
시작 (얽힌 상태)|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2항마다 t의 값이 다름
X(t[0]) 후|0,1⟩ ×1/√2 · |1,0⟩ ×1/√2모든 항에서 t의 기저 값만 뒤집힘
X(t[0]) 한 번 더|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2시작 상태벡터와 완전히 동일 — X 게이트는 자기 역

진폭은 두 번의 X 어디에서도 변하지 않았습니다.
CNOT 게이트도 같은 원리입니다.
컨트롤이 1인 항에서만 타깃의 기저 값을 뒤집는 편집이므로, 같은 CNOT 게이트를 한 번 더 적용하면 정확히 그 항들만 다시 뒤집혀 제자리로 돌아옵니다.

시점상태벡터 (|s,t⟩)일어난 일
시작 (얽힌 상태)|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2항마다 t의 값이 다름
CNOT(s[0], t[0]) 후|0,0⟩ ×1/√2 · |1,0⟩ ×1/√2s가 1인 둘째 항에서만 t가 뒤집혀, t는 모든 항에서 0
CNOT(s[0], t[0]) 한 번 더|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2같은 항만 다시 뒤집혀 시작 상태벡터와 동일 — CNOT 게이트는 자기 역

CCX 게이트는 조건이 "컨트롤 두 개가 모두 1인 항"로 늘어난 같은 XOR 편집이므로 역시 자기 역이고, 아래 첫째 예제에서 절차 전체와 함께 확인합니다.
세 게이트 모두 기저 값만 만졌고 진폭은 어느 시점에도 건드리지 않았습니다.
그럼 기저 값을 뒤집으면서 진폭까지 만지는 문장의 역도, 값을 한 번 더 뒤집는 것으로 충분할까요.

값과 진폭을 동시에 만지는 Y — 역은 X가 아니라 Y†입니다

기저 값을 뒤집으면서, 뒤집은 값에 따라 +i 또는 −i 위상을 그 항의 진폭에 곱하는 게이트가 Y였습니다(8장).
값만 다시 뒤집는 X 게이트를 붙여넣는 것으로는 부족하고, 위상까지 함께 되돌리는 수반(adjoint) 게이트 Y†가 필요합니다.
|1⟩ 한 항짜리 상태벡터에서 Y† 주입과 X 주입 두 미래를 비교하면 차이가 그대로 보입니다.

시점상태벡터 (|t⟩)일어난 일
시작|1⟩ ×1—
Y(t[0]) 후|0⟩ ×−i값이 1→0으로 뒤집히며 진폭에 −i 위상
Y†(t[0])를 주입한 미래|1⟩ ×1값과 위상이 함께 복구 — 시작 상태벡터와 동일
(비교) X(t[0])를 주입한 미래|1⟩ ×−i값은 돌아오지만 −i 위상이 진폭에 남음

이 상태벡터에서는 항이 하나뿐이라 남은 −i가 모든 항에 공통인 위상에 그치지만, 앤실라가 얽혀 있으면 Y 게이트의 위상은 항마다 +i와 −i로 달라지는 상대 위상이 되어, 뒤따르는 간섭에서 관측 가능한 차이를 만듭니다(10장에서 계산한 사고가 정확히 이것입니다).
그래서 카탈로그는 Y 게이트의 역으로 값만 뒤집는 게이트가 아니라 Y†를 지정합니다.
다만 앤실라 자동 청소의 목록에서 이 항목이 직접 나올 일은 없습니다.
앤실라에 대한 Y 쓰기는 되돌림 안전 판정(rung ③)이 NonQfreeWrite로 차단하기 때문입니다.
이 항목이 실제로 일하는 곳은 카탈로그의 마지막 항목인 호출의 역입니다.
Adjoint 호출이 연산 본문을 뒤집다가 본문 속 Y 게이트를 만나면, 이 항목에 따라 Y†를 내놓습니다.

예제 — 3중 컨트롤을 앤실라로 조립하고 자동 청소

카탈로그가 준비되었으니, 이제 목록 만들기·역 꺼내기·사망점에 붙여넣기의 절차 전체를 처음부터 끝까지 실행해 봅니다.
목록에 항목이 하나뿐인 가장 단순한 경우부터 시작합니다. 

// a AND b AND c 일 때만 d를 뒤집고 싶은데 3중 컨트롤 게이트가 없음 → 중간값 t 사용
use t = Qubit[1];
CCX(a, b, t);    // ① compute:  t = a AND b
CCX(t, c, d);    // ② action:   d ^= t AND c   ← t의 마지막 사용 = 사망점
CCX(a, b, t);    // ③ uncompute: 주입된 역 (CCX는 자기 역) — t → |0⟩

상태벡터의 항 종류별로 t와 d의 기저 값을 따라가면, 세 단계는 다음과 같이 움직입니다.

시점a·b가 모두 1인 항그 밖의 항
① compute 후t = 1t = 0 (그대로)
② action 후c까지 1인 항에서만 d가 뒤집힘 · t = 1d 그대로 · t = 0
③ 주입 후t = 0t = 0

t를 편집한 문장은 ① 하나뿐이므로 목록에는 항목이 하나이고, 카탈로그에 따라 그 역은 CCX 자기 자신입니다.
그래서 ①의 복사본 하나가 사망점인 ② 직후에 붙습니다.
②는 t를 컨트롤로 읽기만 했으므로 값을 바꾸지 않았고, 목록에 들어가지 않습니다.
③이 끝나면 t는 모든 항에서 0이므로, 나머지 큐빗과의 얽힘이 풀린 채 |0⟩로 반납됩니다.
①·②·③은 전부 기저 값만 편집하는 문장이어서 진폭은 어디에서도 건드려지지 않고, 생존자 a·b·c·d의 상태는 완전히 보존됩니다.
③은 사용자가 직접 쓰는 코드가 아니라, 측정하지 않은 use 레지스터는 |0⟩로 반납되어야 한다는 언어 규칙(1장)을 이행하기 위해 컴파일러가 넣는 코드입니다.

예제 — 편집이 두 번이면 역순이 보입니다: 호출은 Adjoint 호출로

앞의 예제는 목록에 항목이 하나라서 역순이 드러나지 않았습니다.
이번에는 앤실라 t를 게이트로 한 번, 호출로 한 번, 총 두 번 편집해 봅니다. 

operation Mix(s: Qubit, d: Qubit){
    X(d);                    // d를 뒤집고
    CNOT(s, d);              // s를 XOR
}
operation Main(){
    use q = Qubit[2];
    use t = Qubit[1];
    H(q[0]);
    X(t[0]);                 // ① t 편집: t = 1
    Mix(q[0], t[0]);         // ② t 편집(호출): t = q[0]이 됨
    CNOT(t[0], q[1]);        // ③ action: q[1] ^= t  ← t의 마지막 사용 = 사망점
    Adjoint Mix(q[0], t[0]); // 주입 1: ②의 역 — 호출은 Adjoint 호출로
    X(t[0]);                 // 주입 2: ①의 역 — t가 |0⟩로 복귀
    var m0: bit = M(q[0]);
    var m1: bit = M(q[1]);
    // 항상 m0 == m1 (00 또는 11, 각각 50%)
}

t를 편집한 문장의 목록은 [①, ②] 두 개입니다.
역순이므로 ②의 역이 먼저, ①의 역이 나중에 붙습니다.
②는 호출이므로 카탈로그에 따라 Adjoint Mix가 되는데, 이것은 Mix가 한 일을 정확히 되돌리는 연산입니다.
안을 들여다보면 본문에도 같은 규칙이 적용되어 있습니다.
CNOT(s, d)가 먼저, X(d)가 나중, 즉 Mix 본문 각 문장의 역을 역순으로 실행하는 것입니다.
각 기저 상태에서 t의 값을 따라가 보면, 주입된 두 항이 t를 |0⟩로 데려다 놓는 과정이 그대로 보입니다.

시점q[0]이 0인 기저 상태q[0]이 1인 기저 상태
① X 직후t = 1t = 1
② Mix 직후t = 0t = 1 (t = q[0])
③ action 직후q[1] = 0, t = 0q[1] = 1, t = 1
주입 1 직후t = 1t = 1
주입 2 직후t = 0t = 0

주입 1이 끝난 시점에는 두 기저 상태에서 t가 모두 1이 되고, 주입 2가 그 값을 0으로 내립니다.
t가 두 기저 상태에서 같은 값을 갖게 된 순간 t와 q 사이의 얽힘은 풀린 것이고, q에는 (|00⟩+|11⟩)/√2 상태만 깨끗하게 남습니다.
그래서 측정 결과는 언제나 m0 == m1이 됩니다.

카탈로그 정리 — 확인한 항목을 한 표로 다시 봅니다

두 예제까지 지나며 카탈로그의 항목들이 전부 상태벡터 단계표 또는 실행 예제로 확인되었습니다.
한 표로 모으면 다음과 같습니다.

원래 문장주입되는 역왜 이것이 역인가확인한 곳
X(t[0])X(t[0])두 번 뒤집으면 원래대로 (자기 역)X 단계표
CNOT(c[0], t[0])CNOT(c[0], t[0])같은 XOR을 한 번 더 (자기 역)CNOT 단계표
CCX(a, b, t)CCX(a, b, t)자기 역첫째 예제
Y(t[0])Y†(t[0])위상까지 함께 되돌리는 수반(adjoint) 게이트Y 단계표
Foo(...) 호출Adjoint Foo(...) 호출Foo 본문 전체의 역 (본문에 같은 규칙을 재귀 적용)둘째 예제 (Mix)
주입 지점은 스코프 끝이 아니라 사망점입니다

무엇을 붙여넣는지가 카탈로그로 정해졌으니, 마지막으로 붙여넣는 위치를 확인합니다.
재료 조건(데이터시트 7장 rung ③의 4번 조항)은 "앤실라를 만든 재료가 사망점까지 그 값 그대로"를 요구합니다.
주입이 바로 그 사망점에서 일어나기 때문입니다.
앤실라가 컨테이너(if/for) 안에서 마지막으로 쓰였다면 주입 지점은 최외곽 컨테이너 직후로 늦춰지고, 재료 검사 구간도 같은 지점까지 함께 연장됩니다.
검사와 주입은 이렇게 언제나 같은 지점을 바라보도록 맞춰져 있습니다.
