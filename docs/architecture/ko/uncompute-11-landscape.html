11. 언어 지형 속의 Qora

2026년의 도구들과 Qora의 자리

← 10. 왜 정적 컴파일인가 — 두 개의 런타임 코스 목차

2026년 현재 양자 프로그래밍 도구들은 크게 세 층으로 나뉩니다.

도구방식특징
Qiskit · CirqPython이 회로를 조립최대 생태계 — 다만 언어가 아니어서 컴파일 타임 보증이 없습니다
Silq인터프리터 언어자동 uncomputation의 원조 — 다만 실기계 실행 경로가 없습니다
CUDA-Q (NVIDIA)컴파일러 인프라 (MLIR→QIR)언어가 아니라 언어들이 올라타는 기반 + GPU 시뮬레이터
Guppy (Quantinuum)정적 컴파일, 선형 타입큐빗 누수·복제를 컴파일 에러로 강제 — 청소는 사용자 몫
Classiq Qmod정적 컴파일 + 합성추론 기반 자동 uncomputation을 상용화 — 클로즈드 플랫폼
Qora정적 컴파일 (AOT → OpenQASM 3)주석 없는 추론 + 자동 청소 + 못 치우면 명시적 에러 — 공개 구현, 교육 지향

정적 컴파일 언어들이 공유하는 구조가 하나 있습니다: 자동으로 보증할 수 없는 것은 컴파일 에러로 알리고, 명시적 탈출구를 제공한다 — Silq의 forget, Classiq의 free/within-apply, Guppy의 명시적 discard가 그것이며, Qora의 언어 규칙 (측정하지 않은 레지스터의 |0⟩ 반납, 위반은 에러)도 같은 구조입니다.

비교 항목QoraClassiq Qmod (같은 접근의 상용 시스템)
앤실라 식별추론 (주석·표기 불필요)추론 — 같은 접근
청소 메커니즘역연산 주입역연산 주입 + 합성 엔진이 배치 결정
루프/조건문의 청소보수적 거부 (7·9장 — 완화 예정)복합문 통째 역변환으로 지원
원소 단위 정밀도원소별 판정·리포트 지원배열은 수명상 한 덩어리 — 사용자가 수동 분할
Y·CY 취급차단 (5장)허용하되 얽힌 상대에게 비치환 연산 금지 — 다른 방식의 방어
동적 값 (인덱스·경계)에러문법으로 금지 (컴파일 타임 전용)

두 시스템이 독립적으로 거의 같은 규칙 집합(비치환 쓰기 금지, 재료 보존, 순환 의존 금지)에 도달했다는 사실은, 이 규칙들이 특정 구현의 취향이 아니라 정적 컴파일 + 자동 청소라는 문제 자체의 형태임을 보여줍니다.
