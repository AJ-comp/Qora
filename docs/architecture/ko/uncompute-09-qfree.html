9. 청소의 계약과 qfree

되돌릴 수 있는 쓰기의 조건

← 8. 게이트는 상태벡터를 어떻게 바꾸는가 코스 목차 10. Y·CY가 차단되는 이유 — 위상 →

8장에서 게이트가 상태벡터를 고치는 방식을 기저 값 뒤집기·진폭 조정·중첩 생성 세 종류로 나눠 보았습니다.
자동 청소는 이 편집을 거꾸로 되돌리는 일인데, 세 종류가 모두 똑같이 되돌릴 수 있는 것은 아닙니다.
그래서 이 장에서는 자동 청소가 만족해야 하는 조건을 정확하게 정의하고, 어떤 게이트로 만들어진 값이 그 조건을 지킬 수 있는지를 가려냅니다.

컴파일러가 앤실라를 정리하는 코드를 삽입했을 때, 실행 결과는 다음 두 조건을 동시에 만족해야 합니다.

조건내용
1. 앤실라 복귀정리가 끝난 시점에 앤실라는 |0⟩ 상태여야 합니다.
2. 나머지 보존앤실라를 제외한 모든 큐빗의 양자 상태가, 정리 코드를 넣지 않고 실행했을 때의 상태와 완전히 일치해야 합니다.
측정 확률이 같은 것만으로는 부족하며, 각 기저 상태의 진폭이 부호와 위상까지 같아야 합니다.

2번 조건이 부호와 위상까지 요구하는 이유는, 진폭의 부호가 관측 가능한 양이기 때문입니다.
정리 직후의 측정 확률이 같더라도 부호가 달라져 있으면, 뒤에 오는 H 같은 게이트가 그 차이를 서로 다른 측정 결과로 바꿉니다.
8장의 "같은 값끼리 합치기" 계산에서 +와 −가 상쇄 여부를 갈랐던 것이 정확히 이 현상입니다.
상태벡터의 말로 옮기면, 생존자들의 항 구성과 진폭이 정리 전후로 완전히 같아야 한다는 뜻입니다.

어떤 게이트가 이 조건을 지킬 수 있는지는 판정 기준 하나로 판가름되는데, 바로 그 게이트가 기저 상태의 값 이외에 진폭까지 변경하는가입니다.
기저 상태를 기저 상태로 보내면서 진폭에는 어떤 위상도 추가로 곱하지 않는 연산을 학계에서 qfree라고 부르며, qfree 연산으로만 만들어진 값은 안전하게 청소할 수 있습니다.
이 기준이 8장의 편집 분류를 실제로 어떻게 가르는지, 청소가 성공하는 부류부터 한 부류씩 코드와 상태벡터로 확인합니다.

기저 값만 바꾸는 편집부터 확인합니다 — qfree 체인은 왜 안전한가

가장 먼저 확인할 부류는 기저 상태의 값만 바꾸는 편집, 곧 X·CNOT·CCX·SWAP으로만 값을 만든 경우입니다.
아래 프로그램은 앤실라 a에 X 게이트로 값을 만들고 CNOT 게이트로 out에 그 값을 복사하며, 컴파일러는 쓰기 이력의 역순인 X 한 번을 청소로 주입합니다. 

use a = Qubit[1];  use out = Qubit[1];
X(a[0]);              // a: |0⟩ → |1⟩ (기저 값만)
CNOT(a[0], out[0]);   // a의 값을 out에 복사한다. 이후 a는 쓰이지 않는다
X(a[0]);              // ← 자동 주입될 청소: a → |0⟩
var c: bit = M(out[0]);    // 결과 1, 결정론 — 주입 전과 완전 동일
시점상태벡터 (|a,out⟩)일어난 일
시작|0,0⟩ ×1두 레지스터 모두 |0⟩
X(a) 후|1,0⟩ ×1a의 기저 값만 0→1
CNOT 후|1,1⟩ ×1out의 기저 값만 0→1 (값 복사)
청소 X(a) 후|0,1⟩ ×1a는 |0⟩ 반납, out은 그대로 — c = 1 확정

진폭 열이 처음부터 끝까지 ×1 그대로인 데서 보이듯, 모든 편집이 기저 값만 바꾸었고 진폭은 어느 시점에도 변하지 않았습니다.
그래서 청소를 넣어도 생존자 out의 상태는 청소를 넣지 않았을 때와 완전히 같고, 2번 조건이 그대로 충족됩니다.
앤실라 a도 마지막 항에서 |0⟩로 돌아왔으므로 1번 조건까지 충족됩니다.
이것이 qfree 체인의 안전성이며, 교과서의 앤실라 사용 패턴(가역 논리 체인, Grover 오라클, 산술 회로)이 전부 이 부류에 속합니다.

그럼 체인 중간에 위상 편집이 끼면 어떻게 될까요 — Z는 왜 무해한가

첫 예제의 체인은 기저 값만 만졌는데, 실제 알고리즘에서는 표의 두 번째 항에 해당하는 게이트가 그 체인 중간에 끼는 일이 흔합니다.
Z 게이트는 기저 상태의 값은 그대로 두고, 값이 1인 기저 상태의 진폭에 −1을 곱하는 유니터리입니다.
상태벡터의 말로 옮기면, 기저 값을 전혀 고치지 않고 그 항의 부호만 바꾸는 편집입니다.
값을 바꾸지 않으므로 앤실라에 Z 게이트를 찍어도 쓰기 이력에는 아무것도 추가되지 않고, 이미 세워 둔 청소 계획이 그대로 유효합니다.
아래 프로그램에서 앤실라 a를 쓰는 편집은 CNOT 하나뿐이므로, 중간에 Z 게이트가 끼어 있어도 청소는 여전히 CNOT 한 번입니다. 

use q = Qubit[1];  use a = Qubit[1];
H(q[0]);              // q: 중첩 생성
CNOT(q[0], a[0]);     // a가 q의 값을 기록 (기저 값만 — 쓰기)
Z(a[0]);              // a=1인 줄의 부호만 −로 (진폭만 — 읽기)
CNOT(q[0], a[0]);     // ← 자동 주입될 청소: a → |0⟩ (Z는 쓰기 이력에 없음)
H(q[0]);
var m: bit = M(q[0]);      // m = 1 (확률 100%)
시점상태벡터 (|q,a⟩)일어난 일
H · CNOT 후|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2a가 q의 값을 기록 (얽힘)
Z(a) 후|0,0⟩ ×1/√2 · |1,1⟩ ×−1/√2기저 값 무접촉 — 부호만 편집
청소 CNOT 후|0,0⟩ ×1/√2 · |1,0⟩ ×−1/√2a는 모든 항에서 0 — |0⟩ 반납 완료
H(q) 후|1,0⟩ ×1m = 1 확정

Z 게이트가 기저 값을 건드리지 않았기 때문에 Z 이전과 이후의 기저 값 배치는 완전히 같고, 따라서 쓰기 이력의 역순인 CNOT 한 번으로 a는 정확히 |0⟩로 돌아갑니다.
위상이 사라지는 것도 아닙니다.
−1 부호는 a가 |0⟩로 반납된 뒤에도 q=1인 항에 그대로 남아서, 마지막 H 게이트의 간섭 방향을 바꾸고 측정 결과를 0에서 1로 뒤집습니다.
실제로 위 프로그램에서 Z 한 줄을 지우면 두 CNOT 게이트와 두 H 게이트가 전부 상쇄되어 m은 100% 확률로 0이 됩니다.
값을 안 쓰는 진폭 편집은 이렇게 청소를 방해하지 않으면서 정보(부호)만 생존자에게 옮겨 주며, Grover 오라클의 위상 킥백이 정확히 이 패턴입니다.
