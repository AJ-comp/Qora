5. Y·CY가 차단되는 이유 — 위상 도장

값에 따라 달라지는 위상 도장과 얽힘

← 4. 청소의 계약과 qfree 코스 목차 6. 주입기 — 문장 붙여넣기 기계 →

Y는 X와 마찬가지로 0과 1을 맞바꾸는 치환 게이트입니다. 차이는 하나뿐입니다. Y는 값을 뒤집으면서 뒤집은 값에 따라 서로 다른 위상 도장을 함께 찍습니다. 구체적으로 Y|0⟩ = +i|1⟩이고 Y|1⟩ = −i|0⟩입니다. 2장에서 위상은 큐빗 하나의 소유물이 아니라 줄 전체의 속성이라는 사실을 확인했습니다. 두 사실을 결합하면 문제가 생기는 조건이 정확히 하나로 좁혀집니다. 타깃이 얽혀 있어서 줄마다 타깃 값이 다르면, 도장도 줄마다 달라져서 상대 위상이 됩니다. CY 역시 컨트롤이 1인 줄에서 타깃에 같은 도장을 찍으므로 동일한 문제를 그대로 가집니다.

반례 프로그램 — 단계별 장부

아래 프로그램이 그 조건을 최소 형태로 재현합니다. b를 중첩에 놓고, CNOT으로 a를 b와 얽은 다음, 얽힌 a에 Y를 적용합니다. 

use a = Qubit[1];  use b = Qubit[1];
H(b[0]);  CNOT(b[0], a[0]);  Y(a[0]);  H(b[0]);  bit m = M(b[0]);
시점장부 (|b,a⟩)설명
H(b)(|0,0⟩ + |1,0⟩)/√2줄 2개
CNOT(b,a)(|0,0⟩ + |1,1⟩)/√2a가 b와 얽힘 — 줄마다 a 값이 다름
Y(a)(+i|0,1⟩ −i|1,0⟩)/√21행은 a=0을 뒤집어 +i, 2행은 a=1을 뒤집어 −i — 줄마다 다른 도장

Y 직후가 a의 사망점입니다. a는 그 뒤로 아무 데도 쓰이지 않으므로, 정리 코드가 들어간다면 바로 이 지점입니다. 여기서 가능한 세 가지 미래를 비교하면 결과가 다음과 같이 갈립니다.

갈래이후 H(b); M(b)의 결과
① 청소하지 않음m = 0/1 각 50%
② 계약이 요구하는 상태 (a만 |0⟩, 부호 유지)b가 |−⟩ 상태 → m=1 확률 100%
③ 역연산 주입 (Y†; CNOT†)도장까지 회수되어 b가 |+⟩ 상태 → m=0 확률 100%
② ≠ ③ — 계약을 지킬 방법이 존재하지 않습니다

②와 ③ 어느 갈래에서도 a 자체는 문제없이 |0⟩로 돌아옵니다(①은 a를 얽힌 채 방치하는 갈래로, 비교를 위한 기준선일 뿐입니다). 두 갈래를 가르는 것은 a가 아니라 생존자 b의 부호입니다. Y의 도장은 a의 소유물이 아니라 줄의 공동 재산이어서, 역연산이 도장을 회수하는 순간 b의 재산을 건드린 것이 됩니다. 그렇다고 도장을 남기는 청소는 역연산으로 만들 수 없습니다. 따라서 얽힐 수 있는 앤실라에 대한 Y·CY 쓰기는 차단이 유일하게 정직한 답이며, Silq의 qfree 정의가 Y를 제외하는 것과 같은 결론입니다.

예제 — 얽히지 않은 타깃 위의 Y는 무해합니다

반대로 타깃이 얽혀 있지 않다면, 즉 모든 줄에서 타깃 값이 같다면 도장도 모든 줄에 똑같이 찍힙니다. 전줄 공통 인수는 전역 위상이고, 전역 위상은 어떤 측정으로도 관찰되지 않습니다. 아래 한 줄짜리 사례에서 Y의 +i 도장은 결과에 아무 흔적도 남기지 않습니다. 

use q = Qubit[1];
Y(q[0]);            // |0⟩ → +i|1⟩ — 줄이 하나뿐이므로 +i는 전역 위상
bit m = M(q[0]);    // m = 1 (확률 100%) — X(q[0])를 썼을 때와 완전히 같은 결과

다만 현재 판정은 이 구분을 아직 활용하지 않습니다. 얽힘 여부와 무관하게 Y·CY 쓰기를 일괄 차단하는 보수적 형태이며, 확정 기저값 위의 Y를 다시 허용하는 정밀화는 계획된 개선입니다(9장).

같은 렌즈로 보면 Z·S·T·CZ가 안전한 이유도 분명해집니다. 이 게이트들은 위상 도장만 찍을 뿐 값을 쓰지 않는 읽기입니다. 예를 들어 Z는 타깃이 1인 줄에 −1 도장을 찍지만 타깃의 값 자체는 그대로 둡니다. 값을 쓰지 않으므로 앤실라의 청소 체인에 아예 들어가지 않고, 그 위상은 생존자 값의 함수로 남아 보존됩니다.
