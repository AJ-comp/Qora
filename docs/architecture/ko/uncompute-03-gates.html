3. 게이트는 장부를 어떻게 편집하는가

값 칸 · 계수 칸 · 줄 쪼개기 — 편집의 세 종류

← 2. 상태를 읽는 법 — 장부 모델 코스 목차 4. 청소의 계약과 qfree →

2장에서 양자 상태를 줄과 계수 칸으로 이루어진 장부로 읽는 법을 익혔습니다. 그 장부 위에서 보면 모든 게이트는 장부를 고치는 편집 연산입니다. 어떤 게이트는 줄의 값 칸만 고치고, 어떤 게이트는 값 칸을 그대로 둔 채 계수에만 위상을 곱하며, 어떤 게이트는 줄 하나를 두 줄로 쪼갭니다. 게이트가 이 중 어떤 편집을 하는지가 뒤 장들에서 자동 청소의 안전성을 판정하는 기준이 됩니다.

게이트장부에 하는 일비고
X각 줄에서 해당 값 칸만 뒤집기계수 무접촉
CNOT · CCX · SWAP조건에 따라 값 칸 뒤집기/맞바꾸기값 칸만, 계수 무접촉
Z · S · T · CZ · Rz값 칸은 안 건드리고 계수에만 위상 곱하기효과분석에서는 읽기(Read)로 기록
H · Rx · Ry줄을 두 줄로 쪼개기 (하나의 값 → 두 값의 중첩)1줄 장부 → 2줄 장부
Y · CY값 칸 뒤집기 + 뒤집은 값에 따라 다른 도장을 그 줄 계수에0→1이면 +i, 1→0이면 −i
M (측정)한 값으로 줄들을 삭제하고 결과를 bit로 배달비가역
Reset값을 강제로 |0⟩ (이력 소실)비가역

표에서 값 칸 편집, 계수 편집, 줄 쪼개기는 모두 되돌릴 수 있는 편집입니다. 예를 들어 X를 두 번 적용하면 값 칸이 원래대로 돌아오고, H를 두 번 적용하면 쪼개졌던 줄이 간섭으로 다시 한 줄로 합쳐집니다. 반면 마지막 두 줄의 측정과 Reset은 장부의 줄을 지워 버리는 동작이라서 한 번 지나가면 되돌릴 수 없습니다. 측정이 줄을 삭제한다는 말이 무슨 뜻인지 가장 작은 예로 확인해 봅니다. 

use q = Qubit[1];
H(q[0]);           // 장부: |0⟩ ×1/√2 · |1⟩ ×1/√2 — 두 줄
bit m = M(q[0]);   // 두 줄 중 하나만 남기고 삭제 → m = 0 또는 1 (각 50%)

측정 직전의 장부에는 줄이 두 개 있었습니다. 측정은 그중 한 줄만 남기고 나머지를 삭제한 뒤, 남은 줄에 적힌 값을 bit 변수 m으로 배달합니다. 지워진 줄의 내용은 어디에도 남지 않으므로, 그 뒤에 어떤 게이트를 붙여도 삭제 이전의 장부를 복원할 수 없습니다. 측정과 Reset이 표에서 비가역으로 표시된 이유가 바로 이것입니다.

예제 — 장부의 진화를 단계별로

이번에는 세 종류의 편집이 실제로 어떻게 이어지는지, 두 큐빗 프로그램 하나로 처음부터 끝까지 따라가 봅니다. 

use b = Qubit[1];  use a = Qubit[1];
H(b[0]);           // ① 줄 쪼개기
CNOT(b[0], a[0]);  // ② 조건부 값 편집
Z(a[0]);           // ③ 계수 편집
시점장부 (|b,a⟩)일어난 편집
시작|0,0⟩ ×1줄 하나
① H(b)|0,0⟩ ×1/√2 · |1,0⟩ ×1/√2줄이 둘로 쪼개짐
② CNOT(b,a)|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2b=1인 줄에서만 a 값 칸 뒤집힘 — 얽힘 탄생 (줄마다 a 값이 다름)
③ Z(a)|0,0⟩ ×1/√2 · |1,1⟩ ×−1/√2a=1인 줄의 계수에만 −1 — 값 칸 무접촉 (읽기)

②에서 일어난 일이 곧 얽힘의 장부적 정의입니다. CNOT이 b가 1인 줄에서만 a의 값 칸을 뒤집었기 때문에, 편집이 끝난 장부에서는 줄마다 a의 값이 다릅니다. 이렇게 줄마다 어떤 큐빗의 값이 달라서 장부를 큐빗별 곱으로 쪼갤 수 없는 상태를 얽힘이라 부릅니다. 반면 ③의 Z는 값 칸을 전혀 건드리지 않고 a가 1인 줄의 계수에 −1을 곱했을 뿐입니다. 효과분석은 이처럼 값 칸을 건드리지 않는 계수 전용 편집을 읽기(Read)로 기록하며, 그 이유는 다음 장에서 분명해집니다.
