3. 게이트는 장부를 어떻게 편집하는가

값 칸 · 계수 칸 · 줄 쪼개기 — 편집의 세 종류

← 2. 상태를 읽는 법 — 장부 모델 코스 목차 4. 청소의 계약과 qfree →

모든 게이트는 장부 편집 연산으로 읽을 수 있습니다. 편집의 종류가 게이트의 안전성을 결정합니다.

게이트장부에 하는 일비고
X각 줄에서 해당 값 칸만 뒤집기계수 무접촉
CNOT · CCX · SWAP조건에 따라 값 칸 뒤집기/맞바꾸기값 칸만, 계수 무접촉
Z · S · T · CZ · Rz값 칸은 안 건드리고 계수에만 위상 곱하기효과분석에서는 읽기(Read)로 기록
H · Rx · Ry줄을 두 줄로 쪼개기 (하나의 값 → 두 값의 중첩)1줄 장부 → 2줄 장부
Y · CY값 칸 뒤집기 + 뒤집은 값에 따라 다른 도장을 그 줄 계수에0→1이면 +i, 1→0이면 −i
M (측정)한 값으로 줄들을 삭제하고 결과를 bit로 배달비가역
Reset값을 강제로 |0⟩ (이력 소실)비가역

예제 — 장부의 진화를 단계별로

use b = Qubit[1];  use a = Qubit[1];
H(b[0]);           // ① 줄 쪼개기
CNOT(b[0], a[0]);  // ② 조건부 값 편집
Z(a[0]);           // ③ 계수 편집
시점장부 (|b,a⟩)일어난 편집
시작|0,0⟩ ×1줄 하나
① H(b)|0,0⟩ ×1/√2 · |1,0⟩ ×1/√2줄이 둘로 쪼개짐
② CNOT(b,a)|0,0⟩ ×1/√2 · |1,1⟩ ×1/√2b=1인 줄에서만 a 값 칸 뒤집힘 — 얽힘 탄생 (줄마다 a 값이 다름)
③ Z(a)|0,0⟩ ×1/√2 · |1,1⟩ ×−1/√2a=1인 줄의 계수에만 −1 — 값 칸 무접촉 (읽기)

②가 곧 "얽힘"의 장부적 정의입니다: 줄마다 어떤 큐빗의 값이 달라서, 장부를 큐빗별 곱으로 쪼갤 수 없는 상태. ③처럼 값 칸을 건드리지 않는 편집은 효과분석에서 읽기(Read)로 기록되며, 그 이유는 다음 장에서 분명해집니다.
