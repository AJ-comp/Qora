Hゲートを矢印で理解する

キュービットの状態・振幅・干渉を一歩ずつ明確に説明します

この文書では、量子ゲートの中で最も基本となるHゲート（アダマールゲート）を、 複雑な行列式の代わりに「振幅を表す矢印」で説明します。 特に、|0⟩ 状態にHゲートを2回適用するとなぜ再び |0⟩ 状態に戻るのかを、 可能な測定値0と1の振幅を一歩ずつ実際に足し合わせて確認します。

1キュービットの状態は可能な測定値ごとに振幅を持つ

1つのキュービットを測定すると、測定値は0または1のどちらかになります。 測定する前のキュービットの状態には、可能な測定値0と可能な測定値1のそれぞれに振幅が付いています。 この文書では、各測定値の振幅を矢印で描きます。矢印の長さは振幅の大きさで、 矢印の向き（右/左）は振幅の符号（+/−）です。

最も単純なキュービットの状態である |0⟩ 状態から見てみましょう。|0⟩ 状態では、測定値0の振幅は1で、測定値1の振幅は0です。

可能な測定値振幅の矢印振幅の値
0───▶1（フル）
1·0（なし）

測定値0の振幅が1で、測定値1の振幅が0であるキュービットの状態を |0⟩ と呼びます。

各測定値に付いた振幅の長さを2乗した値 = その測定値が出る確率

そのため、|0⟩ 状態を測定すると、測定値0が出る確率は 1² = 100% で、 測定値1が出る確率は 0² = 0% です。 つまり測定値は必ず0になります。振幅の向きと符号は第2節から使うので、第1節では振幅の長さだけを見れば十分です。

2Hゲートを1回適用すると、1つのキュービットが2つの測定値の振幅を持つ

Hゲートは、測定値0に集まっていた振幅を、そのまま測定値0だけに置いておきません。 Hゲートは、測定値0に集まっていた振幅を測定値0へ向かう振幅の寄与と 測定値1へ向かう振幅の寄与に分けて送ります。 |0⟩ 状態にHゲートを適用すると、測定値0にあった振幅1が、 測定値0の振幅 1/√2 と測定値1の振幅 1/√2 に変わります。

|01⟩ 状態になるわけではありません |0⟩ 状態にHゲートを適用した結果は、 1つのキュービットが測定値0と測定値1に対する振幅を同時に持つ状態です。 数式で書くと (|0⟩ + |1⟩) / √2 です。 |01⟩ は2つのキュービットがあるときに使う状態の表記です。
可能な測定値Hゲート適用前の振幅Hゲート適用後の振幅Hゲート適用後の測定確率
0───▶ 1──▶ 1/√2 ≈ 0.70750%
1· 0──▶ 1/√2 ≈ 0.70750%

Hゲートを適用した後は、測定値0の振幅の矢印が短くなります。振幅の長さが1から 1/√2（約0.707）に減ります。 同時に、Hゲートを適用する前は0だった測定値1の振幅も、1/√2 として新しく生まれます。 したがって、Hゲートを適用した後にキュービットを測定すると、測定値0が出る確率も50%、測定値1が出る確率も50%になります。

3なぜ振幅は1/√2なのか？ — 振幅を2乗してはじめて確率になる

測定値0と測定値1がそれぞれ50%の確率で出るなら、振幅もそれぞれ1/2であるように見えるかもしれません。 しかし、Hゲートを適用した後の振幅は1/2ではなく1/√2です。 理由は、確率 = 振幅²という規則と、すべての測定値の確率を足すと1という規則が 同時に成り立たなければならないからです。

もし測定値0の振幅と測定値1の振幅を両方とも1にすると：

1² + 1² = 2 = 200%   ❌（確率が200%になることはあり得ない）

そこで、測定値0の振幅と測定値1の振幅を両方とも√2で割って、全体の確率が100%になるようにします：

(1/√2)² + (1/√2)² = ½ + ½ = 1 = 100%   ✓
振幅 ↔ 確率（同じ規則、2つの向き） 確率 = 振幅² なら、逆に 振幅 = √確率 です。 （面積 = 辺² ↔ 辺 = √面積 と同じ1つの式です。） そのため、量子状態には「確率の平方根」である振幅を書き、キュービットを測定するときは振幅を2乗して確率を計算します。

図で見る — 振幅は正方形の「辺」、確率は正方形の「面積」

面積（確率）を半分（100% → 50%）に減らすと、 辺（振幅）は半分（0.5）ではなく1/√2（≈0.707）に縮みます。 正方形の面積は辺の2乗だからです。 そのため、Hゲートを適用した後の振幅は1/2ではなく1/√2です。

これはたとえ話ではなく、実際の物理法則です 波（光・音）では、強さ（光の明るさ・音の大きさ）は振幅の2乗に比例します。
そのため、光の明るさを半分にするには、振幅を0.5倍ではなく0.707倍（1/√2）に下げる必要があります。この1/√2という数は、Hゲートに現れる1/√2と同じ数です。

量子力学の確率も同じ2乗の規則を使います — |振幅|² = 確率。この2乗の規則がボルンの規則（Born rule）です。
だからこそ、量子でも「振幅（amplitude）」という言葉を使います。

（2乗されるのは振幅です。周波数は別の軸なので — 光の色、音の高さ — 確率には入りません。）

4Hゲートの正確な振幅変換規則

Hゲートが振幅を変える仕組みは、まず大きな規則として眺め、続いて4つのステップに分けて計算してみることができます。

① 各入力振幅を測定値0側と測定値1側の振幅の寄与に分けて送る  →  ② 各振幅の寄与に符号と1/√2を掛けたあと、同じ測定値どうしで足し合わせる

|0⟩ 状態、つまり測定値0の振幅が1で測定値1の振幅が0の状態にHゲートを適用するとどうなるかを見てみましょう。

測定値0の振幅の矢印は1から1/√2へと短くなり、 振幅のなかった測定値1には1/√2の振幅が新しく生まれます。 測定値0の振幅の減少と測定値1の振幅の生成が、 |0⟩ 状態にHゲートを適用したときの変化です。

どの入力振幅がどの測定値の振幅に寄与するのか？

ステップ1

Hゲート適用前の2つの振幅に名前を付けます

Hゲートは入力状態の片方の振幅だけを使うわけではありません。 適用前の測定値0の振幅と適用前の測定値1の振幅の両方を使います。

記号意味|0⟩ 状態での値
a₀ Hゲート適用前の測定値0の振幅 1
a₁ Hゲート適用前の測定値1の振幅 0
ステップ2

各入力振幅が2つの測定値へ振幅の寄与を送ります

適用前の測定値0の振幅も測定値0と1へ寄与を送り、 適用前の測定値1の振幅も測定値0と1へ寄与を送ります。 そのため、全部で4つの振幅の寄与が生まれます。

出発する入力振幅 測定値0へ向かう寄与 測定値1へ向かう寄与
適用前の測定値0の振幅 (a₀) + a₀ × 1/√2 + a₀ × 1/√2
適用前の測定値1の振幅 (a₁) + a₁ × 1/√2 − a₁ × 1/√2
ステップ3

同じ測定値に到着した振幅の寄与どうしを足します

測定値0に到着した2つの振幅の寄与を足すと、Hゲート適用後の測定値0の振幅になります。 測定値1に到着した2つの振幅の寄与を足すと、Hゲート適用後の測定値1の振幅になります。

Hゲート適用後の振幅足し合わせる振幅の寄与一般式
測定値0の振幅 0から来た寄与 + 1から来た寄与 a₀ × 1/√2 + a₁ × 1/√2
測定値1の振幅 0から来た寄与 + 1から来た寄与 a₀ × 1/√2 + a₁ × (−1/√2)
ステップ4

|0⟩ 状態の実際の振幅の値を代入します

|0⟩ 状態では a₀ = 1 で a₁ = 0 です。 したがって、ステップ1で名前を付けた2つの振幅の値をステップ3の一般式に入れればよいのです。

計算対象代入式結果
Hゲート適用後の測定値0の振幅 1 × 1/√2 + 0 × 1/√2 1/√2
Hゲート適用後の測定値1の振幅 1 × 1/√2 + 0 × (−1/√2) 1/√2

符号の規則 — 4つの振幅移動の組み合わせのうち、1つの組み合わせだけにマイナスが付く

Hゲートの変換式でHゲート適用後の測定値1の振幅を計算するとき、 適用前の測定値1の振幅にだけ −1/√2 が掛けられます。 マイナス符号が付く位置は、 どの測定値から来た振幅がどの測定値へ向かうかで決まります。

核心となる符号規則 4つの振幅移動の組み合わせのうち、 測定値1から来て測定値1へ向かう組み合わせにだけマイナス符号が付きます。 この組み合わせを短く書くと 1 → 1 項です。 「適用前も1側の振幅で、適用後も1側の振幅として寄与する」という意味です。
Hゲート適用前の振幅測定値0へ向かう振幅の寄与測定値1へ向かう振幅の寄与
測定値0の振幅+ ▶ 1/√2+ ▶ 1/√2
測定値1の振幅+ ▶ 1/√2− ◀ 1/√2

測定値0へ向かう振幅の寄与は常に正です。測定値1へ向かう振幅の寄与は、適用前の振幅が測定値0から来たなら正で、適用前の振幅が測定値1から来たなら負です。

5なぜ測定値1から来て測定値1へ向かう項にマイナス符号が付くのか

ステップ1

量子ゲートは入力の情報を失ってはいけません

Hゲートも量子ゲートなので、必ず元に戻せる（可逆である）必要があります。 元に戻せるということは、異なる入力状態が同じ出力状態に合流してはいけないという意味です。

ステップ2

マイナス符号がないと、2つの入力状態が同じ出力状態に合流してしまいます

Hゲートからマイナス符号を取り除いてしまうと、 |0⟩ 状態と |1⟩ 状態が同じ出力状態へ行ってしまいます。

マイナス符号がない場合

入力 |0⟩ (|0⟩ + |1⟩) / √2
入力 |1⟩ (|0⟩ + |1⟩) / √2

2つの入力状態が同じ出力状態になります。出力だけを見ても、元の入力が何だったのか分かりません。

マイナス符号がある場合

入力 |0⟩ (|0⟩ + |1⟩) / √2 = |+⟩
入力 |1⟩ (|0⟩ − |1⟩) / √2 = |−⟩

2つの出力状態は互いに直交します。互いに完全に区別できる2つの状態として残ります。

ステップ3

マイナス符号は出力状態を区別できるようにするために必要です

Hゲートを |0⟩ 状態に適用した結果と、 Hゲートを |1⟩ 状態に適用した結果は、 単に少し違う程度ではなく、完全に区別できる2つの方向（直交）でなければなりません。

2つの出力状態の直交性を作り出しているのが、 測定値1から来て測定値1へ向かう項のマイナス符号です。

ステップ4

現在の表記では、マイナス符号は測定値1から来て測定値1へ向かう項に置かれます

Hゲートを |0⟩ 状態に適用した結果を (|0⟩+|1⟩)/√2 と書き、 測定値0へ向かう振幅の寄与は正にしておくという表記上の約束を決めます。

この約束のもとで、Hゲートを |1⟩ 状態に適用した結果が Hゲート(|0⟩) と直交するためには、 測定値1側の項が反対の符号でなければなりません。

核心 マイナス符号は勝手に入れた記号ではなく、Hゲートが元に戻せるために必要な符号です。 現在の表記上の約束では、その符号は 測定値1から来て測定値1へ向かう項に置かれます。 この符号のおかげで、Hゲートを2回適用すると最初の状態に戻ります。

6Hゲートを2回適用すると、干渉によって |0⟩ 状態がよみがえる

|0⟩ 状態にHゲートを続けて2回適用すると、キュービットの状態は再び |0⟩ 状態に戻ります。 古典的な50/50のコイン投げなら、「ランダム化の過程を2回通してもやはり50/50のまま」と考えるかもしれません。 しかし、Hゲートはコイン投げではなく、振幅を足し合わせる量子ゲートです。なぜ再び |0⟩ 状態になるのか、振幅を実際に足してみましょう。

元に戻る仕組みの核心 第4節のHゲートの振幅変換式を、1回目のHゲート適用後の状態に そのままもう一度適用すればよいのです。 代入すると、測定値0の振幅は強め合って+1になり、 測定値1の振幅は打ち消し合って0になります。
ステップ1

1回目のHゲート適用後の振幅を、2回目のHゲートの入力として置きます

第4節で、Hゲート変換式の入力振幅に a₀ と a₁ という名前を付けました。 2回目のHゲートを適用するときは、1回目のHゲートが作った振幅が そのまま a₀ と a₁ になります。

記号2回目のHゲート適用前の意味値
a₀ 1回目のHゲート適用後の測定値0の振幅 1/√2
a₁ 1回目のHゲート適用後の測定値1の振幅 1/√2
ステップ2

第4節のHゲートの振幅変換式をそのままもう一度使います

新しい規則を追加で覚えるわけではありません。 第4節で作った一般式に a₀ = 1/√2 と a₁ = 1/√2 を代入すればよいのです。

2回目のHゲート適用後の振幅第4節の一般式代入式
測定値0の振幅 a₀ × 1/√2 + a₁ × 1/√2 (1/√2) × 1/√2 + (1/√2) × 1/√2
測定値1の振幅 a₀ × 1/√2 + a₁ × (−1/√2) (1/√2) × 1/√2 + (1/√2) × (−1/√2)
ステップ3

代入式の値を計算すると、元に戻る様子がすぐに見えます

(1/√2) × (1/√2) = 1/2 なので、 2回目のHゲート適用後の2つの振幅は次のように計算されます。

測定値0に到着した振幅の寄与
+1/2 + +1/2 = +1
同じ符号なので、強め合う干渉が起こります。

測定値0の振幅は生き残ります。 測定確率は100%です。

測定値1に到着した振幅の寄与
+1/2 + −1/2 = 0
反対の符号なので、打ち消し合う干渉が起こります。

測定値1の振幅は消えます。 測定確率は0%です。

Hゲート(Hゲート(|0⟩)) = |0⟩ 2回目のHゲート適用の結果 測定値0の振幅 = +1（測定確率 100%） 測定値1の振幅 = 0（測定確率 0%） したがって再び |0⟩ 状態
なぜ2回目のHゲート適用後の測定結果は50/50ではなく0に確定するのか

測定値1に到着する振幅の寄与は2つあります。 |0⟩ 成分から来た振幅の寄与は+1/2で、 |1⟩ 成分から来た振幅の寄与は−1/2です。

測定値1に到着する2つの振幅の寄与は、 符号が反対なので足すと0になります。 測定値1の振幅が0になる現象が打ち消し合う干渉です。

同時に、測定値0に到着する2つの振幅の寄与は +1/2と+1/2なので、足すと+1になります。

測定値1側の −1/2 は、まさに第4節の 「測定値1から来て測定値1へ向かう項には−が付く」という 規則から出てきたものです。

もしマイナスがなかったら？

測定値1の振幅は +1/2 + 1/2 = +1 となり、 打ち消されなかったはずです。

同時に測定値0の振幅も+1になり、 2つの測定値の確率を足すと1を超える、有効でない変換になってしまいます。

測定値1から来て測定値1へ向かう項のマイナス符号が、 測定値1の振幅を打ち消し、全体の確率が1に保たれるようにしています。

Qoraで実習してみる

Hゲートを2回適用する回路をQoraで書いてみます

以下のQoraコードは、キュービットを1つ準備した後、同じキュービットにHゲートを2回適用して測定します。 この文書で計算した Hゲート(Hゲート(|0⟩)) = |0⟩ の回路です。

Qoraコード
operation Main() {
    // キュービットを1つ準備します。
    // 準備直後の q[0] は |0⟩ 状態です。
    // 測定値0の振幅 = 1、測定値1の振幅 = 0 です。
    use q = Qubit[1];

    // 1回目のHゲートを q[0] に適用します。
    // |0⟩ 状態が (|0⟩ + |1⟩) / √2 状態に変わります。
    // 測定値0の振幅 = 1/√2 です。
    // 測定値1の振幅 = 1/√2 です。
    H(q[0]);

    // 2回目のHゲートを同じ q[0] にもう一度適用します。
    // 測定値0の振幅 = +1/2 + +1/2 = +1 です。
    // 測定値1の振幅 = +1/2 + -1/2 = 0 です。
    H(q[0]);

    // 2回目のHゲート適用後の q[0] を測定します。
    // 測定値0が出る確率 = 100% です。
    // 測定値1が出る確率 = 0% です。
    var result: bit = M(q[0]);
}

証拠1

電子や光子を2つのスリットに1個ずつ送っても、スクリーンには 明るい縞と暗い縞が積み上がっていきます。

証拠2

Hゲートを2回適用する回路

理想的な量子回路や量子シミュレータでは、|0⟩ 状態に Hゲートを2回適用して測定すると、測定値0が100%出ます。

実際の量子コンピュータの装置でも、ノイズのために結果が少しにじむだけで、 測定値0が測定値1より圧倒的に多く出ます。

キュービットが単なる50/50のコインなら、Hゲートを2回適用する過程は 「ランダム化の過程を2回通すこと」と同じはずで、 最終的な測定確率もやはり50/50になるはずです。

状態

キュービットの状態は確率そのものではなく振幅で表現します

1つのキュービットの状態は、可能な測定値0と1のそれぞれに付いた 振幅で表現します。

測定確率 = 振幅の長さ²
正規化

確率の合計は必ず1でなければなりません

2つの測定値の確率が50%と50%なら、 各測定値の振幅は1/2ではなく1/√2です。

(1/√2)² + (1/√2)² = 1
Hゲート

Hゲートは入力振幅を2つの測定値に分けて送ります

各入力振幅は、測定値0側と測定値1側の振幅の寄与になります。 同じ測定値に到着した振幅の寄与どうしは足し合わせます。

測定値1から来て測定値1へ向かう項だけが − です。
Hゲート2回

同じ変換式をもう一度適用すると元の状態に戻ります

1回目のHゲートの後は a₀ = 1/√2、 a₁ = 1/√2 です。 この値をHゲートの変換式にもう一度入れればよいのです。

測定値0側: +1/2 + +1/2 = +1
測定値1側: +1/2 + −1/2 = 0
