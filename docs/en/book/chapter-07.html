Chapter 7. How Do We Read Qora Code?

Finally, let's look together at the basic shape of the Qora code you'll be seeing often from now on.
Qora code isn't a notation that shows you the quantum state directly, but it does state clearly which gates are applied to which qubits, and in what order.
So mathematically, you can read it as a circuit description that applies unitary transformations to the state vector one after another.

A Qora program starts with operation, and the operation that runs first is usually named Main.

Basic form

Let's start with the smallest Qora program

operation Main() {
    // write the quantum circuit to run here
}

Preparing qubits

Qubit preparation starts with use.

use q = Qubit[1];

This code prepares one qubit and gives the name q to that bundle of qubits.
Qubit[1] means there is one qubit, and you refer to that qubit as q[0].
In this book, we'll read a freshly prepared qubit as starting in the computational basis state |0⟩.

Applying gates

Gates are written just like function calls.

H(q[0]);

This line applies the H gate to q[0].
Mathematically, it's the step that multiplies the current state vector by the H matrix.
The X gate is written as X(q[0]);, and the two-qubit CNOT gate as CNOT(q[0], q[1]);.

Measuring

For measurement, we use M.

bit result = M(q[0]);

This line measures q[0] and stores the result in the classical bit result.
Before measurement, the qubit is a quantum state with amplitudes, and after measurement, either the measurement outcome 0 or the measurement outcome 1 is stored in the classical bit.

Writing comments

In Qora, you can write single-line comments with //.
Comments don't affect how the circuit runs, so the Qora practice code in this book will also use them often to note state changes.

Qora code with comments

Let's note the state change on each line of code

operation Main() {
    // prepare one qubit
    use q = Qubit[1];

    // apply the H gate to q[0]
    H(q[0]);

    // measure q[0]
    bit result = M(q[0]);
}
