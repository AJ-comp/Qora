Chapter 2. How Are Amplitudes Connected to Probabilities?

We said that a qubit state is written as |ψ⟩ = α|0⟩ + β|1⟩.
Here, α and β are not probabilities — they are probability amplitudes.
The measurement probability is not the amplitude itself; you get it by squaring the absolute value of the amplitude.

Probability of measurement outcome 0 = |α|², probability of measurement outcome 1 = |β|²

This rule is called the Born rule.
When the amplitudes are real numbers you can treat it as a plain square, but in general quantum mechanics amplitudes can be complex numbers, so we compute the squared absolute value.

StateAmplitudesMeasurement probability
|0⟩α = 1, β = 0|1|² = 100%, |0|² = 0%
|1⟩α = 0, β = 1|0|² = 0%, |1|² = 100%

Why isn't the amplitude 1/2 when the probability is 50%?

If measurement outcomes 0 and 1 are fifty-fifty, it seems like each amplitude should also be 1/2 — but setting them that way breaks normalization.
A quantum state must have a total measurement probability of 100%, so it has to satisfy |α|² + |β|² = 1.

Wrong idea

What happens if we set the amplitudes to 1/2?

Each measurement outcome gets a probability of 25%.

(1/2)² + (1/2)² = 1/4 + 1/4 = 1/2

The total probability is only 50%, so this is not a valid qubit state.

Correct amplitude

Why does setting the amplitudes to 1/√2 work?

Each measurement outcome comes out to a 50% probability.

(1/√2)² + (1/√2)² = 1/2 + 1/2 = 1

The total probability is also a correct 100%.

Try it in Qora

Let's make a 50% state with the H gate

operation Main() {
    // Prepare one qubit
    // Right after preparation, q[0] is in the |0⟩ state
    // Amplitude of measurement outcome 0 = 1
    // Amplitude of measurement outcome 1 = 0
    use q = Qubit[1];

    // Apply the H gate
    // Amplitude of measurement outcome 0 = 1/√2
    // Amplitude of measurement outcome 1 = 1/√2
    // Each of the two measurement outcomes has a 50% probability
    H(q[0]);

    // Measure q[0]
    // Ideal execution environment: measurement outcomes 0 and 1 each about 50%
    bit result = M(q[0]);
}

The important point here is that the H gate does not directly split the probability in half.
What the gate changes first is the amplitudes of the state vector, and when you measure, the squared absolute values of those amplitudes appear as probabilities.
