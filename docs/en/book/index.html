Quantum Gates with Qora / Overview

In this book, we write quantum circuits in Qora code and read, step by step, which quantum-mechanical concepts that code actually touches.
The goal is not to memorize gate names, but to become able to follow, on your own, how a single line of code changes the amplitudes of a qubit's state vector.

The reference point of this book

It is okay if the terms below feel unfamiliar at first.
For now, read them as signposts for where the book is headed.
We treat a qubit as a state vector in a complex vector space, and compute measurement probabilities with the Born rule. Quantum gates are described as unitary linear transformations acting on the state vector, and Qora code is used as a tool for writing down the order of those transformations in circuit form.

The Overall Flow

Step 1 See the state as a vector
A single-qubit state is expressed as two amplitudes attached to the computational basis states |0⟩ and |1⟩.
Step 2 Get probabilities from amplitudes
An amplitude itself is not a probability; you must square its absolute value to get the measurement probability.
Step 3 Meet superposition and interference first
First distinguish the situation where amplitudes sit in several basis-state directions from the situation where amplitude contributions meet again and add or cancel.
Step 4 Read gates as linear transformations
Calculate how gates like X, H, Z, and CNOT change the amplitudes of the state vector.
Step 5 Read one small circuit from start to finish
Follow how the state changes when several gates appear in sequence, and how the final measurement probabilities are determined.
