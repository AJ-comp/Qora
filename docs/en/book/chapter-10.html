Chapter 10. How Does the CNOT Gate Connect Two Qubits?

The CNOT gate is a gate that acts on two qubits.
The first qubit is called the control qubit, and the second qubit is called the target qubit.
An X gate is applied to the target qubit only when the control qubit is on the measurement outcome 1 side.

The CNOT gate flips the target qubit only when the control qubit is 1.
Input stateControl qubitChange in target qubitOutput state
|00⟩0Unchanged|00⟩
|01⟩0Unchanged|01⟩
|10⟩10 becomes 1|11⟩
|11⟩11 becomes 0|10⟩

Here, |10⟩ means the two-qubit state where the first qubit's measurement outcome is 1 and the second qubit's measurement outcome is 0.
So if we apply the CNOT gate to |10⟩, the control qubit is 1, so the target qubit flips, and the final state becomes |11⟩.

What Happens When the Control Qubit Is in Superposition?

When we apply an H gate to the first qubit, the first qubit's amplitude splits toward measurement outcomes 0 and 1.
If we then apply the CNOT gate, the target qubit stays the same on the branch where the control qubit is 0, and the target qubit flips on the branch where the control qubit is 1.
As a result, the two qubits end up bound together in a state with just the two possibilities |00⟩ and |11⟩.

|00⟩ → H(q[0]) → (|00⟩ + |10⟩) / √2 → CNOT(q[0], q[1]) → (|00⟩ + |11⟩) / √2
This state is the starting point for understanding entanglement When you measure the final state, the two qubits do not look like they each come out randomly on their own.
The measurement outcomes are only ever 00 or 11.
That is because the measurement outcomes of the first qubit and the second qubit are locked together.
Try it in Qora

Let's Connect Two Qubits with the CNOT Gate

operation Main() {
    // Prepare two qubits
    // Right after preparation, q[0]q[1] is in the |00⟩ state
    use q = Qubit[2];

    // Apply an H gate to the first qubit
    // q[0]'s amplitude splits into measurement outcomes 0 and 1
    H(q[0]);

    // Apply the CNOT gate
    // q[0] is the control qubit, q[1] is the target qubit
    CNOT(q[0], q[1]);

    // Ideal execution environment: 00 or 11
    bit first = M(q[0]);
    bit second = M(q[1]);
}
