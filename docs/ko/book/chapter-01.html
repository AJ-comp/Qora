1장. 큐비트 상태에는 무엇이 들어 있을까?

한 큐비트를 가장 표준적으로 설명하는 방법은 상태를 벡터로 보는 거예요.
이 벡터는 두 개의 기준 방향을 가지고, 그 기준 방향을 보통 계산 기저라고 불러요.
한 큐비트의 계산 기저는 |0⟩와 |1⟩ 두 가지예요.

그래서 한 큐비트 상태는 아래처럼 쓸 수 있어요.

|ψ⟩ = α|0⟩ + β|1⟩

여기서 α는 측정 결과값 0에 붙은 진폭이고, β는 측정 결과값 1에 붙은 진폭이에요.
즉 큐비트 상태는 “나중에 0이 나올지 1이 나올지”라는 결과 하나가 아니라, 가능한 측정 결과값마다 붙어 있는 진폭 전체를 담고 있어요.

|0⟩와 |1⟩는 계산 기저 상태예요 |0⟩는 측정하면 결과값 0이 확실히 나오는 상태이고, |1⟩는 측정하면 결과값 1이 확실히 나오는 상태예요.
수학적으로는 |0⟩와 |1⟩가 한 큐비트 상태 공간의 서로 직교하는 두 기준 벡터 역할을 해요.

먼저 |0⟩ 상태를 보자

|0⟩ 상태는 α = 1, β = 0인 상태예요.
측정 결과값 0 쪽에 진폭이 전부 있고, 측정 결과값 1 쪽에는 진폭이 없어요.

가능한 측정 결과값진폭 값측정하면 나올 확률
측정 결과값 01100%
측정 결과값 100%

이 상태를 측정하면 측정 결과값 0이 반드시 나와요.
Qora에서 큐비트를 새로 준비하면 기본적으로 |0⟩ 상태에서 시작한다고 읽으면 돼요.

Qora로 실습해보기

준비한 직후의 큐비트를 측정해보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    // 측정 결과값 0의 진폭 = 1
    // 측정 결과값 1의 진폭 = 0
    use q = Qubit[1];

    // q[0] 측정
    // 이상적인 실행 환경: 측정 결과값 0
    bit result = M(q[0]);
}

이번에는 |1⟩ 상태를 보자

|1⟩ 상태는 α = 0, β = 1인 상태예요.
측정 결과값 0 쪽에는 진폭이 없고, 측정 결과값 1 쪽에 진폭이 전부 있어요.

가능한 측정 결과값진폭 값측정하면 나올 확률
측정 결과값 000%
측정 결과값 11100%

Qora에서는 |0⟩ 상태로 준비한 큐비트에 X 게이트를 적용하면 |1⟩ 상태를 만들 수 있어요.
X 게이트는 뒤에서 자세히 배우겠지만, 일단은 측정 결과값 0과 측정 결과값 1을 뒤집는 게이트로 보면 돼요.

Qora로 실습해보기

X 게이트로 |1⟩ 상태를 만들어보자

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // X 게이트 적용
    // |0⟩ 상태가 |1⟩ 상태로 바뀜
    X(q[0]);

    // 이상적인 실행 환경: 측정 결과값 1
    bit result = M(q[0]);
}
|0⟩ + |1⟩와 |01⟩: 헷갈리기 쉬운 표기 (|0⟩ + |1⟩) / √2는 한 개의 큐비트가 측정 결과값 0과 측정 결과값 1에 대한 진폭을 동시에 가진 상태예요.
반면에 |01⟩은 두 개의 큐비트가 있을 때 쓰는 기저 상태 표기예요. 첫 번째 큐비트는 0, 두 번째 큐비트는 1인 두 큐비트 상태를 뜻해요.
