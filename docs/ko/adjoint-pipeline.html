Adjoint 역합성 파이프라인

Adjoint 호출이 역연산 IR, 합성 def, OpenQASM 호출로 바뀌는 동작 흐름

Qora에서 Adjoint Foo(q)라고 쓰면, 컴파일러는 Foo의 역연산을 하는 서브루틴(Foo__adj)을 자동으로 합성해서 그 호출로 바꿔치기해요. 이 문서는 그 일이 정확히 어디에서, 어떤 순서로 일어나는지를 실제 컴파일러 출력으로 한 단계씩 따라가요. 아래에 나오는 AST·IR·OpenQASM은 전부 예제를 실제로 컴파일해서 뽑은 결과예요.

1예제 프로그램 — 다섯 구조를 전부 담은 코드

역연산 합성이 지원하는 다섯 가지 정적 구조 — 게이트, 고전 선언, for 루프, if 분기, 중첩 호출 — 를 하나의 프로그램에 전부 넣었어요. 

operation Inner(Qubit[2] q) {          // 중첩 호출의 대상
    H(q[0]);
    T(q[1]);
}

operation Outer(Qubit[2] q, bit b) {
    int k = 2;                         // ① 고전 선언
    Inner(q);                          // ② 중첩 호출
    Rx(pi/k, q[0]);                    // ③ 게이트 (선언값을 각도로 사용)
    for i in 0..1 {                    // ④ for 루프
        X(q[i]);
    }
    if (b == 1) {                      // ⑤ if 분기
        Z(q[0]);
        Adjoint S(q[0]);               //    함자가 이미 붙은 게이트
    }
}

operation Main() {
    use q = Qubit[2];
    bit r = M(q[0]);
    Outer(q, r);
    Adjoint Outer(q, r);               // ★ 이 한 줄이 전체 역합성을 일으킨다
}

21단계. AST — 파서가 만드는 문법 트리

파서(Janglim 엔진)는 소스를 문법 트리로 바꿔요. 이 단계에는 "역"의 흔적이 전혀 없어요. Adjoint Outer(q, r)도 그냥 "Adjoint 토큰이 붙은 호출 문장" 하나일 뿐이에요. Outer의 부분 트리만 보면: 

Operation
├─ "Outer"
├─ Param(2, q)   Param(bit, b)
├─ VarDecl(int, k, Expr: 2)              ← int k = 2;
├─ Gate("Inner", Expr: q)                ← 호출도 게이트도 같은 Gate 태그
├─ Gate("Rx", Expr: pi / k, Qubit(q,0))
├─ For(i, 0, 1)
│   └─ Gate("X", Qubit(q,i))
└─ If
    ├─ Condition(b == 1)
    ├─ Gate("Z", Qubit(q,0))
    └─ Gate("Adjoint", "S", Qubit(q,0))  ← 함자는 자식 토큰
이 트리는 이후에 절대 수정되지 않아요 뒤 단계들은 이 트리를 읽기만 해요. Outer__adj 같은 것은 이 시점엔 존재하지도 않고, 4단계에서 새로 만들어져요.

32단계. QoraIR — 컴파일러가 소유하는 중간 표현

Lowering(QoraLowering)이 엔진의 AST를 Qora 자신의 IR(QProgram)로 한 번 변환해요. 여기부터는 문자열 태그 대신 강타입 노드(QGate, QFor, QIf, …)로 다뤄요. Outer의 IR: 

QOperation Outer(Qubit[2] q, Bit b)
  QDecl(const=False, type=Int, name=k, value=2)
  QGate(functors=[],        name=Inner, args=[q])
  QGate(functors=[],        name=Rx,    args=[pi / k, q[0]])
  QFor(i in 0..1)
    QGate(functors=[],      name=X,     args=[q[i]])
  QIf(cond="b == 1")
    then:
      QGate(functors=[],        name=Z, args=[q[0]])
      QGate(functors=[Adjoint], name=S, args=[q[0]])

눈여겨볼 것: 함자가 functors=[Adjoint]처럼 노드의 데이터가 됐어요. "이 게이트가 역인가"가 텍스트가 아니라 값으로 표현되기 때문에, 다음 단계에서 이 값을 뒤집는 것만으로 역을 만들 수 있어요.

43단계. AdjointMaterializer — Inverter 커널로 역연산 op을 합성한다

Main에 Adjoint Outer(...)가 있으니 컴파일러는 Outer의 역이 필요해요. 실제 최상위 패스는 AdjointMaterializer예요. 이 패스가 필요한 역연산을 추적하고, Inverter 커널로 역방향 몸통을 만든 뒤 Outer__adj 같은 진짜 op으로 프로그램에 추가해요. 수학적 근거는 하나예요:

(U₁ · U₂ · ⋯ · Uₙ)†  =  Uₙ† · ⋯ · U₂† · U₁†
— 순서를 뒤집고, 각각을 역(dagger)으로

Inverter 커널이 실제로 돌려준 Outer의 역 IR:

QDecl(const=False, type=Int, name=k, value=2)     ← ① 선언: 순서 유지, 맨 앞
QIf(cond="b == 1")                                ← ⑤ if: 조건 유지, 분기 내부만 역전
  then:
    QGate(functors=[],        name=S, args=[q[0]])   ← Adjoint S → S (함자가 벗겨짐)
    QGate(functors=[Adjoint], name=Z, args=[q[0]])   ← Z → Adjoint Z
QFor(i in 1..0, step=-1)                          ← ④ for: 범위 스왑 + 역방향
  QGate(functors=[Adjoint], name=X, args=[q[i]])
QGate(functors=[Adjoint], name=Rx, args=[pi / k, q[0]])  ← ③ 역회전
QGate(functors=[Adjoint], name=Inner, args=[q])          ← ② Inner → Adjoint Inner (전이)

구조별 역전 규칙

구조정방향역방향규칙
게이트XAdjoint XAdjoint 함자 토글
이미 Adjoint인 게이트Adjoint SS역의 역 = 원본

왜 선언은 뒤집지 않을까? int k = 2; 같은 고전 계산에는 "역"이라는 것이 없어요. 그리고 Qora는 역합성 대상 안에서 변수 재할당을 거부하기 때문에, 선언된 값은 끝까지 같은 값이에요. 그래서 선언을 앞에 그대로 두면 역방향 게이트들(inv @ rx(pi / k))이 정방향과 정확히 같은 각도를 읽어요.
전이적 합성 — 소스에 없는 역도 자동으로 Outer의 역이 Adjoint Inner를 부르게 됐으니 Inner의 역도 필요해졌어요. 소스 어디에도 Adjoint Inner라고 쓴 적이 없지만, 컴파일러가 필요를 추적해서 Inner__adj까지 만들어요. 이 요구는 호출 사슬을 따라 계속 이어져요(전이적 클로저).

54단계. OpenQASM — 역 IR을 그대로 출력한다

마지막으로 Emitter(QasmEmitter)가 IR을 OpenQASM 3 텍스트로 내려요. 역 IR은 특별할 게 없는 평범한 IR이라서, 일반 문장과 똑같은 경로로 출력돼요. 전체 출력:

(방출 직전에 패스 두 개가 돕니다. 먼저 adjoint 실체화가 Adjoint Outer를 진짜 역방향 def op으로 바꿉니다 — 아래의 Inner__adj·Outer__adj가 여기서 나오며, 그래서 그것들이 평범한 def인 것입니다. 그다음 네임 맹글러가 방출할 이름을 OpenQASM에 안전하게 맞추는데, 키워드·stdgates 게이트·다른 이름과 실제로 충돌할 때만 이름을 바꿉니다 — _를 붙이고 // Qora: 노트를 남깁니다. 여기 이름들은 아무것도 충돌하지 않으므로 그대로 통과합니다. 에러와 컴파일 단계 뷰는 언제나 원래 이름을 유지합니다.) 

// generated by Qora v0.14
OPENQASM 3;
include "stdgates.inc";

def Inner(qubit[2] q) {
  h q[0];
  t q[1];
}

def Outer(qubit[2] q, bit b) {
  int k = 2;
  Inner(q);
  rx(pi / k) q[0];
  for int i in [0:1] {
    x q[i];
  }
  if (b == true) {
    z q[0];
    inv @ s q[0];
  }
}

def Inner__adj(qubit[2] q) {             ← 소스에 없던 def — 전이적으로 합성됨
  inv @ t q[1];
  inv @ h q[0];
}

def Outer__adj(qubit[2] q, bit b) {      ← Adjoint Outer가 부를 역방향 def
  int k = 2;                             ← ① 선언 유지
  if (b == true) {                          ← ⑤ 조건 유지, 내부만 역
    s q[0];
    inv @ z q[0];
  }
  for int i in [1:-1:0] {                ← ④ 역방향 루프 (1, 0 순서)
    inv @ x q[i];
  }
  inv @ rx(pi / k) q[0];                 ← ③ 역회전
  Inner__adj(q);                         ← ② 합성된 역 def 호출
}

qubit[2] q;
bit r;
r = measure q[0];
Outer(q, r);
Outer__adj(q, r);                        ← Adjoint Outer(q, r)가 이걸로 치환됨
def 순서에도 규칙이 있어요 OpenQASM은 선언한 뒤에만 사용할 수 있어요(선언-후-사용). 그래서 Emitter는 모든 def(정방향 + 합성)를 호출 의존성 순서(위상 정렬)로 배치해요 — Inner__adj가 그것을 부르는 Outer__adj보다 항상 먼저 나오는 이유예요.

6무엇이 역전되고, 무엇이 거부되는가

역을 만들 수 없는 op에 Adjoint를 걸면 컴파일 에러가 나요. 번역(Emit) 전에 검증 패스가 IR 전체를 한 바퀴 돌며 의미 에러를 전부 수집하고, 하나라도 있으면 OpenQASM을 아예 생성하지 않아요 — 잘못된 프로그램이 "성공한 척" 조용히 틀린 회로를 내놓는 일이 없도록요. 거부 이유는 두 부류예요.

구조판정이유
게이트 · for · if · 선언 · 중첩 호출✅ 역전 지원이 문서의 4단계 규칙
측정(M) · Reset❌ 영구 거부물리적으로 비가역 — 측정은 상태를 붕괴시키고 정보를 버린다
변수 재할당(k = k + 1;)❌ 아직 거부고전 상태의 시간 흐름 추적 필요 (효과 분석 단계에서)
while · repeat❌ 아직 거부반복 횟수를 컴파일 타임에 알 수 없다
use (로컬 큐비트)❌ 아직 거부보조 큐비트 정리 = 자동 uncomputation의 몫
재귀 호출❌ 거부역도 재귀가 되며 종료를 보장할 수 없다
에러 이유는 호출 사슬을 따라 전파돼요

Caller가 측정하는 Meas를 부르면, Adjoint Caller(q)는 이런 컴파일 에러가 돼요:

[QSEM001] in `Main`: `Adjoint Caller` cannot be compiled: it calls `Meas`, which cannot be inverted (it measures a qubit, and measurement is irreversible)

괄호 체인을 따라가면 근본 원인(Meas의 측정)까지 읽을 수 있어요. 에러는 첫 발견에서 멈추지 않고 한 번의 컴파일에 전부 보고돼요.

7한 장 요약

사용자 코드 ─파싱─▶ AST(역의 흔적 없음) ─Lowering─▶ QoraIR(함자가 데이터) ─검증─▶ 의미 에러면 STOP ─AdjointMaterializer─▶ 역연산 op(Inverter 커널: 토글·스왑·역순) ─Emitter─▶ OpenQASM(__adj def + 호출 치환)

Adjoint Foo(q) 한 줄이 하는 일: 컴파일러가 ① Foo의 역이 필요함을 추적하고 ② IR 위에서 "역순 + 게이트별 역"으로 역 IR을 합성하고 (필요하면 Foo가 부르는 op들의 역까지 전이적으로) ③ Foo__adj라는 def로 출력한 뒤 ④ 호출부를 Foo__adj(q)로 바꿔치기해요. 사용자는 역방향 코드를 한 줄도 쓰지 않았지만, 최종 OpenQASM에는 완전한 역연산이 들어 있어요 — C#의 async가 상태 머신을 대신 만들어주는 것과 같은 방식의 컴파일 타임 코드 합성이에요.

이 구조가 다음에 여는 것 — 자동 uncomputation 임시(보조) 큐비트는 쓰고 나서 역연산으로 되돌려 놓아야 간섭이 오염되지 않아요. 지금은 사용자가 Adjoint라고 명시할 때만 역을 합성하지만, 이 Inverter 커널을 그대로 재사용하면 컴파일러가 임시 큐비트가 더 이상 쓰이지 않는 지점을 찾아 역연산을 자동으로 주입할 수 있어요. 그게 Qora가 향해 가는 Silq식 자동 uncomputation이에요.

이 문서의 AST · QoraIR · OpenQASM 출력은 모두 위 예제를 실제 Qora 컴파일러로 컴파일해 얻은 결과예요.
