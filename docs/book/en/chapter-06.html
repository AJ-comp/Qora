Chapter 6. When Does Interference Appear?

Interference is one of the most important concepts in quantum computing.
Stated precisely, interference appears when multiple amplitude contributions to the same basis state are added together and then the Born rule is applied.
In other words, we do not add probabilities first — we add the amplitudes first and square them later.

Amplitude contributions arriving at the same basis state are added together first.

A gate acts linearly on the state vector.
So when the input state is a sum of several basis states, the gate transforms each basis component separately and then adds the results back together.
In this process, amplitude contributions gathering at the same output basis state reinforce each other when they share the same sign and phase, and cancel each other when they have opposite signs or opposite phases.

Input: α|0⟩ + β|1⟩
After the gate: α·U|0⟩ + β·U|1⟩
Constructive interference

Amplitude contributions with the same sign: reinforcement

+1/2 + +1/2 = +1

The amplitude of measurement outcome 0 grows.

Destructive interference

Amplitude contributions with opposite signs: cancellation

+1/2 + -1/2 = 0

The amplitude of measurement outcome 1 vanishes.

What does interference look like when we apply the H gate twice?

Applying the H gate once to the |0⟩ state creates both a |0⟩ component and a |1⟩ component.
Applying the H gate once more makes each of the two components send contributions toward |0⟩ and |1⟩ again, and the contributions arriving at the same basis state are added together.

Try It in Qora

Let's apply the H gate twice

operation Main() {
    // Prepare one qubit
    // Right after preparation, q[0] is in the |0⟩ state
    use q = Qubit[1];

    // Apply the first H gate
    // The amplitudes of measurement outcome 0 and measurement outcome 1 are both 1/√2
    H(q[0]);

    // Apply the second H gate
    // The amplitude contributions to measurement outcome 0 reinforce
    // The amplitude contributions to measurement outcome 1 cancel
    H(q[0]);

    // In an ideal execution environment: measurement outcome 0
    bit result = M(q[0]);
}

After the first H gate, two amplitudes remain.

Amplitude of measurement outcome 0 = 1/√2
Amplitude of measurement outcome 1 = 1/√2

When we apply the second H gate, the amplitude contributions arriving at measurement outcome 0 are added like this.

+1/2 + +1/2 = +1

The amplitude contributions arriving at measurement outcome 1 are computed the same way.

+1/2 + -1/2 = 0

So the final state returns to the |0⟩ state again.
