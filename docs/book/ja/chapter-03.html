第3章. |0⟩状態と|1⟩状態を直接読んでみよう

第1章では、1キュービット状態を次のように読むと見ました。

|ψ⟩ = α|0⟩ + β|1⟩

この章では、この式に実際の値を入れてみます。
まず |0⟩ 状態と |1⟩ 状態を計算し、そのあと測定値0と1が半々に出る状態も同じ方法で読んでみます。

まず |0⟩ 状態を見てみましょう

|0⟩ 状態は α = 1、β = 0 の状態です。
測定値0の側に振幅がすべてあり、測定値1の側には振幅がありません。

一般形に代入すると、表記の意味がはっきりします。

|ψ⟩ = α|0⟩ + β|1⟩

|0⟩ 状態では α = 1、β = 0 なので、式は次のようになります。

|ψ⟩ = 1|0⟩ + 0|1⟩

0|1⟩ は測定値1の側に振幅がないという意味なので省略できます。
1|0⟩ の係数1もふつうは書かないため、最終的に |0⟩ と書きます。

1|0⟩ + 0|1⟩ = |0⟩
可能な測定値振幅の値測定確率
測定値01|1|² = 100%
測定値10|0|² = 0%

この状態を測定すると、必ず測定値0が出ます。
Qoraで新しく準備したキュービットは、基本的に |0⟩ 状態から始まります。

Qoraで実習してみよう

準備した直後のキュービットを測定してみよう

operation Main() {
    // キュービットを1つ準備
    // 準備直後の q[0] は |0⟩ 状態
    // 測定値0の振幅 = 1
    // 測定値1の振幅 = 0
    use q = Qubit[1];

    // q[0] を測定
    // 理想的な実行環境: 測定値0
    var result: bit = M(q[0]);
}

今度は |1⟩ 状態を見てみましょう

|1⟩ 状態は α = 0、β = 1 の状態です。
測定値0の側には振幅がなく、測定値1の側に振幅がすべてあります。

同じ式に α = 0、β = 1 を入れると、次のようになります。

|ψ⟩ = 0|0⟩ + 1|1⟩

0|0⟩ は測定値0の側に振幅がないという意味なので省略できます。
1|1⟩ の係数1もふつうは書かないため、|1⟩ と書きます。

0|0⟩ + 1|1⟩ = |1⟩
可能な測定値振幅の値測定確率
測定値00|0|² = 0%
測定値11|1|² = 100%

Qoraでは、|0⟩ 状態で準備したキュービットにXゲートを適用すると |1⟩ 状態を作れます。
Xゲートについては後で詳しく見ますが、ひとまず測定値0と測定値1を反転するゲートと読めば大丈夫です。

Qoraで実習してみよう

Xゲートで |1⟩ 状態を作ってみよう

operation Main() {
    // キュービットを1つ準備
    // 準備直後の q[0] は |0⟩ 状態
    use q = Qubit[1];

    // Xゲートを適用
    // |0⟩ 状態が |1⟩ 状態に変わる
    X(q[0]);

    // 理想的な実行環境: 測定値1
    var result: bit = M(q[0]);
}

半々の状態も同じ方法で計算してみましょう

今度は、測定値0と測定値1がそれぞれ50%の確率で出る状態を見ます。
この状態は次のように書けます。

|ψ⟩ = (1/√2)|0⟩ + (1/√2)|1⟩

どのゲートがこの状態を作るのかは、まだ知らなくても大丈夫です。
今は、状態が与えられたときに測定確率をどう計算するのかだけを見ます。

可能な測定値振幅の値測定確率
測定値01/√2|1/√2|² = 1/2 = 50%
測定値11/√2|1/√2|² = 1/2 = 50%
計算の順番だけ覚えておきましょう まず状態の中にある値を振幅として読みます。
そのあと測定確率を求めるときに、各振幅の絶対値を2乗します。
先に確率を半分に分けてから状態を作るのではなく、先に振幅を読み、その振幅から確率を計算します。
|0⟩ + |1⟩ と |01⟩: 混同しやすい表記 (|0⟩ + |1⟩) / √2 は、1つのキュービットが測定値0と測定値1に対する振幅を同時に持つ状態です。
一方、|01⟩ は2つのキュービットがあるときに使う基底状態の表記です。1番目のキュービットが0、2番目のキュービットが1である2キュービット状態を意味します。
