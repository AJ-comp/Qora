7장. 두 큐비트 상태는 어떻게 읽을까?

한 큐비트에서는 가능한 측정 결과값이 0과 1 두 가지였어요.
두 큐비트가 있으면 측정 결과값은 한 자리 숫자가 아니라 두 자리 문자열이 돼요.
이 문서에서는 왼쪽 자리를 첫 번째 큐비트의 결과, 오른쪽 자리를 두 번째 큐비트의 결과로 읽어요.

새 기호가 많아 보여도 괜찮아요 지금 필요한 핵심은 간단해요.
한 큐비트에서 기저 상태가 두 개였다면, 두 큐비트에서는 기저 상태가 네 개로 늘어나요.
그래서 상태 벡터도 두 개의 진폭이 아니라 네 개의 진폭을 담아요.

두 큐비트의 계산 기저는 네 가지예요

두 큐비트를 측정하면 가능한 결과는 00, 01, 10, 11 네 가지예요.
그래서 두 큐비트의 계산 기저 상태도 네 가지로 적어요.

기저 상태첫 번째 큐비트두 번째 큐비트측정하면 보이는 결과
|00⟩0000
|01⟩0101
|10⟩1010
|11⟩1111

상태 벡터는 네 진폭을 담아요

한 큐비트 상태를 α|0⟩ + β|1⟩처럼 썼다면, 두 큐비트 상태는 네 기저 상태에 진폭을 붙여서 써요.

|ψ⟩ = a₀₀|00⟩ + a₀₁|01⟩
+ a₁₀|10⟩ + a₁₁|11⟩

여기서 a₀₀은 측정 결과값 00에 붙은 진폭이고, a₀₁은 측정 결과값 01에 붙은 진폭이에요.
같은 방식으로 a₁₀은 10에 붙은 진폭이고, a₁₁은 11에 붙은 진폭이에요.
측정 확률은 여전히 진폭의 절댓값 제곱으로 얻어요.

측정 결과값진폭측정 확률
00a₀₀|a₀₀|²
01a₀₁|a₀₁|²
10a₁₀|a₁₀|²
11a₁₁|a₁₁|²

그래서 두 큐비트 상태에서도 전체 측정 확률의 합은 100%가 되어야 해요.

|a₀₀|² + |a₀₁|² + |a₁₀|² + |a₁₁|² = 1

두 큐비트 상태가 항상 두 개의 상태를 따로 붙인 것은 아니에요

어떤 두 큐비트 상태는 첫 번째 큐비트의 상태와 두 번째 큐비트의 상태를 따로 적은 뒤, 두 상태를 합쳐서 설명할 수 있어요.
하지만 모든 두 큐비트 상태가 그렇게 분리되는 것은 아니에요.
이 차이가 다음 장에서 살펴볼 얽힘의 출발점이에요.

큐비트 두 개를 준비해보자

operation Main() {
    // 큐비트 두 개 준비
    // 준비 직후 전체 상태는 |00⟩
    // 측정 결과값 00의 진폭 = 1
    // 나머지 측정 결과값의 진폭 = 0
    use q = Qubit[2];

    // 첫 번째 큐비트 측정
    var first: bit = M(q[0]);

    // 두 번째 큐비트 측정
    var second: bit = M(q[1]);
}
다음 장으로 이어지는 질문 네 진폭을 가진 두 큐비트 상태 중에는, 첫 번째 큐비트와 두 번째 큐비트를 따로 떼어 완전히 설명할 수 없는 상태가 있어요.
