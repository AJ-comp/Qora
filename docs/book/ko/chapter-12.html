12장. H 게이트는 진폭을 어떻게 나눌까?

H 게이트는 |0⟩을 그대로 |0⟩에 묶어 두지 않아요.
한 기저 상태에 모여 있던 진폭을 |0⟩ 방향과 |1⟩ 방향으로 나눠 보내는 게이트예요.
그래서 H 게이트를 처음 볼 때는 “확률을 반으로 나누는 게이트”가 아니라, 진폭을 두 방향으로 나누고 다시 만나게 하는 게이트로 읽는 편이 좋아요.

입력 |0⟩ |0⟩ 쪽에 있던 진폭이 |0⟩ 방향과 |1⟩ 방향으로 나뉘어요.
입력 |1⟩ |1⟩ 쪽에 있던 진폭도 두 방향으로 나뉘지만, |1⟩ 방향으로 가는 기여에는 마이너스 부호가 붙어요.
두 번 적용 나뉘었던 진폭 기여가 다시 만나 보강되거나 상쇄되면서 원래 상태로 돌아올 수 있어요.

정식으로는 한 큐비트 상태 벡터에 아래 행렬을 곱하는 유니터리 변환으로 정의해요.
행렬은 나중에 계산 규칙을 압축해서 쓰는 표기라고 생각하면 돼요.

H = 1/√2 · [[1, 1], [1, -1]]

이 행렬은 |0⟩와 |1⟩를 다음 두 상태로 보내요.

H|0⟩ = (|0⟩ + |1⟩) / √2
H|1⟩ = (|0⟩ - |1⟩) / √2

첫 번째 식만 보면 H 게이트가 단순히 “0을 반반 확률로 만든다”처럼 보일 수 있어요.
하지만 핵심은 확률이 아니라 진폭이에요. H 게이트는 입력 진폭을 두 출력 기저 상태로 나눠 보내고, 같은 출력 기저 상태로 도착한 진폭 기여를 더해요.

입력 기저 상태H 게이트 적용 후 상태바로 측정하면
|0⟩(|0⟩ + |1⟩) / √20과 1이 각각 50%
|1⟩(|0⟩ - |1⟩) / √20과 1이 각각 50%

H 게이트의 정확한 진폭 변환 규칙

적용 전 상태를 |ψ⟩ = a₀|0⟩ + a₁|1⟩라고 해볼게요.
H 게이트는 a₀만 따로 보거나 a₁만 따로 보지 않아요.
두 입력 진폭이 모두 적용 후의 |0⟩ 성분과 |1⟩ 성분에 기여하고, 같은 성분으로 도착한 기여가 더해져요.

H(a₀|0⟩ + a₁|1⟩) = ((a₀ + a₁)|0⟩ + (a₀ - a₁)|1⟩) / √2
출발하는 입력 진폭|0⟩으로 가는 기여|1⟩으로 가는 기여
적용 전 |0⟩의 진폭 a₀+ a₀ × 1/√2+ a₀ × 1/√2
적용 전 |1⟩의 진폭 a₁+ a₁ × 1/√2- a₁ × 1/√2
마이너스 부호가 붙는 위치 네 가지 입력-출력 조합 중 적용 전 |1⟩의 진폭이 적용 후 |1⟩ 성분으로 가는 기여에만 마이너스 부호가 붙어요.
갑자기 “1→1 항”이라고 말하면 모호하므로, 이 책에서는 “적용 전 |1⟩에서 와서 적용 후 |1⟩로 가는 진폭 기여”라고 풀어서 부를게요.

|0⟩ 상태에 H 게이트를 적용해보자

|0⟩ 상태는 a₀ = 1, a₁ = 0인 상태예요.
이 값을 H 게이트 변환 규칙에 넣으면 적용 후 |0⟩ 성분과 |1⟩ 성분이 둘 다 1/√2가 돼요.

적용 후 |0⟩의 진폭 = 1 × 1/√2 + 0 × 1/√2 = 1/√2
적용 후 |1⟩의 진폭 = 1 × 1/√2 + 0 × (-1/√2) = 1/√2

그래서 H|0⟩은 (|0⟩ + |1⟩) / √2가 되고, 바로 측정하면 0과 1이 각각 50% 확률로 나와요.
여기까지가 흔히 “H 게이트가 중첩을 만든다”라고 말하는 부분이에요.

왜 마이너스 부호가 필요할까?

9장에서 양자 게이트는 되돌릴 수 있어야 한다고 했어요.
이 조건을 H 게이트에 적용하면, |0⟩ 입력과 |1⟩ 입력이 같은 출력 상태로 합쳐지면 안 돼요.
서로 다른 두 입력 상태는 H 게이트를 지난 뒤에도 서로 구별 가능한 두 출력 상태로 남아야 해요.

이제 위에서 본 H 게이트 변환 규칙을 다시 봐요.
네 가지 진폭 기여 중 적용 전 |1⟩에서 와서 적용 후 |1⟩로 가는 진폭 기여에만 마이너스 부호가 붙어요.
짧게 말하면 1에서 와서 1로 가는 기여, 또는 1 → 1 기여예요.

여기서 1 → 1의 뜻 1 → 1은 “적용 전에도 |1⟩ 방향의 진폭이고, 적용 후에도 |1⟩ 방향의 진폭으로 기여한다”는 뜻이에요.
숫자 1을 숫자 1로 바꾼다는 뜻이 아니라, 어느 방향에서 온 진폭이 어느 방향으로 가는지를 표시한 말이에요.
마이너스 부호가 없다면

두 입력이 같은 출력으로 합쳐져요

H|0⟩ = (|0⟩ + |1⟩) / √2
H|1⟩ = (|0⟩ + |1⟩) / √2

출력이 둘 다 (|0⟩ + |1⟩) / √2예요.
그러면 출력 상태만 보고 원래 입력이 |0⟩였는지 |1⟩였는지 되살릴 수 없어요.

마이너스 부호가 있으면

두 입력이 서로 구별 가능한 출력으로 남아요

H|0⟩ = (|0⟩ + |1⟩) / √2
H|1⟩ = (|0⟩ - |1⟩) / √2

두 출력 상태는 같은 상태가 아니에요.
하나는 |0⟩ 방향과 |1⟩ 방향이 같은 부호이고, 다른 하나는 |1⟩ 방향만 반대 부호예요.

두 출력 상태가 정말로 서로 구별 가능한지 확인하려면, 두 상태가 얼마나 겹치는지 계산해볼 수 있어요.
이 겹침 계산을 정식으로는 내적이라고 불러요.
지금 예제에서는 진폭이 모두 실수이므로, 같은 방향의 진폭끼리 곱한 뒤 더한다고 생각하면 돼요.

겹침 = (1/√2 × 1/√2) + (1/√2 × -1/√2)
= 1/2 - 1/2 = 0

겹침이 0인 상태를 정식으로 직교하는 상태라고 불러요.
그래서 H 게이트의 마이너스 부호는 단순히 모양을 맞추는 기호가 아니라, |0⟩ 입력의 출력과 |1⟩ 입력의 출력을 직교하게 만들기 위해 필요한 부호예요.

여기서 핵심 H 게이트가 되돌릴 수 있는 양자 게이트가 되려면 서로 다른 입력 상태가 같은 출력 상태로 합쳐지면 안 돼요.
현재 표준 H 게이트 표기에서는 그 조건을 만족시키는 마이너스 부호가 적용 전 |1⟩에서 와서 적용 후 |1⟩로 가는 진폭 기여에 놓여요.

H 게이트를 두 번 적용하면 왜 다시 돌아올까?

|0⟩ 상태에서 시작하면 첫 번째 H 게이트 뒤의 상태는 (|0⟩ + |1⟩) / √2예요.
따라서 두 번째 H 게이트를 적용하기 직전에는 a₀ = 1/√2, a₁ = 1/√2예요.
이 값을 같은 변환 규칙에 다시 넣으면 끝이에요.

두 번째 H 뒤 |0⟩의 진폭 = (1/√2) × 1/√2 + (1/√2) × 1/√2 = 1/2 + 1/2 = 1
두 번째 H 뒤 |1⟩의 진폭 = (1/√2) × 1/√2 + (1/√2) × (-1/√2) = 1/2 - 1/2 = 0
H 게이트(H 게이트(|0⟩)) = |0⟩

이 계산에서 |0⟩ 성분으로 가는 두 진폭 기여는 보강되고, |1⟩ 성분으로 가는 두 진폭 기여는 상쇄돼요.
그래서 H 게이트를 두 번 적용하면 “랜덤이 두 번 일어나서 다시 0이 된다”가 아니라, 진폭의 보강과 상쇄 때문에 정확히 처음 상태로 돌아와요.

operation Main() {
    // 큐비트 하나 준비
    // 준비 직후 q[0]은 |0⟩ 상태
    use q = Qubit[1];

    // 첫 번째 H 게이트 적용
    // 진폭이 측정 결과값 0과 1로 나뉨
    H(q[0]);

    // 두 번째 H 게이트 적용
    // 보강과 상쇄가 일어나 다시 |0⟩ 상태가 됨
    H(q[0]);

    // 이상적인 실행 환경: 측정 결과값 0
    bit result = M(q[0]);
}
