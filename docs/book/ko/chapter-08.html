8장. 얽힘은 무엇이 함께 묶인다는 뜻일까?

얽힘은 양자컴퓨팅에서 가장 유명하지만, 처음 배울 때 가장 쉽게 오해되는 개념이기도 해요.
얽힘을 “두 큐비트가 신비하게 서로 신호를 주고받는 일”처럼 이해하면 곤란해요.
정석적으로는 두 큐비트 전체 상태가 두 개의 한 큐비트 상태로 분해되지 않는 경우를 얽힘 상태라고 해요.

어렵게 들리지만 핵심은 하나예요 어떤 두 큐비트 상태는 “첫 번째 큐비트는 이런 상태, 두 번째 큐비트는 이런 상태”라고 따로 설명할 수 있어요.
하지만 어떤 상태는 두 큐비트 전체를 하나의 상태 벡터로 봐야만 정확히 설명돼요.
바로 그때 얽힘이 등장해요.

분리 가능한 상태부터 보자

예를 들어 첫 번째 큐비트가 α|0⟩ + β|1⟩ 상태이고, 두 번째 큐비트가 γ|0⟩ + δ|1⟩ 상태라고 따로 말할 수 있다면, 두 큐비트 전체 상태를 이렇게 쓸 수 있어요.
(α|0⟩ + β|1⟩) ⊗ (γ|0⟩ + δ|1⟩)

여기서 ⊗는 텐서곱이라고 읽어요.
텐서곱은 두 상태를 합쳐서 더 큰 상태 공간의 상태로 만든다는 뜻이에요.
지금은 기호를 외우기보다, “각 큐비트를 따로 설명한 뒤 합친 상태”라고 읽으면 돼요.
이런 상태는 분리 가능한 상태라고 불러요.

이 식을 두 큐비트의 네 가지 측정 결과값 기준으로 펼치면 아래처럼 돼요.
첫 번째 큐비트에서 온 진폭과 두 번째 큐비트에서 온 진폭이 서로 곱해져서, 두 자리 측정 결과값의 진폭을 만들어요.

(α|0⟩ + β|1⟩) ⊗ (γ|0⟩ + δ|1⟩)
= αγ|00⟩ + αδ|01⟩ + βγ|10⟩ + βδ|11⟩
측정 결과값첫 번째 큐비트에서 온 진폭두 번째 큐비트에서 온 진폭전체 진폭
00αγαγ
01αδαδ
10βγβγ
11βδβδ
분리 가능한 상태의 중요한 특징 네 진폭을 마음대로 따로 고르는 것이 아니에요.
분리 가능한 상태에서는 네 진폭이 αγ, αδ, βγ, βδ처럼 두 큐비트의 진폭 곱으로 묶여 나와요.
중첩과 얽힘은 같은 말이 아니에요 위 상태는 여러 기저 상태 항이 함께 있으면 중첩 상태일 수 있어요.
하지만 전체 상태를 (α|0⟩ + β|1⟩) ⊗ (γ|0⟩ + δ|1⟩)처럼 두 큐비트의 상태로 나누어 쓸 수 있으므로, 이 상태는 얽힘 상태가 아니에요.

이번에는 앞의 곱 형태로 만들 수 없는 상태를 보자

앞에서 분리 가능한 상태는 네 진폭이 αγ, αδ, βγ, βδ처럼 두 큐비트의 진폭 곱으로 나온다고 했어요.
이제 그 곱 형태로는 만들 수 없는 대표적인 두 큐비트 상태를 봐요.

(|00⟩ + |11⟩) / √2

이 상태에는 |00⟩ 성분과 |11⟩ 성분만 있어요.
측정하면 00 또는 11이 각각 50% 확률로 나오고, 01과 10은 나오지 않아요.
그래서 두 큐비트를 따로따로 무작위로 본다기보다, 두 자리 결과가 함께 묶여 있다고 읽어야 해요.

측정 결과값진폭측정 확률
001/√250%
0100%
1000%
111/√250%
왜 이 상태는 앞의 곱 형태로 만들 수 없을까? 분리 가능한 상태라면 00의 진폭은 αγ이고, 11의 진폭은 βδ예요.
그런데 위 상태에서는 00과 11의 진폭이 둘 다 0이 아니므로, α, β, γ, δ도 모두 0이 아니어야 해요.
그러면 01의 진폭 αδ와 10의 진폭 βγ도 0이 아니게 돼요.
하지만 위 상태에서는 01과 10의 진폭이 정확히 0이에요. 그래서 이 상태는 두 큐비트의 상태로 나누어 쓸 수 없고, 얽힘 상태예요.
분리 가능한 상태

각 큐비트를 따로 설명할 수 있어요

첫 번째 큐비트의 상태와 두 번째 큐비트의 상태를 각각 적은 뒤, 두 상태를 합쳐서 전체 상태를 만들 수 있어요.

얽힘 상태

전체 상태를 먼저 봐야 해요

각 큐비트만 따로 떼어서는 전체 상태의 측정 결과 관계를 완전히 설명할 수 없어요.

얽힘은 단순한 중첩 두 개가 아니에요 한 큐비트가 중첩 상태에 있다는 말과 두 큐비트가 얽혀 있다는 말은 달라요.
중첩은 하나의 상태 벡터 안에 여러 기저 방향의 진폭이 함께 있는 상황이고, 얽힘은 여러 큐비트의 전체 상태가 각 큐비트의 상태로 분리되지 않는 상황이에요.

CNOT 게이트는 나중에 자세히 볼 거예요

얽힘 상태는 보통 한 큐비트에 H 게이트를 적용한 뒤, CNOT 같은 두 큐비트 게이트를 적용해서 만들 수 있어요.
여기서는 CNOT의 세부 규칙을 아직 계산하지 않고, 얽힘이 무엇을 뜻하는지 먼저 잡고 갈게요.
CNOT 게이트는 13장에서 다시 차근차근 볼 거예요.

Qora로 미리보기

얽힘을 만드는 회로 모양만 먼저 보자

operation Main() {
    // 큐비트 두 개 준비
    // 시작 상태는 |00⟩
    use q = Qubit[2];

    // 첫 번째 큐비트에 H 게이트 적용
    // 첫 번째 큐비트 쪽에 중첩이 생김
    H(q[0]);

    // 두 큐비트를 CNOT으로 연결
    // 자세한 규칙은 CNOT 장에서 다시 봄
    CNOT(q[0], q[1]);

    // 이상적인 실행 환경: 00 또는 11이 함께 나옴
    var first: bit = M(q[0]);
    var second: bit = M(q[1]);
}
