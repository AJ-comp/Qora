1큐비트 상태는 가능한 측정 결과값마다 진폭을 가진다

큐비트 하나를 측정하면 측정 결과값은 0 또는 1 중 하나입니다. 측정하기 전의 큐비트 상태에는 가능한 측정 결과값 0과 가능한 측정 결과값 1에 각각 진폭이 붙어 있습니다. 이 문서에서는 각 측정 결과값의 진폭을 화살표로 그립니다. 화살표의 길이는 진폭의 크기이고, 화살표의 방향(오른쪽/왼쪽)은 진폭의 부호(+/−)입니다.

가장 단순한 큐비트 상태인 |0⟩ 상태부터 봅시다. |0⟩ 상태에서는 측정 결과값 0의 진폭은 1이고, 측정 결과값 1의 진폭은 0입니다.

가능한 측정 결과값진폭 화살표진폭 값
0───▶1 (꽉 참)
1·0 (없음)

측정 결과값 0의 진폭이 1이고 측정 결과값 1의 진폭이 0인 큐비트 상태를 |0⟩이라고 부릅니다.

각 측정 결과값에 붙은 진폭의 길이를 제곱한 값 = 해당 측정 결과값이 나올 확률

그래서 |0⟩ 상태를 측정하면 측정 결과값 0이 나올 확률은 1² = 100%이고, 측정 결과값 1이 나올 확률은 0² = 0%입니다. 즉 측정 결과값은 반드시 0입니다. 진폭의 방향과 부호는 2번 절부터 사용하므로, 1번 절에서는 진폭의 길이만 보면 됩니다.

2H 게이트를 한 번 적용하면 한 큐비트가 두 측정 결과값의 진폭을 가진다

H 게이트는 측정 결과값 0에 모여 있던 진폭을 그대로 측정 결과값 0에만 두지 않습니다. H 게이트는 측정 결과값 0에 모여 있던 진폭을 측정 결과값 0으로 가는 진폭 기여와 측정 결과값 1로 가는 진폭 기여로 나눠 보냅니다. |0⟩ 상태에 H 게이트를 적용하면 측정 결과값 0에 있던 진폭 1이 측정 결과값 0의 진폭 1/√2와 측정 결과값 1의 진폭 1/√2로 바뀝니다.

|01⟩ 상태가 되는 것은 아닙니다 |0⟩ 상태에 H 게이트를 적용한 결과는 한 개의 큐비트가 측정 결과값 0과 측정 결과값 1에 대한 진폭을 동시에 가진 상태입니다. 수식으로 쓰면 (|0⟩ + |1⟩) / √2입니다. |01⟩은 두 개의 큐비트가 있을 때 쓰는 상태 표기입니다.

H 게이트를 적용한 뒤에는 측정 결과값 0의 진폭 화살표가 짧아집니다. 진폭의 길이가 1에서 1/√2(약 0.707)로 줄어듭니다. 동시에 H 게이트를 적용하기 전에는 0이었던 측정 결과값 1의 진폭도 1/√2로 새로 생깁니다. 따라서 H 게이트를 적용한 뒤에 큐비트를 측정하면 측정 결과값 0이 나올 확률도 50%, 측정 결과값 1이 나올 확률도 50%입니다.

3왜 진폭이 1/√2인가? — 진폭을 제곱해야 확률이 된다

측정 결과값 0과 측정 결과값 1이 각각 50% 확률로 나온다면 진폭도 각각 1/2일 것처럼 보일 수 있습니다. 하지만 H 게이트를 적용한 뒤의 진폭은 1/2이 아니라 1/√2입니다. 이유는 확률 = 진폭²이라는 규칙과, 모든 측정 결과값의 확률을 더하면 1이라는 규칙이 동시에 성립해야 하기 때문입니다.

만약 측정 결과값 0의 진폭과 측정 결과값 1의 진폭을 모두 1로 두면:

1² + 1² = 2 = 200%   ❌ (확률이 200%일 순 없다)

그래서 측정 결과값 0의 진폭과 측정 결과값 1의 진폭을 모두 √2로 나눠서 전체 확률이 100%가 되게 만듭니다:

(1/√2)² + (1/√2)² = ½ + ½ = 1 = 100%   ✓
진폭 ↔ 확률 (같은 규칙, 두 방향) 확률 = 진폭² 이면, 거꾸로 진폭 = √확률 입니다. (넓이 = 변² ↔ 변 = √넓이 와 같은 한 식.) 그래서 양자 상태에는 "확률의 제곱근"인 진폭을 적고, 큐비트를 측정할 때는 진폭을 제곱해서 확률을 계산합니다.

그림으로 — 진폭은 정사각형의 "변", 확률은 정사각형의 "넓이"

넓이(확률)를 반(100% → 50%)으로 줄이면, 변(진폭)은 절반(0.5)이 아니라 1/√2(≈0.707)로 줄어듭니다. 정사각형의 넓이는 변의 제곱이기 때문입니다. 그래서 H 게이트를 적용한 뒤의 진폭은 1/2이 아니라 1/√2입니다.

이건 비유가 아니라 실제 물리 법칙이에요 파동(빛·소리)에서 세기(빛의 밝기·소리의 세기)는 진폭의 제곱에 비례합니다.
그래서 빛의 밝기를 절반으로 줄이려면 진폭을 0.5배가 아니라 0.707배(1/√2)로 낮춰야 합니다. 1/√2라는 숫자는 H 게이트에서 나타나는 1/√2와 같은 숫자입니다.

양자역학의 확률도 같은 제곱 규칙을 씁니다 — |진폭|² = 확률, 이 제곱 규칙이 보른 규칙(Born rule)이에요.
그래서 양자에서도 "진폭(amplitude)"이라는 말을 씁니다.

(제곱되는 건 진폭이에요. 주파수는 다른 축이라 — 빛의 색, 소리의 음높이 — 확률로 들어가지 않습니다.)

H 게이트가 진폭을 바꾸는 방식은 먼저 큰 규칙으로 보고, 이어서 네 단계로 쪼개서 계산해 볼 수 있습니다.

① 각 입력 진폭을 측정 결과값 0과 측정 결과값 1 쪽의 진폭 기여로 나눠 보낸다  →  ② 각 진폭 기여에 부호와 1/√2를 곱한 뒤, 같은 측정 결과값끼리 더한다

|0⟩ 상태, 즉 측정 결과값 0의 진폭이 1이고 측정 결과값 1의 진폭이 0인 상태에 H 게이트를 적용하면 어떻게 되는지 봅시다.

5왜 측정 결과값 1에서 와서 측정 결과값 1로 가는 항에 마이너스 부호가 붙는가

1단계

양자 게이트는 입력 정보를 잃으면 안 됩니다

H 게이트도 양자 게이트이므로 반드시 되돌릴 수 있어야 합니다. 되돌릴 수 있다는 말은 서로 다른 입력 상태가 같은 출력 상태로 합쳐지면 안 된다는 뜻입니다.

2단계

마이너스 부호가 없으면 두 입력 상태가 같은 출력 상태로 합쳐집니다

H 게이트에서 마이너스 부호를 빼 버리면, |0⟩ 상태와 |1⟩ 상태가 같은 출력 상태로 갑니다.

마이너스 부호가 없는 경우

입력 |0⟩ (|0⟩ + |1⟩) / √2
입력 |1⟩ (|0⟩ + |1⟩) / √2

두 입력 상태가 같은 출력 상태가 됩니다. 출력만 보고 원래 입력이 무엇이었는지 알 수 없습니다.

마이너스 부호가 있는 경우

입력 |0⟩ (|0⟩ + |1⟩) / √2 = |+⟩
입력 |1⟩ (|0⟩ − |1⟩) / √2 = |−⟩

두 출력 상태가 서로 직교합니다. 서로 완전히 구별 가능한 두 상태로 남습니다.

3단계

마이너스 부호는 출력 상태를 구별 가능하게 만들기 위해 필요합니다

H 게이트를 |0⟩ 상태에 적용한 결과와 H 게이트를 |1⟩ 상태에 적용한 결과는 단지 조금 다른 정도가 아니라 완전히 구별 가능한 두 방향(직교)이어야 합니다.

두 출력 상태의 직교성을 만드는 것이 측정 결과값 1에서 와서 측정 결과값 1로 가는 항의 마이너스 부호입니다.

4단계

현재 표기에서는 마이너스 부호가 측정 결과값 1에서 와서 측정 결과값 1로 가는 항에 놓입니다

H 게이트를 |0⟩ 상태에 적용한 결과를 (|0⟩+|1⟩)/√2로 쓰고, 측정 결과값 0으로 가는 진폭 기여는 양수로 두겠다는 표기 약속을 잡습니다.

이 약속 아래에서 H 게이트를 |1⟩ 상태에 적용한 결과가 H 게이트(|0⟩)와 직교하려면, 측정 결과값 1 쪽 항이 반대 부호여야 합니다.

핵심 마이너스 부호는 임의로 넣은 기호가 아니라, H 게이트가 되돌릴 수 있으려면 필요한 부호입니다. 현재 표기 약속에서는 그 부호가 측정 결과값 1에서 와서 측정 결과값 1로 가는 항에 놓입니다. 이 부호 덕분에 H 게이트를 두 번 적용하면 처음 상태로 돌아옵니다.

6H 게이트를 두 번 적용하면 간섭으로 |0⟩ 상태가 되살아난다

|0⟩ 상태에 H 게이트를 연속으로 두 번 적용하면 큐비트 상태가 다시 |0⟩ 상태로 되돌아옵니다. 고전적인 50/50 동전 던지기라면 "무작위화 과정을 두 번 거쳐도 여전히 50/50"이라고 생각할 수 있습니다. 하지만 H 게이트는 동전 던지기가 아니라 진폭을 더하는 양자 게이트입니다. 왜 다시 |0⟩ 상태가 되는지 진폭을 직접 더해 봅시다.

되돌아감의 핵심 4번 절의 H 게이트 진폭 변환식을 첫 번째 H 게이트 적용 후 상태에 그대로 한 번 더 적용하면 됩니다. 대입하면 측정 결과값 0의 진폭은 보강되어 +1이 되고, 측정 결과값 1의 진폭은 상쇄되어 0이 됩니다.
1단계

첫 번째 H 게이트 적용 후의 진폭을 두 번째 H 게이트의 입력값으로 놓습니다

4번 절에서 H 게이트 변환식의 입력 진폭 이름을 a₀와 a₁로 붙였습니다. 두 번째 H 게이트를 적용할 때는 첫 번째 H 게이트가 만든 진폭이 그대로 a₀와 a₁입니다.

기호두 번째 H 게이트 적용 전의 의미값
a₀ 첫 번째 H 게이트 적용 후 측정 결과값 0의 진폭 1/√2
a₁ 첫 번째 H 게이트 적용 후 측정 결과값 1의 진폭 1/√2
2단계

4번 절의 H 게이트 진폭 변환식을 그대로 한 번 더 사용합니다

새 규칙을 추가로 외우는 것이 아닙니다. 4번 절에서 만든 일반식에 a₀ = 1/√2와 a₁ = 1/√2를 대입하면 됩니다.

두 번째 H 게이트 적용 후의 진폭4번 절의 일반식대입식
측정 결과값 0의 진폭 a₀ × 1/√2 + a₁ × 1/√2 (1/√2) × 1/√2 + (1/√2) × 1/√2
측정 결과값 1의 진폭 a₀ × 1/√2 + a₁ × (−1/√2) (1/√2) × 1/√2 + (1/√2) × (−1/√2)
3단계

대입식의 값을 계산하면 되돌아감이 바로 보입니다

(1/√2) × (1/√2) = 1/2이므로, 두 번째 H 게이트 적용 후의 두 진폭은 다음처럼 계산됩니다.

측정 결과값 0으로 도착한 진폭 기여
+1/2 + +1/2 = +1
같은 부호라서 보강 간섭이 일어납니다.

측정 결과값 0의 진폭이 살아남습니다. 측정 확률은 100%입니다.

측정 결과값 1로 도착한 진폭 기여
+1/2 + −1/2 = 0
반대 부호라서 상쇄 간섭이 일어납니다.

측정 결과값 1의 진폭은 사라집니다. 측정 확률은 0%입니다.

H 게이트(H 게이트(|0⟩)) = |0⟩ 두 번째 H 게이트 적용 결과 측정 결과값 0의 진폭 = +1 (측정 확률 100%) 측정 결과값 1의 진폭 = 0 (측정 확률 0%) 따라서 다시 |0⟩ 상태
왜 두 번째 H 게이트 적용 후 측정 결과가 50/50이 아니라 0으로 확정되는가

측정 결과값 1에 도착하는 진폭 기여는 두 개입니다. |0⟩ 성분에서 온 진폭 기여는 +1/2이고, |1⟩ 성분에서 온 진폭 기여는 −1/2입니다.

측정 결과값 1에 도착하는 두 진폭 기여는 부호가 반대라서 더하면 0이 됩니다. 측정 결과값 1의 진폭이 0이 되는 현상이 상쇄 간섭입니다.

동시에 측정 결과값 0에 도착하는 두 진폭 기여는 +1/2와 +1/2이므로 더하면 +1이 됩니다.

측정 결과값 1 쪽의 −1/2는 정확히 4번 절의 "측정 결과값 1에서 와서 측정 결과값 1로 가는 항에는 −가 붙는다"는 규칙에서 나온 것입니다.

만약 마이너스가 없었다면?

측정 결과값 1의 진폭은 +1/2 + 1/2 = +1이 되어 상쇄되지 않았을 것입니다.

동시에 측정 결과값 0의 진폭도 +1이 되어, 두 측정 결과값의 확률을 더하면 1을 넘는 유효하지 않은 변환이 됩니다.

측정 결과값 1에서 와서 측정 결과값 1로 가는 항의 마이너스 부호가 측정 결과값 1의 진폭을 상쇄시키고 전체 확률이 1로 유지되게 합니다.

Qora로 실습해보기

H 게이트를 두 번 적용하는 회로를 Qora로 써 봅니다

아래 Qora 코드는 큐비트 하나를 준비한 뒤, 같은 큐비트에 H 게이트를 두 번 적용하고 측정합니다. 이 문서에서 계산한 H 게이트(H 게이트(|0⟩)) = |0⟩ 회로입니다.

Qora 코드
operation Main() {
    // 큐비트 하나를 준비합니다.
    // 준비 직후 q[0]은 |0⟩ 상태입니다.
    // 측정 결과값 0의 진폭 = 1, 측정 결과값 1의 진폭 = 0입니다.
    use q = Qubit[1];

    // 첫 번째 H 게이트를 q[0]에 적용합니다.
    // |0⟩ 상태가 (|0⟩ + |1⟩) / √2 상태로 바뀝니다.
    // 측정 결과값 0의 진폭 = 1/√2입니다.
    // 측정 결과값 1의 진폭 = 1/√2입니다.
    H(q[0]);

    // 두 번째 H 게이트를 같은 q[0]에 다시 적용합니다.
    // 측정 결과값 0의 진폭 = +1/2 + +1/2 = +1입니다.
    // 측정 결과값 1의 진폭 = +1/2 + -1/2 = 0입니다.
    H(q[0]);

    // 두 번째 H 게이트 적용 후의 q[0]을 측정합니다.
    // 측정 결과값 0이 나올 확률 = 100%입니다.
    // 측정 결과값 1이 나올 확률 = 0%입니다.
    bit result = M(q[0]);
}

확인할 점은 H(q[0]);가 두 번 연속으로 나온다는 것입니다. 이상적인 실행 환경에서는 두 번째 H 게이트 적용 후 측정 결과값 0이 100% 나옵니다. Qora 코드의 // 주석은 실행에 영향을 주지 않습니다.

7진폭과 간섭은 실제 물리인가, 아니면 표기 약속인가?

이 문서의 화살표 계산은 단순히 "이렇게 정의하자"라고 정한 표기 놀이가 아닙니다. 자연에서 실제로 관측되는 진폭의 보강과 상쇄를 수식으로 옮긴 것입니다. 진폭과 간섭은 임의로 발명한 장치가 아니라, 실험 결과를 설명하기 위해 필요한 물리 개념입니다.

표기 약속

사람이 정한 부분

"H 게이트"라는 이름, 진폭의 양수와 음수를 오른쪽·왼쪽 화살표로 그리는 방식, |0⟩와 |1⟩ 같은 표기는 사람이 정한 약속입니다.

실제 물리

실험으로 확인되는 부분

진폭 기여가 서로 보강되거나 상쇄되는 현상, H 게이트를 두 번 적용하면 |0⟩ 상태가 다시 |0⟩ 상태로 돌아오는 현상은 표기 약속이 아니라 실험 결과를 설명하는 물리 내용입니다.

어떤 현상으로 확인되는가?

증거 1

이중슬릿 간섭무늬

전자·광자를 슬릿 두 개에 하나씩 보내도 화면에는 밝고 어두운 띠가 쌓입니다.

어두운 띠가 생기는 위치는 화면의 해당 위치에 도착한 진폭 기여들이 서로 상쇄된 지점입니다.

보강 간섭 → 밝은 띠
상쇄 간섭 → 어두운 띠
증거 2

H 게이트 두 번 적용 회로

이상적인 양자 회로나 양자 시뮬레이터에서는 |0⟩ 상태에 H 게이트를 두 번 적용하고 측정하면 측정 결과값 0이 100% 나옵니다.

실제 양자컴퓨터 장비에서도 잡음 때문에 결과가 조금 흐려질 뿐, 측정 결과값 0이 측정 결과값 1보다 압도적으로 많이 나옵니다.

|0⟩ → H 게이트 → (|0⟩ + |1⟩) / √2 → H 게이트 → |0⟩
왜 "진짜"라고 말할 수 있나

큐비트가 단순한 50/50 동전이라면 H 게이트를 두 번 적용하는 과정은 "무작위화 과정을 두 번 거치는 것"과 같아야 하고, 최종 측정 확률도 여전히 50/50이어야 합니다.

그런데 이상적인 H 게이트 두 번 적용 회로는 다시 |0⟩ 상태로 돌아옵니다. 이 결과는 동전에는 없는 "부호 달린 진폭 기여들이 서로 상쇄되는 일"을 계산에 넣어야 설명됩니다.

그래서 진폭의 부호와 간섭은 단순한 말장난이 아니라, 실험을 설명하는 데 필요한 물리량입니다.

더 깊이 "왜 자연이 이런 양자 규칙을 따르는가"까지 묻기 시작하면 현재 이론의 기본 공준까지 내려가지만, H 게이트의 진폭 변환 규칙이 실험 결과를 설명하고 예측한다는 사실은 반복적으로 확인됩니다.

8한 장 요약

상태

큐비트 상태는 확률 자체가 아니라 진폭으로 표현합니다

한 큐비트 상태는 가능한 측정 결과값 0과 1 각각에 붙은 진폭으로 표현합니다.

측정 확률 = 진폭 길이²
정규화

확률의 합은 반드시 1이어야 합니다

두 측정 결과값의 확률이 50%와 50%라면, 각 측정 결과값의 진폭은 1/2이 아니라 1/√2입니다.

(1/√2)² + (1/√2)² = 1
H 게이트

H 게이트는 입력 진폭을 두 측정 결과값으로 나눠 보냅니다

각 입력 진폭은 측정 결과값 0과 측정 결과값 1 쪽의 진폭 기여가 됩니다. 같은 측정 결과값에 도착한 진폭 기여끼리는 서로 더합니다.

측정 결과값 1에서 와서 측정 결과값 1로 가는 항만 −입니다.
H 게이트 두 번

같은 변환식을 한 번 더 적용하면 원래 상태로 돌아옵니다

첫 번째 H 게이트 뒤에는 a₀ = 1/√2, a₁ = 1/√2입니다. 이 값을 H 게이트 변환식에 다시 넣으면 됩니다.

측정 결과값 0 쪽: +1/2 + +1/2 = +1
측정 결과값 1 쪽: +1/2 + −1/2 = 0
실제 물리

진폭과 간섭은 계산 장식이 아니라 실험 결과를 설명하는 물리 개념입니다

이중슬릿의 밝고 어두운 띠, H 게이트를 두 번 적용하면 다시 |0⟩ 상태로 돌아오는 회로 결과는 진폭의 보강 간섭과 상쇄 간섭을 넣어야 설명됩니다.

양자 알고리즘의 핵심 아이디어 = 진폭의 상쇄 간섭과 보강 간섭을 설계하는 것 —
원하지 않는 측정 결과값의 진폭은 상쇄되게 만들고, 원하는 측정 결과값의 진폭은 보강되게 만드는 과정.
