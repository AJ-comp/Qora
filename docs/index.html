H 게이트를 화살표로 이해하기

큐비트·진폭·간섭을 한 단계씩 — 누가 봐도 따라올 수 있게

이 글은 양자 게이트 중 가장 기본인 H(하다마드) 게이트를, 수식 덩어리가 아니라 "화살표" 하나로 끝까지 따라가며 설명합니다. 특히 "H를 두 번 걸면 왜 다시 처음으로 돌아오는가"라는 신기한 일(간섭)을, 한 단계도 건너뛰지 않고 직접 손으로 더해 봅니다.

  2. H를 한 번 — 0을 반반으로
  3. 왜 1/√2? — 진폭과 확률

  5. 왜 마이너스가 붙는가 — 되돌릴 수 있어야 한다

1큐비트는 "화살표"다

큐비트 하나는 나올 수 있는 답이 0 아니면 1, 둘뿐입니다. 각 답마다 화살표가 하나씩 앉는 자리("칸")가 있다고 보세요. 화살표의 길이가 그 답의 진폭이고, 방향(오른쪽/왼쪽)은 부호(+/−)입니다.

가장 단순한 상태부터 봅시다 — 0칸에만 꽉 찬 화살표가 있고, 1칸은 비어 있는 상태:

칸(답)화살표진폭
0칸───▶1 (꽉 참)
1칸·0 (없음)

이 상태를 |0⟩이라고 부릅니다.

화살표 길이를 제곱한 값 = 그 답이 측정될 확률

그래서 위 |0⟩을 재면 — 0칸은 1² = 100%, 1칸은 0² = 0%, 즉 무조건 0이 나옵니다. (방향/부호는 2번 절부터 본격적으로 쓰니, 지금은 길이만 봐도 됩니다.)

2H를 한 번 — 0을 반반으로

H는 0을 0으로 두는 게 아니라, 받은 큐비트를 50/50으로 펴는 게이트입니다. |0⟩에 H를 걸면 0칸에 있던 화살표 하나가 0과 1로 절반씩 갈라집니다.

칸H 전H 후H 후 확률
0칸───▶ 1──▶ 1/√2 ≈ 0.70750%
1칸· 0──▶ 1/√2 ≈ 0.70750%

H 후에 0칸 화살표가 더 짧아진 게 보이죠 — 길이가 1 → 0.707로 줄었습니다. 대신 비어 있던 1칸에 같은 길이의 화살표가 새로 생겼고요. 그래서 측정하면 0도 50%, 1도 50%가 됩니다.

3왜 1/√2? — 진폭과 확률

"반반이면 진폭이 1/2 아닌가?" 싶지만, 진폭은 1/2이 아니라 1/√2입니다. 이유는 확률 = 진폭²이라는 규칙과, 확률을 다 더하면 1이라는 규칙 두 개가 함께 강제하기 때문입니다.

만약 그냥 진폭을 1, 1로 두면:

1² + 1² = 2 = 200%   ❌ (확률이 200%일 순 없다)

그래서 둘 다 √2로 나눠 줄입니다:

(1/√2)² + (1/√2)² = ½ + ½ = 1 = 100%   ✓
진폭 ↔ 확률 (같은 규칙, 두 방향) 확률 = 진폭² 이면, 거꾸로 진폭 = √확률 입니다. (넓이 = 변² ↔ 변 = √넓이 와 같은 한 식.) 그래서 상태에는 "확률의 제곱근"인 진폭을 적고, 측정할 때 제곱해서 확률을 되찾습니다.

4H의 정확한 규칙

H가 하는 일은 딱 두 단계입니다.

① 입력 진폭을 0과 1 양쪽에 복사  →  ② 각각에 1/√2를 곱한다

예를 들어 진폭 1짜리 0을 펴면: 0칸·1칸에 각각 1을 복사 → 각각 ×1/√2 → 0칸 1/√2, 1칸 1/√2.

부호 규칙 — 딱 한 군데만 마이너스

그런데 0을 펼 때와 1을 펼 때 부호가 다릅니다. H를 한 번 걸 때 생기는 "길"을 세어 보면 — 입력이 0이냐 1이냐(2가지) × 그게 0칸으로 가느냐 1칸으로 가느냐(2가지) = 모두 네 가지 길(0→0, 0→1, 1→0, 1→1)이 있습니다. 그 네 길 중 "입력도 1이고 AND 출력도 1"인 길(1→1) 하나에만 마이너스가 붙습니다.

입력 → 출력부호
0 → 0+ (오른쪽)
0 → 1+ (오른쪽)
1 → 0+ (오른쪽)
1 → 1− (왼쪽) ← 유일하게 여기만

정리하면:

펴는 대상0칸1칸
0 펴기+ ▶+ ▶
1 펴기+ ▶− ◀

0칸은 언제나 +(오른쪽). 1칸만 — 0을 펴면 +, 1을 펴면 −.

5왜 마이너스가 붙는가 — 되돌릴 수 있어야 한다

"그 마이너스는 무슨 논리로 붙는가?" 답은 — 양자 게이트는 반드시 되돌릴 수 있어야 하기 때문입니다. (정보를 잃으면 안 된다 — 양자의 철칙.)

만약 마이너스가 없어서 H(0)과 H(1)이 똑같다면:

H(0) = (0 + 1)/√2
H(1) = (0 + 1)/√2   ← 둘이 완전히 같음!

그러면 누군가 (0+1)/√2 를 건넸을 때 원래 0이었는지 1이었는지 알 방법이 없습니다. 두 입력이 같은 출력으로 뭉개졌으니까요 → 되돌리기 불가능 → 양자에서 금지된 일입니다.

그래서 H(0)과 H(1)은 반드시 달라야 하고, 그 차이를 만드는 것이 바로 마이너스입니다:

H(0) = (0 + 1)/√2   (오른쪽끝, |+⟩)
H(1) = (0 − 1)/√2   (왼쪽끝, |−⟩)   → 구별됨 ✓

그런데 왜 하필 1→1 자리일까요? — 0칸은 항상 +(오른쪽)로 두기로 약속하면, 두 출력 H(0)과 H(1)을 다르게 만들 수 있는 자리는 1칸뿐입니다. 그래서 마이너스가 갈 수 있는 곳이 1→1 한 군데로 정해지는 거죠. (4번 절의 "0칸은 언제나 +"가 바로 이 약속이었습니다.)

핵심 마이너스는 끼워 맞추려고 넣은 게 아니라, "H가 되돌릴 수 있는 게이트이려면 반드시 있어야" 하는 것입니다. 그리고 "되돌릴 수 있다"는 곧 H를 두 번 하면 자기 자신을 취소한다는 뜻입니다.

6H를 두 번 — 간섭으로 0이 되살아난다

H를 두 번 걸면 |0⟩이 100%로 되돌아옵니다. 동전이라면 "랜덤 두 번 = 여전히 랜덤" 이어야 하는데, 양자는 그렇지 않습니다. 왜 그런지 손으로 더해 봅시다.

먼저 기억할 규칙: 게이트는 각 부분에 따로 적용하고, 결과를 합친다. 1차 H 후 상태는 "0-줄기 1/√2, 1-줄기 1/√2"입니다. 2차 H는 이 두 줄기에 따로 적용합니다.

1

0-줄기(1/√2)를 편다 — 0을 펴므로 둘 다 +. 1/√2 × 1/√2 = 1/2.

0칸 ← +1/2    1칸 ← +1/2
2

1-줄기(1/√2)를 편다 — 1을 펴므로 1칸만 −. 양(1/2)은 똑같다.

0칸 ← +1/2    1칸 ← −1/2   ← 마이너스!
3

칸별로 더한다

칸0-줄기에서1-줄기에서합결과
0칸 +1/2 +1/2 +1 같은 방향 → 보강
1칸 +1/2 −1/2 0 반대 방향 → 상쇄
결과: 0칸 = +1 (확률 100%), 1칸 = 0 (확률 0%)  →  다시 |0⟩!
왜 50/50이 아니라 0인가 1로 가는 길이 두 개였습니다 — 0을 거친 길(+1/2)과 1을 거친 길(−1/2). 그 둘이 부호가 반대라 만나서 사라졌습니다. 이것이 상쇄 간섭입니다. 동시에 0으로 가는 두 길은 같은 부호라 합쳐졌고요. 그 −1/2는 정확히 4번 절의 "1 → 1 = −" 한 칸에서 나온 것입니다.
만약 마이너스가 없었다면? 1칸은 +1/2 + 1/2 = +1이 되어 안 사라졌을 겁니다(여전히 50/50). 그 마이너스 하나가 상쇄를 만드는 전부입니다.

7이건 진짜인가, 그냥 표기인가?

이 화살표 계산은 "이렇게 정의하자"가 아니라, 자연이 실제로 그렇게 동작하는 것을 관측해서 수식으로 옮긴 것입니다. 발명이 아니라 발견입니다.

간섭은 눈으로 보인다 — 이중슬릿

전자·광자를 슬릿 두 개에 하나씩 쏘면 화면에 밝고 어두운 띠(간섭무늬)가 쌓입니다. 그 어두운 띠가 바로 진폭이 실제로 상쇄된 자리 — 위에서 1칸이 0이 된 것과 똑같은 일이 실제 화면에 "어둠"으로 찍히는 겁니다.

H–H도 실제 양자컴에서 매일 돌아간다

진짜 큐비트(광자 편광, 원자, 초전도 큐비트)에 H를 두 번 걸고 측정하면 실제로 0이 ~100% 나옵니다. 50/50이 아닙니다.

이게 "진짜"라는 결정적 증거 큐비트가 그냥 50/50 동전이라면 H 두 번 = 랜덤 두 번 = 여전히 50/50이어야 합니다. 그런데 실험은 100% 0이 나옵니다. 동전엔 없는 "부호 달린 진폭이 상쇄되는 일"이 실제로 일어나야만 설명됩니다. 그래서 실험이 우리에게 "진폭과 간섭은 진짜다"라고 강제로 알려줍니다.

딱 하나, "H"라는 이름이나 +/− 표기 방식엔 인간의 약속이 조금 섞여 있습니다. 하지만 그것이 가리키는 물리적 결과(두 번 하면 진짜 0으로 돌아옴)는 약속이 아니라 관측된 사실입니다. "왜 자연이 이렇게 도는가"라는 가장 깊은 질문은 아직 아무도 모르는 바닥 공준이지만, "그렇게 동작한다"는 것만큼은 의심의 여지가 없습니다.
