第3章 重ね合わせはただの「わからない状態」なのか？

重ね合わせは、量子コンピューティングを学ぶときに最も誤解されやすい言葉です。
ここでは重ね合わせを「結果がまだわからない」という意味ではなく、状態ベクトルが複数の計算基底方向の振幅を同時に持っているという意味として読んでいきます。

(|0⟩ + |1⟩) / √2

上の状態は、|0⟩ 方向の振幅が 1/√2 で、|1⟩ 方向の振幅も 1/√2 である状態です。
2つの計算基底方向がどちらも状態ベクトルの構成に関わっているので、この状態を重ね合わせ状態と呼びます。

コインを投げて手で覆っておくと、コインはすでに表か裏かに決まっていて、私たちがその結果を知らないだけです。
しかし量子の重ね合わせは、そのような古典的な無知ではありません。測定前の状態ベクトルそのものが |0⟩ 成分と |1⟩ 成分を一緒に持っています。

確率だけを見ても状態のすべてはわかりません 量子状態には、測定確率だけでなく振幅の符号と位相も含まれています。
この情報はすぐに測定すると消えてしまうように見えることもありますが、別のゲートを通るときに干渉を生み出すため、実際の計算結果に影響を与えます。

同じ50%に見えても本当に同じ状態なのか？

状態 A

測定値 1 の振幅が正である状態

(|0⟩ + |1⟩) / √2

すぐに測定すると、測定値 0 と 1 がそれぞれ 50% の確率で出ます。

状態 B

測定値 1 の振幅が負である状態

(|0⟩ - |1⟩) / √2

すぐに測定すると、この状態もやはり測定値 0 と 1 がそれぞれ 50% の確率で出ます。

2つの状態は、すぐに測定すると同じ確率を示しますが、同じ状態ではありません。
測定値 1 の振幅の符号が異なり、この符号の違いが、別のゲートをもう一度適用したときに結果の違いを生み出します。
この違いを正式に言うと、2つの状態は互いに異なる状態ベクトルであり、H ゲートのような変換に再び出会うと、互いに異なる方向へ移動します。
