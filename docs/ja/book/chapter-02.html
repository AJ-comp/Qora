第2章 振幅と確率はどうつながっているのか？

キュービットの状態は |ψ⟩ = α|0⟩ + β|1⟩ と書けるのでした。
ここで α と β は確率ではなく、確率振幅です。
測定確率は振幅そのものではなく、振幅の絶対値を2乗して得られます。

測定値 0 の確率 = |α|²、測定値 1 の確率 = |β|²

この規則は Born の規則（ボルンの規則）と呼ばれています。
振幅が実数のときは単純に2乗と考えても構いませんが、一般の量子力学では振幅が複素数になることがあるため、絶対値の2乗で計算します。

状態振幅測定確率
|0⟩α = 1, β = 0|1|² = 100%, |0|² = 0%
|1⟩α = 0, β = 1|0|² = 0%, |1|² = 100%

なぜ確率は50%なのに振幅は 1/2 ではないのか？

測定値 0 と測定値 1 が半々なら、振幅もそれぞれ 1/2 になりそうに見えますが、そうすると規格化が崩れてしまいます。
量子状態は全体の測定確率が100%にならなければならないので、|α|² + |β|² = 1 を満たす必要があります。

間違った考え方

振幅を 1/2 にするとどうなる？

各測定値の確率は25%になります。

(1/2)² + (1/2)² = 1/4 + 1/4 = 1/2

全体の確率が50%にしかならないため、キュービットの状態として成り立ちません。

正しい振幅

振幅を 1/√2 にするとなぜ正しいのか？

各測定値の確率がちょうど50%になります。

(1/√2)² + (1/√2)² = 1/2 + 1/2 = 1

全体の確率もきちんと100%になります。

Qoraで実際に試してみよう

Hゲートで50%の状態を作ってみよう

operation Main() {
    // キュービットを1つ用意
    // 用意した直後、q[0] は |0⟩ 状態
    // 測定値 0 の振幅 = 1
    // 測定値 1 の振幅 = 0
    use q = Qubit[1];

    // Hゲートを適用
    // 測定値 0 の振幅 = 1/√2
    // 測定値 1 の振幅 = 1/√2
    // 2つの測定値の確率はそれぞれ50%
    H(q[0]);

    // q[0] を測定
    // 理想的な実行環境では、測定値 0 と 1 がそれぞれ約50%
    bit result = M(q[0]);
}

ここで重要なのは、Hゲートが確率を直接半分に分けているわけではない、という点です。
ゲートがまず変えるのは状態ベクトルの振幅であり、測定するときにその振幅の絶対値の2乗が確率として現れます。
