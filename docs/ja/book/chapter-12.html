第12章. Hゲートは振幅をどう分けるのか？

Hゲートは、片方の基底状態に集まっていた振幅を2つの基底状態の方へ分けて送るゲートです。
正式には、1キュービットの状態ベクトルに下の行列を掛けるユニタリ変換として定義します。

H = 1/√2 · [[1, 1], [1, -1]]

この行列は |0⟩ と |1⟩ を次の2つの状態に送ります。

H|0⟩ = (|0⟩ + |1⟩) / √2
H|1⟩ = (|0⟩ - |1⟩) / √2

最初の式だけを見ると、Hゲートは単に「0を半々の確率にする」だけのように見えるかもしれません。
しかし核心は確率ではなく振幅です。Hゲートは入力の振幅を2つの出力基底状態へ分けて送り、同じ出力基底状態に届いた振幅の寄与を足し合わせます。

入力基底状態Hゲート適用後の状態すぐに測定すると
|0⟩(|0⟩ + |1⟩) / √20と1がそれぞれ50%
|1⟩(|0⟩ - |1⟩) / √20と1がそれぞれ50%

Hゲートの正確な振幅変換ルール

適用前の状態を |ψ⟩ = a₀|0⟩ + a₁|1⟩ としてみましょう。
Hゲートは a₀ だけを単独で見たり、a₁ だけを単独で見たりはしません。
2つの入力振幅がどちらも適用後の |0⟩ 成分と |1⟩ 成分に寄与し、同じ成分に届いた寄与が足し合わされます。

H(a₀|0⟩ + a₁|1⟩) = ((a₀ + a₁)|0⟩ + (a₀ - a₁)|1⟩) / √2
出発する入力振幅|0⟩ へ向かう寄与|1⟩ へ向かう寄与
適用前の |0⟩ の振幅 a₀+ a₀ × 1/√2+ a₀ × 1/√2
適用前の |1⟩ の振幅 a₁+ a₁ × 1/√2- a₁ × 1/√2
マイナス符号が付く位置 4つの入力・出力の組み合わせのうち、適用前の |1⟩ の振幅が適用後の |1⟩ 成分へ向かう寄与にだけマイナス符号が付きます。
いきなり「1→1 の項」と言うと曖昧なので、本書では「適用前の |1⟩ から来て適用後の |1⟩ へ向かう振幅の寄与」とかみ砕いた言い方で呼ぶことにします。

|0⟩ 状態にHゲートを適用してみよう

|0⟩ 状態は a₀ = 1、a₁ = 0 の状態です。
この値をHゲートの変換ルールに入れると、適用後の |0⟩ 成分と |1⟩ 成分はどちらも 1/√2 になります。

適用後の |0⟩ の振幅 = 1 × 1/√2 + 0 × 1/√2 = 1/√2
適用後の |1⟩ の振幅 = 1 × 1/√2 + 0 × (-1/√2) = 1/√2

そのため H|0⟩ は (|0⟩ + |1⟩) / √2 になり、すぐに測定すると0と1がそれぞれ50%の確率で出ます。
ここまでが、よく「Hゲートが重ね合わせを作る」と言われる部分です。

なぜマイナス符号が必要なのか？

第9章では、量子ゲートは元に戻せなければならないと見ました。
この条件をHゲートに当てはめると、|0⟩ 入力と |1⟩ 入力が同じ出力状態に合流してはいけません。
異なる2つの入力状態は、Hゲートを通ったあとも互いに区別できる2つの出力状態として残る必要があります。

ここで、上で見たHゲートの変換ルールをもう一度見ます。
4つの振幅の寄与のうち、適用前の |1⟩ から来て適用後の |1⟩ へ向かう振幅の寄与にだけマイナス符号が付きます。
短く言えば、1から来て1へ向かう寄与、つまり 1 → 1 の寄与です。

ここでの 1 → 1 の意味 1 → 1 は、適用前には |1⟩ 方向の振幅で、適用後にも |1⟩ 方向の振幅として寄与する、という意味です。
数字の1を数字の1に変えるという意味ではなく、振幅がどの方向から来て、どの方向へ向かうのかを示す言い方です。
マイナス符号がなければ

2つの入力が同じ出力に合流します

H|0⟩ = (|0⟩ + |1⟩) / √2
H|1⟩ = (|0⟩ + |1⟩) / √2

出力がどちらも (|0⟩ + |1⟩) / √2 になります。
そうなると、出力状態だけを見ても元の入力が |0⟩ だったのか |1⟩ だったのか復元できません。

マイナス符号があれば

2つの入力は互いに区別できる出力として残ります

H|0⟩ = (|0⟩ + |1⟩) / √2
H|1⟩ = (|0⟩ - |1⟩) / √2

2つの出力状態は同じ状態ではありません。
片方は |0⟩ 方向と |1⟩ 方向が同じ符号で、もう片方は |1⟩ 方向だけが反対符号です。

2つの出力状態が本当に区別できるかどうかは、どれくらい重なっているかを計算すると確認できます。
この重なりの計算は、正式には内積と呼びます。
今の例では振幅がすべて実数なので、同じ方向の振幅どうしを掛けてから足す、と考えれば大丈夫です。

重なり = (1/√2 × 1/√2) + (1/√2 × -1/√2)
= 1/2 - 1/2 = 0

重なりが0の状態を、正式には直交する状態と呼びます。
つまりHゲートのマイナス符号は、式の見た目を整えるための記号ではなく、|0⟩ 入力の出力と |1⟩ 入力の出力を直交させるために必要な符号です。

ここがポイント Hゲートが元に戻せる量子ゲートであるためには、異なる入力状態が同じ出力状態に合流してはいけません。
標準的なHゲートの表記では、この条件を満たすためのマイナス符号が、適用前の |1⟩ から来て適用後の |1⟩ へ向かう振幅の寄与に置かれます。

Hゲートを2回適用するとなぜ元に戻るのか？

|0⟩ 状態から始めると、1回目のHゲートの後の状態は (|0⟩ + |1⟩) / √2 です。
したがって2回目のHゲートを適用する直前は a₀ = 1/√2、a₁ = 1/√2 です。
この値を同じ変換ルールにもう一度入れれば終わりです。

2回目のHの後の |0⟩ の振幅 = (1/√2) × 1/√2 + (1/√2) × 1/√2 = 1/2 + 1/2 = 1
2回目のHの後の |1⟩ の振幅 = (1/√2) × 1/√2 + (1/√2) × (-1/√2) = 1/2 - 1/2 = 0
Hゲート(Hゲート(|0⟩)) = |0⟩

この計算では、|0⟩ 成分へ向かう2つの振幅の寄与は強め合い、|1⟩ 成分へ向かう2つの振幅の寄与は打ち消し合います。
そのためHゲートを2回適用しても「ランダムが2回起きてまた0になる」のではなく、振幅の強め合いと打ち消し合いによって、正確に最初の状態に戻ります。

Qoraで実習してみよう

Hゲートを2回適用してみよう

operation Main() {
    // キュービットを1つ準備
    // 準備直後の q[0] は |0⟩ 状態
    use q = Qubit[1];

    // 1回目のHゲートを適用
    // 振幅が測定値 0 と 1 に分かれる
    H(q[0]);

    // 2回目のHゲートを適用
    // 強め合いと打ち消し合いが起きて再び |0⟩ 状態になる
    H(q[0]);

    // 理想的な実行環境: 測定値 0
    bit result = M(q[0]);
}
