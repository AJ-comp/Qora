第13章. Zゲートは何の符号を変えるのか？

Z ゲートは、測定値 0 の振幅はそのままにして、測定値 1 の振幅にだけマイナスの符号を付けるゲートです。
そのため、Z ゲートの直後にすぐ測定だけをしてみると、何も起きていないように見えることがあります。
しかし後で H ゲートのように振幅を再び混ぜるゲートに出会うと、その符号の違いが測定結果を変えます。

入力状態Z ゲート適用後の状態すぐに測定すると？
|0⟩|0⟩測定値 0
|1⟩-|1⟩測定値 1

測定確率は、振幅の長さを2乗して得られます。
1 と -1 は符号が異なりますが長さは同じなので、すぐに測定するとどちらも同じ確率になります。
そのため Z ゲートの効果は、「確率を変える効果」というより「後で干渉するときに現れる符号の変化」として読むのがよいでしょう。

Z ゲートの効果は H ゲートに再び出会うときに現れます

まず H ゲートで |0⟩ 状態を (|0⟩ + |1⟩) / √2 状態にします。
次に Z ゲートを適用すると、測定値 1 の振幅の符号が変わって (|0⟩ - |1⟩) / √2 状態になります。
最後に H ゲートをもう一度適用すると、打ち消し合いと強め合いの向きが変わり、最終状態は |1⟩ になります。

|0⟩ → H ゲート → (|0⟩ + |1⟩) / √2 → Z ゲート → (|0⟩ - |1⟩) / √2 → H ゲート → |1⟩
Qoraで実習してみよう

H、Z、H の順に適用してみよう

operation Main() {
    // キュービットを1つ準備
    // 準備直後の q[0] は |0⟩ 状態
    use q = Qubit[1];

    // H ゲートを適用
    // 測定値 0 と 1 の振幅がどちらも 1/√2
    H(q[0]);

    // Z ゲートを適用
    // 測定値 1 の振幅の符号が負に変わる
    Z(q[0]);

    // H ゲートを適用
    // 符号の違いが干渉として現れ、|1⟩ 状態になる
    H(q[0]);

    // 理想的な実行環境: 測定値 1
    bit result = M(q[0]);
}
