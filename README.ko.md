# Qora

**양자 프로그래밍을 배우기 좋게 만든 학습용 언어.**

**Qora는 양자 프로그래밍을 배우기 위한 학습용 언어입니다.** 양자 코드는 보통 Qiskit이나 Cirq 같은
Python 라이브러리로 작성하거나, 이론을 이미 알고 있다고 전제하는 연구용 언어로 작성하게 되는데,
처음 배우는 입장에서는 빠른 실험과 정확한 개념 사이를 오가기가 쉽지 않습니다.

그래서 Qora는 그 사이를 메우는 데 초점을 맞춥니다. 익숙한 형태의 프로그래밍 문법으로 회로를 작성하면, 컴파일러가 먼저 “이 프로그램이 정말 양자 컴퓨터가 할 수 있는 일인가?”를 확인합니다.
그다음 표준 **OpenQASM 3**를 만들어서, 결과를 한 줄씩 읽어보거나 실제 툴체인에 넘길 수 있게 해줍니다.

이 방향은 설계 기준에도 그대로 이어집니다.

- **에러가 출력보다 먼저 나옵니다.** 컴파일을 통과했다면 그 OpenQASM은 유효합니다.
- **컴파일러 내부를 확인할 수 있습니다.** 명령 하나로 소스가 AST, 타입 있는 IR, 심볼 테이블, 합성된 역연산, QASM으로 바뀌는 과정을 볼 수 있습니다.
- **양자 규칙은 컴파일러가 직접 검사합니다.** 이미 사용한 큐비트를 다시 사용하거나, 측정을 되돌리거나, 단일 큐비트 자리에 레지스터를 넘기면 컴파일 단계에서 막힙니다.

## 바로 시작하기

가장 빠른 길은 VS Code 확장으로 시작하는 것입니다.
컴파일러가 확장 안에 통째로 들어 있어서 **따로 설치할 것이 없고**, .NET이나 Python도 필요하지 않습니다.

1. VS Code 마켓플레이스에서 [Qora Language](https://marketplace.visualstudio.com/items?itemName=qora-lang.qora-language)  확장을 설치합니다.
2. `bell.qor` 파일을 만들고 위 예제를 붙여넣습니다.
3. 저장하면 바로 아래 기능을 사용할 수 있습니다.

- **타이핑하는 동안** 에러에 밑줄이 생기고, 밑줄에 마우스를 올리면 이유를 볼 수 있습니다. `CNOT(q[0], q[0])`도 한번 입력해 보세요.
- 게이트나 키워드에 마우스를 올리면 `H`, `use`, `M` 같은 항목의 짧은 설명이 뜹니다.
- **`Ctrl+Shift+P` → `Qora: Transpile to OpenQASM`** 명령을 실행하면 컴파일 결과가 옆 편집기에 열립니다.
- **`Qora: Show Compilation Stages`** 명령은 소스 → AST → IR → 심볼 테이블 → (역 IR) → QASM 흐름을 저장할 때마다 새로 보여줍니다.

저장소에서 직접 실행하려면 .NET 10 SDK만 있으면 됩니다.

```bash
git clone https://github.com/AJ-comp/Qora.git
cd Qora
dotnet run --project src/Qora          # 데모 프로그램을 파싱하고 OpenQASM을 출력
```

VS Code 확장도 같은 바이너리를 `qora --json`으로 실행하며, 컴파일 한 번마다 `{success, qasm, errors[]}` 형태의 JSON 한 줄을 받습니다.

## 5분 만에 훑는 Qora

**큐비트와 게이트.** 새 큐비트 레지스터는 `use q = Qubit[N];` 형태로 할당하고, 배열처럼 인덱스로 접근합니다.
여기서 `use`는 C#의 `using`처럼 블록 끝에서 자동 해제를 표현한다기보다, Qora 프로그램 안에 사용할 큐비트 레지스터를 만드는 구문입니다.
게이트 호출은 함수 호출과 같은 모양입니다.

```
use q = Qubit[3];              // 큐비트 레지스터 q를 할당, 크기는 3
H(q[0]);                       // 중첩
CNOT(q[0], q[1]);              // 얽힘
Rx(pi/2, q[2]);                // 회전 게이트는 각도가 먼저
Reset(q[2]);                   // 다시 |0⟩으로
```

내장 게이트: `H X Y Z S T` · `CNOT/CX CY CZ SWAP CCX` · `Rx Ry Rz` · `Reset ResetAll`.

**측정**은 큐비트의 측정 결과를 고전 `bit`에 담는 연산이며, Qora에서 `bit`를 얻는 방법도 측정뿐입니다.

```
bit r = M(q[0]);               // 측정, 되돌릴 수 없습니다
r = M(q[0]);                   // 같은 bit에 다시 측정
```

**고전 제어흐름**은 익숙한 모양 그대로 읽으면 되고, 측정 결과에 반응하는 회로도 작성할 수 있습니다.

```
const int n = 2;
var count = 0;
for i in 0..n {  H(q[i]);  }
if (r == 1) {  X(q[1]);  } else {  Z(q[1]);  }
while (count < 2) {  count = count + 1;  }
repeat {  H(q[0]);  r = M(q[0]);  } until (r == 1);
```

사용자 정의 연산은 이름과 매개변수 뒤에 블록을 붙여 작성하고, `Main`이 진입점입니다. 그리고 **함자(functor)** 는 사용자 정의 연산이나 게이트를 변형합니다.

```
Prep(Qubit[2] q)
{
    H(q[0]);
    Controlled X(q[0], q[1]);   // 어떤 게이트에든 제어를 추가
    T(q[1]);
}

Adjoint Prep(q);                // ★ Prep의 역연산을 컴파일러가 합성
                                //   게이트는 역순으로 뒤집고, for 루프는 거꾸로 진행
                                //   if 분기는 내부를 역전, 중첩 호출은 전이적으로 처리
```

이 `Adjoint` 한 줄은 Qora의 대표 기능 가운데 하나로, “이 계산을 되돌려줘”라는 요청을 코드 한 줄로 표현하면 컴파일러가 실제 역방향 회로를 만들어줍니다.
또한 이 기능은 [자동 uncomputation](docs/TODO.md)으로 가는 출발점이기도 합니다.

**네임스페이스와 import**를 사용하면 큰 프로그램을 여러 파일로 나눌 수 있습니다.
`import`는 Python처럼 파일 경로를 문자열로 적습니다.

`gates_lib.qor`

```qora
namespace GatesLib
{
    Bell(Qubit[2] q)
    {
        H(q[0]);
        CNOT(q[0], q[1]);
    }
}
```

`rotations.qor`

```qora
namespace Rotations
{
    Tilt(Qubit q)
    {
        Ry(pi/4, q);
        Rz(pi/8, q);
    }
}
```

`main.qor`

```qora
import "gates_lib.qor";         // 같은 디렉터리의 gates_lib.qor를 불러옴
import "rotations.qor";         // 같은 디렉터리의 rotations.qor를 불러옴

namespace App
{
    open GatesLib;
    open Rotations;

    Prepare(Qubit[2] q)
    {
        Bell(q);                // open으로 GatesLib.Bell을 이름만으로 호출
        Tilt(q[0]);             // open으로 Rotations.Tilt를 이름만으로 호출
    }
}

Main()
{
    use q = Qubit[2];
    App.Prepare(q);             // App 네임스페이스의 연산을 정규화해서 호출
    Rotations.Tilt(q[1]);       // import한 다른 파일의 연산을 네임스페이스로 직접 호출
}
```

이름 해소 규칙은 네임스페이스를 사용하는 익숙한 프로그래밍 언어의 방식과 비슷합니다.
이름은 자기 네임스페이스, 전역 네임스페이스, `open`한 네임스페이스 순서로 찾습니다.
그런데 `open`한 두 곳에 같은 이름이 있으면 몰래 하나를 고르지 않고, 후보를 보여주는 컴파일 에러를 냅니다.

`import "lib/gates.qor";`는 현재 파일이 있는 디렉터리를 기준으로 `lib/gates.qor`를 읽습니다.
경로는 따옴표 안에 그대로 적고, 확장자도 포함합니다.
같은 파일은 여러 경로로 도달해도 한 번만 로드되고, 순환 import가 생기면 그 고리를 에러 메시지로 보여줍니다.

내장 게이트 이름도 같은 규칙을 따르며, 내장 게이트는 암묵적인 `Qora.Intrinsic` 네임스페이스에 살기 때문에, 네임스페이스 안에서는 `Rx` 같은 이름의 연산을 직접 정의할 수 있습니다.
다만 어떤 호출이 사용자 정의 연산과 내장 게이트 둘 다를 뜻할 수 있으면 명시적인 모호성 에러가 납니다.
이때는 사용자 정의 연산은 `L.Rx(q)`, 내장 게이트는 `Qora.Intrinsic.Rx(pi/2, q)`처럼 적어주면 됩니다.

스코프 안에 같은 이름의 사용자 후보가 없다면 맨이름 게이트는 그대로 내장 게이트를 뜻합니다. 따라서 문서에서 배운 게이트 이름을 프로그램에서도 그대로 사용할 수 있습니다.

**오류가 있으면 컴파일러가 이유를 알려줍니다.** 그리고 출력 전에, 한 번의 컴파일에서 모든 위반을 모아 보여줍니다.

```
[QSEM001] in `Main`: `Adjoint Meas` cannot be compiled: it measures a qubit,
          and measurement is irreversible
[QSEM006] in `Main`: `Bell` expects 1 argument(s) but got 2
[QSEM016] in `Main`: index `q[5]` is out of range; `q` has 2 qubit(s) (valid: 0..1)
[QSEM018] in `Go`: `Flip` is ambiguous here: it could be `LibA.Flip` or `LibB.Flip`
          — qualify the call (e.g. `LibA.Flip(...)`)
[QSEM021] imports form a cycle: main.qor -> lib.qor -> main.qor — remove one of these imports
```

**프로그램 실행하기.** 사용자 정의 연산, 루프, 고전 변수, `while`까지 포함한 Qora 프로그램을 지금 바로
Amazon Braket 로컬 시뮬레이터에서 실행할 수 있습니다.

```bash
pip install amazon-braket-sdk            # Python 3.10–3.13
python tools/run_braket.py yourfile.qor --shots 1000
```

두 실행 경로는 실제 측정 통계로 릴리스 검증에도 사용됩니다.
`tools/braket_validation.py`는 전체 사용자 정의 연산의 `Adjoint`가 상태를 `|0…0⟩`으로 되돌리는 것까지 포함해 전체 언어가 실행됨을 확인하고, `tools/qiskit_validation.py`는 Qiskit의 `qasm3.loads(...)`가 받아들이는 평평한 부분집합을
검증합니다.
다만 Qiskit 임포터는 아직 `def`, `int`, 변수 루프 인덱스를 읽지 못하기 때문에, Qiskit과 실제 하드웨어로 이어질 평탄화 출력 모드를 추가할 예정입니다.

## 컴파일러 흐름

단계별 정식 설명은  **[Qora 컴파일 파이프라인 설계](https://aj-comp.github.io/Qora/ko/architecture.html)** 문서에 있습니다.

간단히 말하면 Qora는 파서가 만든 AST를 자기 IR로 옮긴 뒤, 그 IR 위에서 이름 해소, 검증, 특수화,
역합성, 이름 정리, 출력을 차례로 진행합니다.

| 묶음 | 단계 | 산출물 | 역할 |
|---|---|---|---|
| 1. 읽기 | 소스 → AST | 엔진 AST | Janglim 파서가 소스 텍스트를 문법 트리로 변환 |
| 2. 프로그램 넘겨받기 | AST → Qora IR | 타입 있는 IR | `QoraLowering`이 파서 트리를 Qora 전용 구조로 재구성 |
| 3. 이름 해소 | import 로드 → 병합 → 이름 해소 | FQN IR | 여러 파일을 합치고 모든 호출 대상을 완전한 이름으로 확정 |
| 4. 의미 검증 | 검증 | 에러 목록 또는 깨끗한 IR | 문제가 있으면 모든 에러를 모아서 보고한 뒤 중단 |
| 5. 변환 | 제네릭 특수화 → adjoint 실체화 | 출력 준비 IR | `Qubit[n]`을 구체적인 크기로 만들고, `Adjoint` 호출을 합성된 역연산으로 변환 |
| 6. 출력 준비 | 이름 맹글링 → 참조 검사 | OpenQASM 안전 IR | OpenQASM 이름 충돌을 피하고 모든 참조가 남아 있는지 확인 |
| 7. 출력 | OpenQASM 출력 | OpenQASM 3 | 결정이 끝난 IR을 마지막 이미터가 텍스트로 출력 |

- **타입 있는 IR**(`src/Qora.Core/Ir/`)은 처음부터 끝까지 컴파일러가 소유하며, 파서 엔진은 lowering 경계에서 역할을 마칩니다.
- **모듈 로더**는 `import` 그래프 전체를 하나의 프로그램으로 병합합니다.<br>그 뒤 **리졸버**가 연산 호출 대상을 완전한 이름(FQN)으로 고정하고, 이후 모든 호출부는 딱 하나의 연산을 가리킵니다.<br>네임스페이스를 쓰는 다른 프로그래밍 언어들처럼 모호하면 에러가 나며, 자세한 내용은 [설계 문서](docs/namespaces-design.md)에 있습니다.
- **검증기**(QSEM001–025)는 수집형입니다.<br>한 번의 컴파일에서 인자 종류와 개수, 레지스터 크기, 인덱스 범위, 예약 이름, 재귀 호출, 잘못 놓인 `use`, 선언되지 않았거나 선언 전에 사용한 식별자, 되돌릴 수 없는 `Adjoint` 대상까지 한꺼번에 보고합니다.<br>또한 검증기가 사용하는 통합 심볼 테이블도 컴파일 단계 뷰에 같이 나옵니다.
- **네임 맹글러**는 네임스페이스의 점을 평탄화하고(`MyLib.Bell` → `MyLib_Bell`), 출력되는 식별자가 키워드·stdgates 게이트 이름·이미 출력된 다른 이름과 부딪힐 때만 `_`를 덧붙입니다. 따라서 에러와 컴파일 단계 뷰는 원래 이름/FQN을 유지하고, 충돌을 피한 이름은 QASM에서만 보입니다.
- **Adjoint 실체화기**는 순수 IR→IR 인버터를 사용해 전체 사용자 정의 연산의 `Adjoint` 호출을 합성된 역연산으로 바꾼 뒤 이름 맹글링 단계로 넘깁니다. 나중에 자동 uncomputation을 주입할 패스도 이 구조를 그대로 재사용합니다.
